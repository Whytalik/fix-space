# 🚀 FIX Space

> **Персоналізована веб-платформа для професійної організації діяльності CFD-трейдера.** 📈

FIX Space — це спеціалізована інформаційна система, створена для вирішення проблеми інформаційної фрагментації в роботі приватного трейдера. Платформа об'єднує журнал угод, аналітику сесій, базу знань та систему відстеження помилок в єдиному гнучкому середовищі.

---

## 🎯 Проблема: Інформаційна фрагментація

Сучасний трейдер змушений розподіляти інформацію між 4–7 незалежними сервісами (Excel, Notion, TradingView, брокерські виписки, паперові нотатки). Це призводить до:
- **Когнітивного навантаження:** втрата до 40% продуктивності через переключення контексту.
- **Неможливості ретроспективного аналізу:** дані існують у різних форматах без взаємозв'язку.
- **Прихованих поведінкових патернів:** важко встановити кореляцію між емоційним станом, порушенням правил та фінансовим результатом.

**FIX Space вирішує це**, надаючи інтегроване середовище, де кожен аспект діяльності трейдера (від планування до рефлексії) пов'язаний в єдину модель даних.

---

## ⚡ Ключові можливості

- **📊 Професійний Журнал Угод:** Повна фіксація параметрів (PnL, RR, MAE/MFE, комісії) з автоматичним розрахунком Net PnL та відхилення від плану.
- **🧠 Система аналізу рутин:** Спеціалізовані бази `Daily Routine` та `Routine Library` для структурування передсесійної підготовки та пост-сесійного огляду.
- **⚠️ Трекер помилок:** База `Mistakes` з алгоритмом автоматичного визначення серйозності (Severity) на основі частоти та фінансового впливу.
- **⚙️ Динамічна структура:** Можливість створювати власні типи властивостей, формули та автоматизації без написання коду.
- **🖼️ Контентна область:** Потужний редактор для створення розборів угод з вбудованими графіками, скріншотами та калькуляторами ризику.

## 🚀 Швидкий старт

### Попередні вимоги
- Node.js v20+
- pnpm v9+
- Docker Desktop (для локальної БД)

### Встановлення та запуск
1. **Клонування та інсталяція:**
   ```bash
   git clone <repository-url>
   pnpm install
   ```
2. **Налаштування оточення:**
   Створіть `apps/api/.env.development` (використовуйте `.env.example` як зразок).
3. **Запуск інфраструктури:**
   ```bash
   docker-compose up -d
   turbo db:generate
   turbo db:migrate:dev
   ```
4. **Запуск розробки:**
   ```bash
   turbo dev
   ```
