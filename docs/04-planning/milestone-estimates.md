# Milestone Estimates

## Припущення

| Параметр               | Значення                                                          |
| ---------------------- | ----------------------------------------------------------------- |
| Доступний час          | 4–5 год/день (суміщення з навчанням)                              |
| Одиниця вимірювання    | Робочі дні                                                        |
| Методологія оцінювання | Декомпозиція задач по шарах: DTO → DB → Service → Controller → UI |
| Буфер на тестування    | 20–30% від оцінки основного часу                                  |

Буфер охоплює: написання unit-тестів, перевірку в Postman, оновлення тест-кейсів і RTM після кожної фічі. Тестування ведеться паралельно з розробкою — не окремою фазою наприкінці.

## Склад milestone

### M0 — Аналіз та планування

Формування вимог (SRS), архітектурне проектування, схема бази даних, вибір технологічного стеку, тест-стратегія, Kanban-дошка, ризик-реєстр.

Результат: підписані вимоги, затверджена архітектура, Prisma schema v1.

---

### M1 — v0.1 MVP Core

**Auth:** реєстрація, верифікація email, вхід, refresh-токен, вихід, скидання пароля.  
**Workspace:** Space (CRUD), Section (CRUD), Database (CRUD).  
**Property:** 12 типів (TEXT, NUMBER, DATE, CHECKBOX, DURATION, SELECT, STATUS, RELATION, FORMULA, RATING, PROGRESS, BUTTON), type handlers, валідація.  
**Record:** CRUD, значення властивостей, фільтрація, пошук.  
**View:** серверне збереження конфігурації відображення, фільтри, сортування.  
**Template:** CRUD, шаблон за замовчуванням, TemplatePropertyValue.  
**Content:** контентна область запису (get + update).  
**Formula:** 10 пресетів + покроковий конструктор.  
**Ініціалізація:** стартовий набір баз даних і властивостей при реєстрації.

Результат: повністю функціональний REST API для основного домену.

---

### M2 — v0.2 Feature Complete

**Settings:** 8 груп налаштувань (профіль, безпека, зовнішній вигляд, простір, секції, бази, записи, інтеграції).  
**Statistics:** торгова статистика (Win Rate, Profit Factor, Expectancy), власні звіти.  
**Search:** пошук у базі та глобальний пошук (Ctrl+K).  
**Import/Export:** CSV-імпорт (до 25 МБ) і CSV-експорт.  
**Automation:** правила КОЛИ→ТО (3 тригери, 3 дії, до 5 дій на правило, журнал виконань).  
**Button:** BUTTON-властивість і BUTTON-компонент контентної області.

Результат: система готова до повноцінної щоденної роботи трейдера.

---

### M3 — v0.3 Polished

**Onboarding:** демо-простір, покроковий тур (10–12 кроків), чеклист активації, контекстні підказки.  
**Notification:** in-app сповіщення (інтеграція, автоматизація, система); badge-лічильник, flyout-панель.  
**Duplicate:** дублювання простору, бази, запису.  
**Content blocks:** бібліотека блоків, 19 типів компонентів, LINKED_VIEW, трейдерські компоненти (CHART, POSITION_SIZE_CALC, ECONOMIC_CALENDAR).

Результат: продукт готовий до першого реального користувача.

---

### M4 — v1.0 Integrations

**Торгові підключення:** Binance, Bybit, OKX (read-only API Key + Secret), MetaTrader 5 (Investor Password), cTrader (OAuth 2.0).  
**Синхронізація:** авто-синхронізація кожні 5 хв, ручний запуск, вибір діапазону дат, вибір типу ринку, обробка помилок (3 невдалих спроби → сповіщення).  
**Безпека:** credentials зашифровано, ключ з правами торгівлі відхиляється.

Результат: автоматичний імпорт закритих позицій із реальних торгових рахунків.

---

### M5 — Документація та звітність

Тест-план, тест-кейси, Postman-колекція, RTM, coverage report, bug reports, архітектурна документація, user guide, deployment guide.

Результат: повний комплект документів для захисту дипломної роботи.
