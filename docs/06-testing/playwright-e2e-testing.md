# E2E UI Тестування: Playwright

Цей документ описує архітектуру, конфігурацію та інструкції з написання та запуску автоматизованих End-to-End (E2E) UI-тестів для веб-інтерфейсу системи **FIX Space** на базі фреймворку **Playwright**.

---

## 1. Загальний огляд

Playwright використовується для повного функціонального тестування користувацького досвіду (UI/UX) у реальних браузерах. На відміну від компонентних тестів у `jsdom` (Jest), Playwright запускає реальний додаток Next.js і виконує тестування за допомогою наступних браузерних рушіїв:

- **Chromium** (Google Chrome, Microsoft Edge)
- **Firefox** (Mozilla Firefox)
- **WebKit** (Apple Safari)

Тести розташовані в директорії `apps/web/e2e`.

---

## 2. Конфігурація та налаштування

Конфігураційний файл знаходиться за шляхом `apps/web/playwright.config.ts`.

### Ключові параметри конфігурації:

- **`baseURL`**: `http://localhost:3001` — порт Next.js додатка у режимі розробки.
- **`fullyParallel`**: `true` — усі тести всередині файлів запускаються паралельно на доступних ядрах процесора для прискорення.
- **`webServer`**: Автоматично запускає локальний dev-сервер (`pnpm run dev` на порту `3001`) перед початком виконання тестів та зупиняє його після завершення.
- **Ізоляція середовища (змінна `CI`)**:
  - На локальній машині тести виконуються паралельно на багатьох ядрах, без повторних спроб (`retries: 0`), а сервер розробки перевикористовується, якщо він уже запущений.
  - На CI-сервері (наприклад, GitHub Actions) кількість воркерів обмежується до `1` для запобігання перевантаженню, додається `2` повторних спроби для нестабільних (flaky) тестів, а Next.js сервер щоразу піднімається з нуля.

## 3. Запуск тестів (Команди)

Всі команди рекомендується запускати з кореня монорепозиторію через Turbo або Pnpm.

### 3.1 Запуск у фоновому режимі (Headless Mode)

Цей режим запускає браузери у фоні без графічного вікна (чудово підходить для CI або швидкої локальної перевірки):

```bash
pnpm --filter @fixspace/web test:e2e
```

### 3.2 Запуск в інтерактивному інтерфейсі (UI Mode)

Запускає графічне вікно Playwright UI Mode. Це основне середовище для розробки та дебагу тестів. Ви можете бачити таймлайн виконання, скріншоти кожного кроку та стан DOM:

```bash
# Перейдіть у папку додатка
cd apps/web
pnpm playwright test --ui
```

### 3.3 Автоматична генерація тестів (Codegen)

Playwright може автоматично генерувати код тесту на основі ваших дій у браузері. Ви клікаєте по сайту, а він пише TypeScript код:

```bash
cd apps/web
pnpm playwright codegen http://localhost:3001
```

## 4. Найкращі практики тестування з Playwright

1. **Використовуйте локатори за ролями (Role Locators)**:
   Надавайте перевагу стійким селекторам, які відображають доступність сторінки для користувачів, замість CSS-класів чи XPath:
   - `page.getByRole('button', { name: /увійти/i })` — **Рекомендовано**
   - `page.locator('.submit-btn-class')` — **Не рекомендовано**

2. **Довіряйте авто-очікуванню (Auto-waiting)**:
   Playwright автоматично чекає, поки елемент з'явиться на сторінці, стане видимим та клікабельним. Не використовуйте штучні таймаути (на зразок `await page.waitForTimeout(2000)`), використовуйте веб-асерти:

   ```typescript
   await expect(page.getByText("Успішно збережено")).toBeVisible();
   ```

3. **Ізоляція тестів**:
   Кожен тест виконується в абсолютно новому, ізольованому браузерному контексті (чисті куки, сховище). Уникайте залежностей між тестами.

---

## 5. Приклад тесту

Нижче наведено приклад тесту авторизації (`apps/web/e2e/auth.spec.ts`):

```typescript
import { test, expect } from "@playwright/test";

test.describe("Authentication Flows", () => {
  test("should load the login page successfully", async ({ page }) => {
    // 1. Переходимо на сторінку входу
    await page.goto("/login");

    // 2. Перевіряємо, чи перенаправило на локалізований роут
    await expect(page).toHaveURL(/.*login/);

    // 3. Зчитуємо та виводимо заголовок
    const title = await page.title();
    console.log("Loaded login page with title:", title);

    // 4. Очікуємо, що назва сторінки відповідає системній
    await expect(page).toHaveTitle(/.*login|.*увійти|.*вхід|.*fix.*space/i);
  });
});
```
