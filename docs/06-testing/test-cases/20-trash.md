### TC-TRASH-006: RELATION chip — приглушений вигляд при переміщенні в Кошик

| Поле         | Значення            |
| ------------ | ------------------- |
| **US**       | US-046              |
| **Issue**    | #190                |
| **TS**       | —                   |
| **Метод**    | Manual (Browser)    |
| **Техніка**  | Exploratory Testing |
| **Priority** | P2                  |

**Кроки:**

1. Створити записи A і B; у A встановити RELATION на B
2. Видалити запис B (переміщується до Кошика)
3. Відкрити запис A; перевірити RELATION chip
4. Відновити запис B з Кошика
5. Перевірити RELATION chip у A знову

**Очікуваний результат:**

- Крок 3: chip присутній, але відображається приглушено (dimmed), клік неактивний
- Крок 5: chip повертається до звичайного вигляду, навігація активна

### TC-TRASH-007: Перегляд та відновлення запису з кошика (Playwright UI E2E)

| Поле         | Значення            |
| ------------ | ------------------- |
| **US**       | US-134              |
| **Issue**    | #48                 |
| **TS**       | —                   |
| **Метод**    | E2E UI (Playwright) |
| **Техніка**  | Exploratory Testing |
| **Priority** | P2                  |

**Кроки:**

1. Увійти в систему та відкрити базу даних Trading Journal.
2. Створити новий тестовий запис із назвою `"Тимчасова угода для видалення"`.
3. У меню опцій цього запису обрати пункт "Видалити" (Move to Trash).
4. Переконатися, що запис зник із таблиці бази даних.
5. У боковому меню (Sidebar) знайти та відкрити розділ "Кошик" (Trash).
6. Знайти у списку видалених об'єктів запис `"Тимчасова угода для видалення"`.
7. Натиснути кнопку "Відновити" (Restore) поруч із назвою запису.
8. Повернутися до Trading Journal та переконатися, що запис `"Тимчасова угода для видалення"` знову відображається в таблиці та має всі збережені раніше властивості.

**Очікуваний результат:**

- Запис успішно переміщується до кошика, видаляється з активних переглядів, а при відновленні повертається до свого початкового стану без втрати даних.
