# Playwright UI E2E Tests: Properties

### [ ] TC-PROP-012: Відображення всіх 11 типів у table view

| Поле         | Значення            |
| ------------ | ------------------- |
| **US**       | US-036              |
| **Issue**    | #7, #8, #19         |
| **TS**       | TS-PROP-01          |
| **Метод**    | Manual (Browser)    |
| **Техніка**  | Exploratory Testing |
| **Priority** | P1                  |

**Кроки:**

1. Створити БД з 11 властивостями (TEXT, NUMBER, DATE, CHECKBOX, DURATION, SELECT, STATUS, RELATION, RATING, PROGRESS, FORMULA)
2. Відкрити table view
3. Перевірити що кожен тип відображається коректно

**Очікуваний результат:**

- TEXT: текстове поле
- NUMBER: число з форматуванням
- DATE: дата/час
- CHECKBOX: чекбокс
- DURATION: тривалість
- SELECT: випадаючий список
- STATUS: статус з кольором
- RELATION: посилання на запис
- RATING: зірки
- PROGRESS: прогрес-бар
- FORMULA: read-only значення

### [ ] TC-PROP-026: Додавання нової властивості до структури бази даних (Playwright UI E2E)

| Поле         | Значення                              |
| ------------ | ------------------------------------- |
| **US**       | US-032                                |
| **Issue**    | #9                                    |
| **TS**       | TS-PROP-01                            |
| **Метод**    | E2E UI (Playwright)                   |
| **Техніка**  | Equivalence Partitioning (valid path) |
| **Priority** | P1                                    |

**Кроки:**

1. Увійти в систему та відкрити створену кастомну базу даних
2. У табличному вигляді (Table View) натиснути на кнопку "+" в останній колонці таблиці
3. У меню вибору властивості ввести назву `"Об'єм угоди"`
4. Вибрати тип властивості `"NUMBER"`
5. Натиснути "Зберегти" або клікнути поза меню
6. Перевірити, що в таблиці з'явилася нова колонка `"Об'єм угоди"` з числовим форматом

**Очікуваний результат:**

- Нова колонка додається до структури таблиці в реальному часі
- Тип NUMBER коректно застосовується до комірок нової властивості
- Зміни структури зберігаються в базі даних та відображаються після оновлення сторінки

### [ ] TC-PROP-027: Видалення властивості через UI редактора (Playwright)

| Поле         | Значення            |
| ------------ | ------------------- |
| **US**       | US-034              |
| **Issue**    | —                   |
| **TS**       | TS-PROP-01          |
| **Метод**    | E2E UI (Playwright) |
| **Техніка**  | State Transition    |
| **Priority** | P1                  |

**Кроки:**

1. Відкрити сторінку Edit бази даних (`/database/:id/edit`)
2. У секції Properties знайти наявну кастомну властивість
3. Клікнути на іконку видалення або вибрати "Delete" з меню властивості
4. Підтвердити видалення у діалоговому вікні
5. Перевірити що колонка зникла з таблиці

**Очікуваний результат:**

- Властивість видаляється зі списку та зникає як колонка у Table View
- Після оновлення сторінки колонка відсутня та дані в БД видалені
