### TC-FORM-011: Автоматичний перерахунок формул у таблиці після зміни значень полів (Playwright UI E2E)

| Поле         | Значення            |
| ------------ | ------------------- |
| **US**       | US-098              |
| **Issue**    | #19                 |
| **TS**       | —                   |
| **Метод**    | E2E UI (Playwright) |
| **Техніка**  | Exploratory Testing |
| **Priority** | P1                  |

**Кроки:**

1. Увійти в систему та відкрити базу даних Trading Journal.
2. Знайти або створити запис з повною інформацією про угоду.
3. Переконатися, що колонка формули Net PnL відображає коректну різницю (наприклад, PnL - комісії - збори).
4. Подвійним кліком на клітинку властивості `PnL` (або `Commissions`) змінити її значення (наприклад, збільшити PnL на 100).
5. Зберегти зміну (натиснути Enter або клікнути поза межами клітинки).
6. Спостерігати за значенням у колонці Net PnL.
7. Перевірити, що формула автоматично перерахувала Net PnL відповідно до нової суми без оновлення сторінки.

**Очікуваний результат:**

- Зміна значення залежного поля миттєво тригерить реактивний перерахунок формули в UI та відображає актуальне нове значення.
