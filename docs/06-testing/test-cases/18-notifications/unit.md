### [ ] TC-NOTIF-U-001: NotificationService — створення сповіщення та валідація вмісту

| Поле         | Значення       |
| ------------ | -------------- |
| **US**       | US-072         |
| **Issue**    | #102           |
| **TS**       | —              |
| **Метод**    | Unit (Jest)    |
| **Техніка**  | Boundary Value |
| **Priority** | P2             |

**Кроки:**

1. Створити сповіщення з невідомим типом `"ALERT"`.
2. Створити сповіщення з порожнім текстом `""`.
3. Створити сповіщення з типом `"ERROR"` та валідним описом помилки синхронізації.

**Очікуваний результат:**

- Крок 1: викидається виняток `BadRequestException` (недійсний тип сповіщення).
- Крок 2: викидається виняток `BadRequestException` (опис події є обов'язковим).
- Крок 3: сповіщення успішно створюється в базі даних зі статусом `unread` та поточним часом.
