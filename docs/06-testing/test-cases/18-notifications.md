### TC-NOTIF-010: Отримання та відображення миттєвого сповіщення в інтерфейсі (Playwright UI E2E)

| Поле         | Значення            |
| ------------ | ------------------- |
| **US**       | US-130, US-131      |
| **Issue**    | #102, #103          |
| **TS**       | —                   |
| **Метод**    | E2E UI (Playwright) |
| **Техніка**  | Exploratory Testing |
| **Priority** | P2                  |

**Кроки:**

1. Увійти в систему під тестовим обліковим записом.
2. Відкрити іншу вкладку або зімітувати через фонову команду тригер сповіщення (наприклад, викликати помилку API-інтеграції).
3. Переконатися, що у верхньому правому кутку екрана з'явилося спливаюче повідомлення (Toast notification) про помилку інтеграції.
4. Перевірити наявність червоного або помаранчевого бейджа із цифрою `"1"` на іконці дзвоника (сповіщень) у верхньому меню.
5. Натиснути на іконку дзвоника, щоб відкрити панель сповіщень.
6. Переконатися, що бейдж зник, а сповіщення відображається у списку як прочитане.

**Очікуваний результат:**

- Сповіщення надходить миттєво через WebSockets або Server-Sent Events.
- Поп-ап та бейдж непрочитаних сповіщень синхронізуються в інтерфейсі в реальному часі.

### TC-NOTIF-011: Відображення та закриття панелі сповіщень (Playwright)

| Поле         | Значення                              |
| ------------ | ------------------------------------- |
| **US**       | US-130                                |
| **Issue**    | —                                     |
| **TS**       | —                                     |
| **Метод**    | E2E UI (Playwright)                   |
| **Техніка**  | Equivalence Partitioning (valid path) |
| **Priority** | P3                                    |

**Кроки:**

1. Увійти в систему
2. Клікнути іконку сповіщень у header
3. Перевірити відкриття панелі зі списком сповіщень або повідомленням "немає сповіщень"
4. Клікнути поза панеллю або натиснути ESC
5. Перевірити що панель закрилась

**Очікуваний результат:**

- Панель сповіщень відкривається та закривається без помилок
- Список відображається коректно (порожній або з наявними сповіщеннями)
