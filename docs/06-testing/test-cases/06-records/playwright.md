### [ ] TC-REC-029: Створення нового запису в табличному вигляді (Playwright UI E2E)

| Поле         | Значення                              |
| ------------ | ------------------------------------- |
| **US**       | US-043                                |
| **Issue**    | #11                                   |
| **TS**       | TS-REC-01                             |
| **Метод**    | E2E UI (Playwright)                   |
| **Техніка**  | Equivalence Partitioning (valid path) |
| **Priority** | P1                                    |

**Кроки:**

1. Увійти в систему та відкрити базу даних
2. Внизу таблиці натиснути кнопку "+ Додати запис" (New Record)
3. Перевірити, що в таблиці з'явився новий порожній рядок
4. Двічі клацнути на першу комірку (Name) та ввести текст `"Купівля EURUSD"`
5. Кликнути поза коміркою або натиснути Enter
6. Перевірити, що назва запису оновилася в базі даних

**Очікуваний результат:**

- Рядок запису миттєво додається до таблиці
- Введене значення зберігається, а інтерфейс відображає оновлену назву запису `"Купівля EURUSD"`
- При оновленні сторінки новий запис залишається на місці

### [ ] TC-REC-030: Фільтрація записів через UI панель (Playwright)

| Поле         | Значення                              |
| ------------ | ------------------------------------- |
| **US**       | US-046                                |
| **Issue**    | —                                     |
| **TS**       | TS-REC-02                             |
| **Метод**    | E2E UI (Playwright)                   |
| **Техніка**  | Equivalence Partitioning (valid path) |
| **Priority** | P1                                    |

**Кроки:**

1. Відкрити базу даних з кількома записами у Table View
2. Натиснути кнопку "Filter" на тулбарі
3. Додати умову фільтрації за текстовою властивістю (наприклад, "Name contains X")
4. Ввести значення що відповідає лише частині записів
5. Перевірити що таблиця відображає лише відповідні записи

**Очікуваний результат:**

- Кількість відображених рядків зменшується відповідно до умови фільтра
- Записи що не відповідають умові приховані без видалення з БД
- При очищенні фільтра всі записи знову відображаються

### [ ] TC-REC-031: Видалення запису через контекстне меню UI (Playwright)

| Поле         | Значення            |
| ------------ | ------------------- |
| **US**       | US-050              |
| **Issue**    | —                   |
| **TS**       | TS-REC-04           |
| **Метод**    | E2E UI (Playwright) |
| **Техніка**  | State Transition    |
| **Priority** | P1                  |

**Кроки:**

1. Відкрити базу даних з наявними записами у Table View
2. Навести курсор на рядок запису
3. Клікнути на кнопку "..." (more options) у рядку
4. Вибрати "Delete / Видалити" з контекстного меню
5. Підтвердити видалення якщо з'явиться діалог підтвердження

**Очікуваний результат:**

- Запис зникає з таблиці миттєво без перезавантаження сторінки
- Після оновлення сторінки видалений запис відсутній
