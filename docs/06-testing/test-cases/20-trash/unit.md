### [ ] TC-TRASH-U-001: TrashService — логічне видалення запису (Soft Delete)

| Поле         | Значення         |
| ------------ | ---------------- |
| **US**       | US-134           |
| **Issue**    | #48              |
| **TS**       | —                |
| **Метод**    | Unit (Jest)      |
| **Техніка**  | State Transition |
| **Priority** | P2               |

**Кроки:**

1. Викликати `moveToTrash(recordId)`.
2. Перевірити стан запису в базі даних.
3. Спробувати отримати запис через звичайний `findOne` без фільтрації `isDeleted`.

**Очікуваний результат:**

- Крок 1: запис оновлено.
- Крок 2: прапорець `isDeleted = true`, поле `deletedAt` встановлено на поточний час (ISO 8601).
- Крок 3: метод `findOne` повертає `null` або помилку `NotFoundException` (якщо застосовано глобальний soft-delete filter у Prisma/TypeORM).
