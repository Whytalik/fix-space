## Загальні вимоги до веб-інтерфейсу (Core UI Guidelines)

1. **Дизайн-система:** Всі сторінки та компоненти повинні базуватися на дизайн-системі (`docs/08-design-system/`). Заборонено використовувати випадкові ad-hoc класи чи кольорові токени.
2. **Адаптивність:** Кожен інтерфейс має бути адаптивним (mobile-first або responsive) та коректно відображатися на екранах від 320px до 1920px.
3. **Стани завантаження та помилок:** Жоден динамічний блок не повинен завантажуватися без скелетона (skeleton) або індикатора прогресу. Помилки API мають виводитися у людиночитаному вигляді з можливістю повторного запиту.
4. **Доступність (a11y):** Семантична HTML5 розмітка, використання `aria-*` атрибутів для складних інтерактивних елементів (конструктори, дропдауни, модальні вікна).
5. **Клавіатурна навігація:** Підтримка гарячих клавіш (наприклад, `Ctrl+K` для пошуку, `ESC` для закриття вікон, навігація стрілками у списках).
