# UI Specification — Сторінка запису

Документ описує інтерфейси та сценарії перегляду й редагування конкретного запису (рядка бази даних).

---

## 4.1 Запис <!-- US-043 -->

**Маршрут:** `/record/[id]`  
**Доступ:** Захищена (або публічна за секретним лінком, якщо ввімкнено «Поділитися»)  
**Призначення:** Детальний перегляд та редагування властивостей та контенту окремого запису.

### Макет сторінки

Двопанельний гнучкий макет:

- [ ] **Ліва панель (ширина 320px - 400px, фіксована):** Метадані та властивості запису.
- [ ] **Права панель (широка, адаптивна):** Відображення структурованого контенту (Record Content Area).
- [ ] _На мобільних пристроях панелі розташовуються вертикально одна під одною._

---

## 4.2 Елементи інтерфейсу <!-- US-043, US-045, US-053, US-054, US-055 -->

### 1. Шапка запису (Record Header)

- [ ] **Хлібні крихти:** `Назва простору › Назва бази › Назва запису`.
- [ ] **Назва запису:** Велике текстове поле (H1), доступне для інлайн-редагування. При кліці з'являється рамка, зміни зберігаються при втраті фокусу (blur) або натисканні `Enter`.
- [ ] **Іконка запису:** Поруч із назвою, при кліці відкриває емодзі-пікер для швидкої зміни.
- [ ] **Блок дій (Buttons Group):**
  - [ ] **Дублювати (Duplicate):** Створює копію запису. Дублюються лише значення властивостей. Контентна область нового запису залишається порожньою.
  - [ ] **Видалити (Delete):** Переміщує запис до Кошика (Trash).
  - [ ] **Поділитися (Share):** Відкриває поп-овер із тоглом `Публічний доступ`. При ввімкненні генерує унікальне публічне посилання (read-only) для сторонніх користувачів без авторизації.
  - [ ] **Знімки (Snapshots):** Відкриває бічну панель історії версій контенту.

### 2. Ліва панель: Властивості (Property Inspector)

- [ ] **Список властивостей:** Вертикальний перелік полів «Назва властивості — Значення».
- [ ] **Назва властивості:** Невеликий текст, іконка типу поля. Якщо структура бази не заблокована, при наведенні з'являється шестірня налаштування властивості.
- [ ] **Введення значень (Inline inputs):** Клік на значення властивості активує відповідний тип вводу:
  - [ ] `TEXT`: Просте поле вводу.
  - [ ] `NUMBER`: Поле з валідацією цифр, символами валюти (`$`, `€`) або відсотків `%`.
  - [ ] `DATE`: Інтерактивний календар (datepicker) з можливістю обрати точний час та часовий пояс.
  - [ ] `CHECKBOX`: Кастомний булевий перемикач.
  - [ ] `DURATION`: Текстове поле, яке підтримує формати типу `1h 30m` або `45s`, і автоматично перераховує їх у секунди на сервері.
  - [ ] `SELECT / STATUS`: Дропдаун із кольоровими опціями та можливістю створити новий варіант безпосередньо звідси.
  - [ ] `RELATION`: Селектор пошуку, який дозволяє знайти та зв'язати записи з іншої бази даних у вигляді клікабельних чіпсів (chips). Клік на чіпс відкриває зв'язаний запис.
  - [ ] `FORMULA`: Заблоковане для редагування сіре поле (відображає результат обчислення).
  - [ ] `RATING`: Набір інтерактивних зірочок (підтримує 5 чи 10 зірок, а також напівзірки).
  - [ ] `PROGRESS`: Візуальна шкала (Progress bar) із відсотком виконання.
  - [ ] `BUTTON`: Фізична кнопка дії (див. розділ 8.6).

### 3. Права область: Контентний конструктор (Record Content Builder)

- [ ] **Структура розмітки:**
  - [ ] **Рядки (Rows):** Контейнери, які можна ділити на колонки (від 1 до 5 колонок в одному ряду).
  - [ ] **Блоки (Blocks):** Стилізовані блоки всередині колонок (можна задавати колір фону, внутрішні відступи та рамки).
  - [ ] **Компоненти (Components):** Конкретні контентні елементи (23 типи: параграфи, заголовки, списки, картинки, файли, графіки, таблиці тощо).
- [ ] **Панель інструментів (Floating Formatting Toolbar):** З'являється над виділеним компонентом. Містить налаштування тексту (жирний, курсив, підкреслений, колір) та вирівнювання.
- [ ] **Drag-and-Drop:** Кожен рядок та блок має маркер перетягування (drag handle) зліва. Користувач може візуально міняти структуру сторінки місцями.

## 4.3 Інтерфейсні стани (UI States) <!-- US-043, US-045, US-056, US-057, US-135 -->

- [ ] **Loading:** Ліва панель показує скелетони властивостей, права область відображає великий Spinner по центру.
- [ ] **Autosave Status Indicator (Індикатор автозбереження):**  
       Невеликий текст у верхньому правому кутку контентної області:
  - [ ] `Зберігання...` — при внесенні змін (анімація пульсації).
  - [ ] `Зміни збережено` — після успішної відповіді сервера.
  - [ ] `Помилка збереження` (червоний колір) — якщо виникли проблеми з мережею чи сервером. Кнопка `[Повторити]` стає активною.
- [ ] **Empty Content Area (Порожній контент):**
  - [ ] Якщо в записі немає жодного контентного блоку, відображається сірий текст-placeholder: `Клікніть тут, щоб почати писати або натисніть "/" для виклику команд`.
- [ ] **Snapshots Panel State (Панель знімків):**
  - [ ] Бічна панель, яка висувається праворуч.
  - [ ] Показує список знімків (snapshots) із часом створення (наприклад, `Сьогодні, 14:35`, `Вчора, 18:20`).
  - [ ] **Hover на знімок:** З'являється кнопка `Попередній перегляд` та `Відновити`.
  - [ ] **Попередній перегляд знімка:** Контентна область переходить у режим «Лише читання» із жовтим банером зверху: `Перегляд версії від {дата}. [Відновити цю версію] [Скасувати перегляд]`.
  - [ ] **Кількість знімків:** Максимум 50 на один запис. При досягненні 50, старі автоматично перезаписуються новими.
