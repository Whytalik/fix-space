# UI Specification — Кошик

Документ описує інтерфейси та сценарії екрану Кошика (Trash).

---

## 7.1 Кошик <!-- US-134 -->

**Маршрут:** `/trash`  
**Доступ:** Захищена  
**Призначення:** Перегляд, відновлення та остаточне видалення раніше вилучених записів.

### Макет сторінки

Загальний layout захищеного простору. Основна зона містить заголовок, інформаційний блок про правила зберігання записів, таблицю вилучених елементів та панель керування.

---

## 7.2 Елементи інтерфейсу <!-- US-134, US-048, US-049 -->

1. **Заголовок:** H1 «Кошик» із червоною іконкою сміттєвого бака.
2. **Інформаційне попередження (Notice Bar):** Жовта плашка зверху: `Вилучені записи зберігаються в кошику протягом 30 днів, після чого автоматично видаляються назавжди. Записи в кошику не враховуються у статистиці, формулах та лімітах бази`.
3. **Таблиця вилучених записів:**
   - [ ] **Чекбокси:** Зліва від кожного рядка для масового вибору.
   - [ ] **Колонка «Назва запису»:** Назва видаленого елемента (посилання на сторінку перегляду вимкнене, оскільки запис у кошику).
   - [ ] **Колонка «Оригінальна база даних»:** Назва бази, з якої було видалено запис.
   - [ ] **Колонка «Дата видалення»:** Точний час видалення.
   - [ ] **Колонка «Залишилось часу»:** Кількість днів до остаточного очищення (наприклад, `12 днів залишилось`).
   - [ ] **Кнопки дій (Row Actions):**
     - [ ] **Відновити (Restore):** Повертає запис до його бази даних.
     - [ ] **Видалити назавжди (Delete Permanently):** Остаточно стирає запис.
4. **Кнопка «Очистити кошик»:** Розташована у шапці таблиці праворуч. Дозволяє видалити всі записи з кошика в один клік після підтвердження.

---

## 7.3 Інтерфейсні стани (UI States) <!-- US-134, US-048, US-049 -->

- [ ] **Loading:** Напівпрозорий оверлей на таблиці, Spinner по центру.
- [ ] **Empty State (Порожній кошик):**
  - [ ] Замість таблиці показується зелена іконка чистого кошика.
  - [ ] Текст: `Кошик порожній. Тут відображатимуться записи, видалені за останні 30 днів`.
  - [ ] Сайдбар при цьому ховає лінк `Кошик`.
- [ ] **Restore Error (Помилка відновлення):**
  - [ ] Якщо оригінальна база даних, з якої походить запис, була видалена користувачем безповоротно, при спробі відновлення з'являється червоний toast-Alert: `Не вдалося відновити запис. Базу даних "{Назва бази}" було видалено. Відновіть спочатку базу даних або створіть нову з такою ж назвою`.
- [ ] **Confirmation Modals (Підтвердження дій):**
  - [ ] **Поодиноке/Масове видалення:** При натисканні `Видалити назавжди` з'являється вікно: `Ви дійсно хочете остаточно видалити вибрані записи (N шт.)? Цю дію неможливо скасувати. [Видалити назавжди] [Скасувати]`.
  - [ ] **Очищення кошика:** При кліці `Очистити кошик` показується вікно: `Остаточно видалити ВСІ записи з кошика? Ця дія повністю очистить кошик і знищить дані безповоротно. [Очистити все] [Скасувати]`.
- [ ] **Bulk Actions State (Виділення кількох записів):**
  - [ ] Внизу екрану висувається плаваюча панель дій: `Вибрано записів: N`.
  - [ ] Кнопки: `Відновити вибрані` та `Видалити вибрані назавжди`.
