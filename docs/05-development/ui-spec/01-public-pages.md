# UI Specification — Публічні сторінки

Документ описує інтерфейси та сценарії публічних сторінок (доступних без авторизації).

---

## 1.1 Головна (незалогінений) <!-- US-001, US-003 -->

**Маршрут:** `/`  
**Доступ:** Публічна (для залогінених користувачів відбувається автоматичний редірект на Dashboard `/`)  
**Призначення:** Ознайомча сторінка платформи, опис переваг для трейдерів та заклик до реєстрації.

### Макет

Вертикальна прокрутна сторінка з фіксованим хедером (Header) зверху, контентом по центру та футером (Footer) внизу.

### Елементи

- [ ] **Хедер (фіксований):**
  - [ ] Логотип FIX Space зліва.
  - [ ] Навігаційні лінки (Features, How it works, Templates) по центру.
  - [ ] Кнопки «Увійти» (secondary) та «Спробувати безкоштовно» (primary) справа.
- [ ] **Секції сторінки:**
  - [ ] **Hero:** Заголовок, підзаголовок, CTA-кнопка «Почати» та вторинна «Дізнатися більше».
  - [ ] **Specialized:** Порівняльна таблиця або блоки про переваги для CFD-трейдингу порівняно з загальними нотатниками.
  - [ ] **Features:** Сітка ключових фіч (Торговий журнал, статистика, шаблони, автоматизації).
  - [ ] **Templates:** Приклади вбудованих шаблонів.
  - [ ] **How it works:** Покрокова візуальна інструкція з 3-4 кроків.
  - [ ] **Import:** Опис імпорту з CSV.
  - [ ] **CTA:** Фінальна кнопка «Зареєструватися безкоштовно».
- [ ] **Футер:**
  - [ ] Копірайт, посилання на Privacy Policy та Terms of Service.

### Поведінка та стани

- [ ] **Стан завантаження:** Відсутній (статична HTML-сторінка).
- [ ] **Взаємодія:**
  - [ ] Клік на CTA-кнопки перенаправляє на `/register`.
  - [ ] Кліки на навігаційні лінки виконують плавний скрол (smooth scroll) до відповідної секції.

## 1.2 Вхід <!-- US-003, US-006 -->

**Маршрут:** `/login`  
**Доступ:** Публічна (редірект на `/` якщо користувач уже залогінений)  
**Призначення:** Автентифікація існуючого користувача за паролем чи Google.

### Макет

Центрована картка форми (Auth Card) на нейтральному градієнтному фоні.

### Елементи

- [ ] Заголовок «З поверненням» та короткий опис під ним.
- [ ] Поле **Email** (type=email, autocomplete=username, placeholder="Введіть ваш email").
- [ ] Поле **Пароль** (type=password, autocomplete=current-password, placeholder="Введіть пароль").
- [ ] Посилання «Забули пароль?» (розташоване праворуч над/під полем пароля).
- [ ] Кнопка «Увійти» (primary, submit, займає всю ширину).
- [ ] Роздільник «або» з лініями по боках.
- [ ] Кнопка «Увійти через Google» з логотипом Google (OAuth 2.0).
- [ ] Посилання «Немає акаунту? Зареєструватися» в самому низу.

### Інтерфейсні стани (UI States)

- [ ] **Default (Початковий):** Порожні поля, кнопка входу активна.
- [ ] **Validation (Валідація):**
  - [ ] При виході з фокусу (blur) порожнього поля Email або невірного формату з'являється червона підказка: `Некоректний формат email`.
  - [ ] Пароль підсвічується червоним, якщо він порожній під час submit.
- [ ] **Loading (Завантаження):**
  - [ ] Поля Email, Пароль та кнопка Google стають `disabled`.
  - [ ] Кнопка «Увійти» стає `disabled`, текст замінюється на анімований Spinner.
- [ ] **Success (Успіх):** Запис токенів у сесію/cookie, плавний перехід на Dashboard `/`.
- [ ] **Error (Помилка):**
  - [ ] При невірних облікових даних над формою з'являється червоний блок помилки (Alert): `Невірний email або пароль`.
  - [ ] Якщо email не підтверджено: блок помилки `Ваш email не верифіковано. [Надіслати лист підтвердження знову]`. Клік на лінк ініціює повторну відправку.

## 1.3 Реєстрація <!-- US-001, US-006 -->

**Маршрут:** `/register`  
**Доступ:** Публічна  
**Призначення:** Створення нового облікового запису користувача.

### Макет

Центрована картка форми, аналогічна до сторінки входу.

### Елементи

- [ ] Заголовок «Створити акаунт» та опис.
- [ ] Поле **Нікнейм** (type=text, placeholder="Ваш нікнейм").
- [ ] Поле **Email** (type=email, placeholder="Ваша пошта").
- [ ] Поле **Пароль** (type=password, placeholder="Мінімум 8 символів, цифра, велика літера").
- [ ] Поле **Підтвердження пароля** (type=password, placeholder="Повторіть пароль").
- [ ] Кнопка «Зареєструватися» (primary, submit).
- [ ] Кнопка «Зареєструватися через Google» (OAuth 2.0).
- [ ] Посилання «Вже є акаунт? Увійти».

### Інтерфейсні стани (UI States)

- [ ] **Default:** Чиста форма.
- [ ] **Validation:**
  - [ ] Паролі перевіряються на відповідність правилу: ≥8 символів, велика літера, цифра. Якщо не відповідає: `Пароль занадто простий`.
  - [ ] Поле підтвердження пароля перевіряється на відповідність: `Паролі не збігаються`.
- [ ] **Loading:** Поля заблоковані, кнопка показує Spinner.
- [ ] **Success (Екран успіху реєстрації):**
  - [ ] Картка форми очищується і показує ілюстрацію листа (Inbox icon).
  - [ ] Заголовок `Перевірте вашу пошту`.
  - [ ] Текст: `Ми надіслали лист підтвердження на {email}. Посилання дійсне протягом 24 годин. Будь ласка, перейдіть за ним для активації акаунту`.
  - [ ] Кнопки: `Відкрити пошту` та `Повернутися до входу`.
- [ ] **Error:** Блок помилки `Користувач з таким email вже існує` (якщо API повертає 409).

## 1.4 Скидання пароля <!-- US-004 -->

**Маршрут:** `/reset-password`  
**Доступ:** Публічна (може містити query параметр `token`)  
**Призначення:** Відновлення доступу за допомогою email.

### Етапи та стани (UI States)

#### Етап 1: Запит посилання

Коли в URL немає токена (`/reset-password`).

- [ ] **Елементи:** Поле Email, кнопка «Надіслати інструкцію», лінк «Назад до входу».
- [ ] **Стан Loading:** Spinner на кнопці.
- [ ] **Стан Success:** Блок форми замінюється повідомленням: `Інструкцію надіслано. Якщо цей email зареєстрований у системі, ви отримаєте лист із посиланням для скидання пароля`.
- [ ] **Стан Error:** Текст помилки від сервера.

#### Етап 2: Встановлення нового пароля

Коли в URL є токен (`/reset-password?token=...`).

- [ ] **Елементи:** Поля «Новий пароль» та «Підтвердження пароля», кнопка «Встановити пароль».
- [ ] **Стан Validation:** Валідація складності пароля та збігу полів.
- [ ] **Стан Loading:** Spinner на кнопці.
- [ ] **Стан Success:** Текст `Пароль успішно змінено. Зараз ви будете перенаправлені на сторінку входу...` (автоматичний редірект на `/login` за 3 секунди).
- [ ] **Стан Error:** Червона плашка: `Токен прострочений або недійсний. Будь ласка, запросіть скидання пароля знову`.

## 1.5 Підтвердження email <!-- US-002 -->

**Маршрут:** `/auth/verify`  
**Доступ:** Публічна (вимагає query параметр `token` в URL)  
**Призначення:** Верифікація акаунту користувача після кліку на посилання в листі.

### Макет

Центрована міні-картка зі статусом верифікації.

### Елементи та стани (UI States)

- [ ] **Loading (Стан перевірки):**
  - [ ] Великий Spinner по центру.
  - [ ] Текст: `Підтверджуємо вашу пошту, будь ласка, зачекайте...`.
- [ ] **Success (Успішне підтвердження):**
  - [ ] Зелений маркер успіху (іконка ✓).
  - [ ] Заголовок `Пошту успішно підтверджено!`.
  - [ ] Текст: `Ваш акаунт активовано, а робочий простір ініціалізовано стартовими базами даних`.
  - [ ] Кнопка «Увійти» (перенаправляє на `/login`). Автоматичний перехід здійснюється через 3 секунди.
- [ ] **Error (Помилка верифікації):**
  - [ ] Червоний маркер помилки (іконка ✗).
  - [ ] Заголовок `Помилка верифікації`.
  - [ ] Текст: `Посилання недійсне, прострочене або вже було використане`.
  - [ ] Кнопка «Надіслати новий лист» (відкриває міні-поле для вводу Email, і при сабміті надсилає новий токен верифікації).
