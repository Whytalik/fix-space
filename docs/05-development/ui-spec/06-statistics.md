# UI Specification — Статистика та аналітика

Документ описує інтерфейси та сценарії сторінки торгової статистики та аналітики.

---

## 6.1 Статистика

**Маршрут:** `/statistics`  
**Доступ:** Захищена  
**Призначення:** Аналіз ефективності трейдингу за допомогою графіків, KPI та звітів.

---

## 6.2 Глобальні фільтри (Filter Bar)

Розташовані у верхній частині сторінки, діють на всі вкладки:

- [ ] **Вибір акаунту (Account Selector):** Дропдаун з чекбоксами для вибору одного, кількох або всіх торгових рахунків.
- [ ] **Діапазон дат (Date Range Picker):** Поля швидкого вибору (`Сьогодні`, `Цей тиждень`, `Останні 30 днів`, `Цей рік`, `Власний діапазон` з календарем).
- [ ] **Фільтр по активах (Asset Filter):** Мультиселект активів (наприклад, `EUR/USD`, `BTC/USD`).
- [ ] **Режим порівняння (Compare Mode):** Тогл-кнопка. При активації відкриває друге поле вибору діапазону дат для зіставлення метрик двох різних періодів.

---

## 6.3 Вкладки аналітики (Tabs System)

### 1. Вкладка «Overview» (Загальний огляд)

- [ ] **Summary KPI Grid:** Сітка з 7 ключовими показниками:
  - [ ] `Total P&L` (Загальний прибуток/збиток).
  - [ ] `Win Rate` (% прибуткових угод).
  - [ ] `Profit Factor` (Відношення валового прибутку до валового збитку).
  - [ ] `Avg P&L per Trade` (Середній прибуток на угоду).
  - [ ] `Avg RR` (Середній Risk-to-Reward).
  - [ ] `Total Trades` (Загальна кількість угод).
  - [ ] `Avg Hold Time` (Середній час утримання позиції).
- [ ] **P&L Curve:** Великий графік кумулятивного прибутку (Area Chart). При ввімкненому порівнянні відображає дві лінії різного кольору.
- [ ] **Equity Curve:** Графік зміни балансу акаунтів (Area Chart).
- [ ] **Картки Best/Worst Trade:** Два блоки з детальними параметрами найкращої (за P&L) та найгіршої угоди за вибраний період. Клік на картку відкриває повну сторінку запису цієї угоди.

### 2. Вкладка «Patterns» (Торгові патерни)

Розділяє статистику за категоріальними ознаками (по активу, моделі входу, Point A, Point B, таймфрейму, днях тижня, сесіях):

- [ ] Кожен розріз представлений карткою з таблицею: `Патерн / Win Rate / Avg P&L / Кількість угод`.
- [ ] Під таблицею відображається стовпчикова діаграма (Bar chart) відсотка Win Rate.

### 3. Вкладка «Behavior» (Поведінкові фактори)

- [ ] **Якість входів та виходів:** Графіки середньої оцінки якості входу (Entry Quality) та виходу (Exit Quality) по шкалі від 1 до 5.
- [ ] **Дотримання правил (Rule Adherence):** Графік середньої оцінки дотримання торгових правил у часі.
- [ ] **Відхилення RR (RR Deviation):** Порівняльний графік фактичного RR проти запланованого.
- [ ] **Таблиця помилок (Mistakes Frequency & Cost):** Список зафіксованих помилок, відсортований за частотою виникнення та сумарним збитком, який вони принесли (Mistake Name / Frequency / Total Cost).
- [ ] **Rules vs P&L:** Стовпчикова діаграма, яка показує середній P&L угод залежно від оцінки дотримання правил (наприклад, угоди з оцінкою 5/5 порівняно з угодами 1/5).

### 4. Вкладка «Accounts» (Торгові рахунки)

Детальна аналітика по окремо вибраному акаунту:

- [ ] **Balance Dynamics:** Стовпчастий або лінійний графік балансу по місяцях.
- [ ] **Drawdown Curve:** Графік просідання рахунку (Drawdown) у відсотках із відображенням максимального зафіксованого просідання.
- [ ] **Daily Limit Usage:** Стовпчикова діаграма використання денного ліміту збитків (у відсотках від встановленого ліміту).

### 5. Вкладка «Custom Reports» (Власні звіти)

Дозволяє будувати статистику по будь-яких несистемних (власних) базах даних робочого простору:

- [ ] Список усіх БД простору з тоглом підключення до аналітики.
- [ ] **Автогенерація графіків:** Для кожної підключеної бази система автоматично генерує:
  - [ ] Кругові чи стовпчикові діаграми розподілу значень для SELECT та STATUS полів.
  - [ ] Лінійні графіки динаміки середніх значень для NUMBER полів.
  - [ ] Графіки активності додавання записів по днях/тижнях.

## 6.4 Інтерфейсні стани (UI States)

- [ ] **Loading:** Відображення скелетонів замість графіків та таблиць, Spinner у шапці глобальних фільтрів.
- [ ] **Empty State (Немає даних за період):**
  - [ ] Замість графіків відображається текст: `За вибраний період не знайдено закритих угод. Змініть налаштування фільтрів або додайте записи у Trading Journal`.
- [ ] **Compare Mode Delta State:**
  - [ ] При активному порівнянні під кожним основним числом KPI відображається зелена (якщо покращення) або червона (погіршення) дельта у відсотках (наприклад, `Win Rate: 48% (+5% порівняно з минулим періодом)`).
- [ ] **Error:** Текст помилки `Не вдалося завантажити статистику. [Повторити запит]`.
