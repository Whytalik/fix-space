# UI Specification — Профіль та налаштування

Документ описує інтерфейси та сценарії сторінки налаштувань профілю користувача.

---

## 5.1 Профіль

**Маршрут:** `/profile`  
**Доступ:** Захищена  
**Призначення:** Керування персональною інформацією, безпекою, глобальним виглядом та налаштуваннями робочого простору.

### Макет сторінки

Основна контентна зона містить заголовок `Профіль та налаштування` із підзаголовком. Нижче розташовується гнучка сітка (Grid, 2 колонки на десктопі, 1 на мобільних) з тематичними картками налаштувань.

---

## 5.2 Елементи та картки налаштувань

### 1. Секція профілю та аватара (Profile Identity)

- [ ] **Аватар:** Велике кругле зображення по центру. Якщо фото не завантажене — відображає ініціали користувача на фоні випадкового градієнту.
- [ ] **Дії з аватаром:** Кнопка `Завантажити фото` (відкриває системний вибір файлів, обмеження 5 МБ, формати JPG/PNG) та кнопка `Видалити` (червона, з'являється лише якщо фото завантажене).
- [ ] **Поля даних:**
  - [ ] **Нікнейм (Username):** Текстове поле (мінімум 3 символи).
  - [ ] **Email:** Поле введення. Поруч відображається кольоровий бейдж:
    - [ ] `Верифікований` (зелений).
    - [ ] `Неверифікований` (жовтий, з посиланням `Надіслати лист підтвердження`).
  - [ ] **Кнопка «Змінити email»:** Ініціює процедуру зміни (надсилається лист верифікації на нову адресу).

### 2. Картка безпеки (Security Settings)

- [ ] **Зміна пароля:** Кнопка `Змінити пароль`. Клік відкриває форму безпосередньо у картці: `Поточний пароль`, `Новий пароль`, `Підтвердження нового пароля`.
- [ ] **Сесії (Active Sessions List):**
  - [ ] Таблиця або список активних пристроїв, які увійшли під цим акаунтом.
  - [ ] Інформація про сесію: IP-адреса, назва пристрою та браузера (наприклад, `Chrome на macOS`), локація (визначається по IP, наприклад, `Київ, Україна`), дата та час входу.
  - [ ] Сесія, з якої зараз переглядається сторінка, має позначку `Поточна сесія`.
  - [ ] **Дії:** Кнопка `Завершити сесію` (іконка хрестика) біля кожної сторонньої сесії та велика червона кнопка `Завершити всі інші сесії` внизу.

### 3. Зовнішній вигляд (Appearance)

- [ ] **Тема оформлення:** Перемикач (Radio group або картки-кнопки): `Світла (Light)`, `Темна (Dark)`, `Системна (System)`.
- [ ] **Мова інтерфейсу:** Дропдаун вибору мови: `Українська`, `English`.

### 4. Налаштування робочого простору (Workspace Configuration)

- [ ] **Формати відображення:**
  - [ ] Формат дати (наприклад, `ДД.ММ.РРРР`, `РРРР-ММ-ДД`).
  - [ ] Формат часу: `12-годинний (AM/PM)` або `24-годинний`.
  - [ ] Перший день тижня: `Понеділок` або `Неділя`.
- [ ] **Часовий пояс:** Список вибору часових поясів (IANA timezone database). За замовчуванням визначається автоматично за даними браузера.
- [ ] **Список Spaces (Робочих просторів):**
  - [ ] Перелік усіх просторів користувача. Навпроти кожного кнопка `Редагувати` (зміна назви) та `Видалити` (червоний кошик).
  - [ ] Позначка `За замовчуванням` біля основного простору, який відкривається при старті застосунку.

### 5. Небезпечна зона (Danger Zone)

- [ ] Розміщується окремим блоком з червоною рамкою.
- [ ] **Елементи:** Заголовок `Видалення акаунту`, опис: `Видалення облікового запису є безповоротним. Всі ваші простори, бази даних, записи та інтеграції будуть знищені назавжди`.
- [ ] **Кнопка:** `Видалити акаунт` (червона, ghost/outline).

---

## 5.3 Інтерфейсні стани (UI States)

- [ ] **Loading:** Картки мають напівпрозорий оверлей, кнопки показують Spinner під час збереження налаштувань.
- [ ] **Success Notification:** Після збереження будь-якої картки у правому верхньому кутку з'являється зелений toast: `Налаштування успішно збережено`.
- [ ] **Error State (Помилки валідації):**
  - [ ] При невірному поточному паролі під час спроби його зміни: червоний текст `Поточний пароль вказано невірно`.
  - [ ] При невідповідності паролів: `Нові паролі не збігаються`.
- [ ] **Confirmation Modals (Модальні вікна підтвердження):**
  - [ ] **Вихід з сесій:** При натисканні `Завершити інші сесії` показується вікно: `Завершити всі активні сесії на інших пристроях? Вам доведеться увійти знову на тих пристроях. [Так, завершити] [Скасувати]`.
  - [ ] **Видалення акаунту:** Модальне вікно вимагає ввести пароль облікового запису та написати фразу `ВИДАЛИТИ АКАУНТ` у текстовому полі для підтвердження серйозності намірів. Кнопка видалення активується лише після цього.
