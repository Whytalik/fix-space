# UI Specification — Профіль та налаштування <!-- US-103, US-120, US-121, US-122, US-123, US-124, US-009, US-004, US-007, US-008, US-012, US-013, US-014 -->

Документ описує інтерфейси та сценарії сторінки налаштувань профілю користувача.

---

## 5.1 Профіль <!-- US-120, US-121, US-122, US-123, US-124, US-009 -->

**Маршрут:** `/profile`  
**Доступ:** Захищена  
**Призначення:** Керування персональною інформацією, безпекою, глобальним виглядом та налаштуваннями робочого простору.

### Макет сторінки

Основна контентна зона містить заголовок `Профіль та налаштування` із підзаголовком. Нижче розташовується гнучка сітка (Grid, 2 колонки на десктопі, 1 на мобільних) з тематичними картками налаштувань.

---

## 5.2 Елементи та картки налаштувань

### 1. Секція профілю та аватара (Profile Identity) <!-- US-120 -->

- [ ] **Аватар:** Велике кругле зображення по центру. Якщо фото не завантажене — відображає ініціали користувача на фоні випадкового градієнту. <!-- US-120 -->
- [ ] **Дії з аватаром:** Кнопка `Завантажити фото` (відкриває системний вибір файлів, обмеження 5 МБ, формати JPG/PNG) та кнопка `Видалити` (червона, з'являється лише якщо фото завантажене). <!-- US-120 -->
- [ ] **Поля даних:** <!-- US-120 -->
  - [ ] **Нікнейм (Username):** Текстове поле (мінімум 3 символи). <!-- US-120 -->
  - [ ] **Email:** Поле введення. Поруч відображається кольоровий бейдж: <!-- US-120 -->
    - [ ] `Верифікований` (зелений). <!-- US-120 -->
    - [ ] `Неверифікований` (жовтий, з посиланням `Надіслати лист підтвердження`). <!-- US-120 -->
  - [ ] **Кнопка «Змінити email»:** Ініціює процедуру зміни (надсилається лист верифікації на нову адресу). <!-- US-120 -->

### 2. Картка безпеки (Security Settings) <!-- US-004, US-007, US-124, US-008 -->

- [ ] **Зміна пароля:** Кнопка `Змінити пароль`. Клік відкриває форму безпосередньо у картці: `Поточний пароль`, `Новий пароль`, `Підтвердження нового пароля`. <!-- US-004 -->
- [ ] **Сесії (Active Sessions List):** <!-- US-007, US-124, US-008 -->
  - [ ] Таблиця або список активних пристроїв, які увійшли під цим акаунтом. <!-- US-007 -->
  - [ ] Інформація про сесію: IP-адреса, назва пристрою та браузера (наприклад, `Chrome на macOS`), локація (визначається по IP, наприклад, `Київ, Україна`), дата та час входу. <!-- US-007 -->
  - [ ] Сесія, з якої зараз переглядається сторінка, має позначку `Поточна сесія`. <!-- US-007 -->
  - [ ] **Дії:** Кнопка `Завершити сесію` (іконка хрестика) біля кожної сторонньої сесії та велика червона кнопка `Завершити всі інші сесії` внизу. <!-- US-124, US-008 -->

### 3. Зовнішній вигляд (Appearance) <!-- US-120, US-121 -->

- [ ] **Тема оформлення:** Перемикач (Radio group або картки-кнопки): `Світла (Light)`, `Темна (Dark)`, `Системна (System)`. <!-- US-120 -->
- [ ] **Мова інтерфейсу:** Дропдаун вибору мови: `Українська`, `English`. <!-- US-121 -->

### 4. Налаштування робочого простору (Workspace Configuration) <!-- US-122, US-012, US-013, US-014 -->

- [ ] **Формати відображення:** <!-- US-122 -->
  - [ ] Формат дати (наприклад, `ДД.ММ.РРРР`, `РРРР-ММ-ДД`). <!-- US-122 -->
  - [ ] Формат часу: `12-годинний (AM/PM)` або `24-годинний`. <!-- US-122 -->
  - [ ] Перший день тижня: `Понеділок` або `Неділя`. <!-- US-122 -->
- [ ] **Часовий пояс:** Список вибору часових поясів (IANA timezone database). За замовчуванням визначається автоматично за даними браузера. <!-- US-122 -->
- [ ] **Список Spaces (Робочих просторів):** <!-- US-012, US-013, US-014 -->
  - [ ] Перелік усіх просторів користувача. Навпроти кожного кнопка `Редагувати` (зміна назви) та `Видалити` (червоний кошик). <!-- US-012, US-013 -->
  - [ ] Позначка `За замовчуванням` біля основного простору, який відкривається при старті застосунку. <!-- US-014 -->

### 5. Небезпечна зона (Danger Zone) <!-- US-009 -->

- [ ] Розміщується окремим блоком з червоною рамкою. <!-- US-009 -->
- [ ] **Елементи:** Заголовок `Видалення акаунту`, опис: `Видалення облікового запису є безповоротним. Всі ваші простори, бази даних, записи та інтеграції будуть знищені назавжди`. <!-- US-009 -->
- [ ] **Кнопка:** `Видалити акаунт` (червона, ghost/outline). <!-- US-009 -->

## 5.3 Інтерфейсні стани (UI States) <!-- US-120, US-124, US-008, US-009 -->

- [ ] **Loading:** Картки мають напівпрозорий оверлей, кнопки показують Spinner під час збереження налаштувань.
- [ ] **Success Notification:** Після збереження будь-якої картки у правому верхньому кутку з'являється зелений toast: `Налаштування успішно збережено`.
- [ ] **Error State (Помилки валідації):**
  - [ ] При невірному поточному паролі під час спроби його зміни: червоний текст `Поточний пароль вказано невірно`.
  - [ ] При невідповідності паролів: `Нові паролі не збігаються`.
- [ ] **Confirmation Modals (Модальні вікна підтвердження):** <!-- US-124, US-008, US-009 -->
  - [ ] **Вихід з сесій:** При натисканні `Завершити інші сесії` показується вікно: `Завершити всі активні сесії на інших пристроях? Вам доведеться увійти знову на тих пристроях. [Так, завершити] [Скасувати]`. <!-- US-124, US-008 -->
  - [ ] **Видалення акаунту:** Модальне вікно вимагає ввести пароль облікового запису та написати фразу `ВИДАЛИТИ АКАУНТ` у текстовому полі для підтвердження серйозності намірів. Кнопка видалення активується лише після цього. <!-- US-009 -->
