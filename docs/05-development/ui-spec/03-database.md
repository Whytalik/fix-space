# UI Specification — База даних

Документ описує інтерфейси та сценарії екранів перегляду та редагування структури баз даних.

---

## 3.1 Перегляд бази даних

**Маршрут:** `/database/[id]`  
**Доступ:** Захищена  
**Призначення:** Основна робоча область для перегляду та маніпуляції записами бази даних у табличному вигляді.

### Елементи інтерфейсу

1. **Заголовок бази даних (Database Title):**
   - [ ] Назва БД, велика іконка (клік відкриває емодзі-пікер).
   - [ ] Індикатор замок (Structure Lock): показує, чи заблокована структура від змін властивостей.
   - [ ] Індикатор ліміту записів (Record Limit): наприклад, `124 / 200 записів` (червоніє при наближенні до ліміту >90%).
   - [ ] Хлібні крихти: `Простір › Секція › База`.
2. **Вкладки подань (Views Tabs):**
   - [ ] Ряд вкладок зверху (до 10). Кожне подання — це збережена конфігурація фільтрів, сортування та видимості стовпчиків.
   - [ ] Кнопка `+ Нове подання` справа від вкладок.
   - [ ] Контекстне меню на вкладці: `Перейменувати`, `Дублювати`, `Заблокувати налаштування подання`, `Видалити`.
3. **Панель керування (Toolbar):**
   - [ ] **Поле швидкого пошуку:** Фільтрація по назві запису (миттєвий пошук з дебаунсом 150мс).
   - [ ] **Кнопка «Фільтр»:** Відображає кількість активних правил (наприклад, `Фільтр (2)`), відкриває панель фільтрації.
   - [ ] **Кнопка «Сортування»:** Показує статус, відкриває панель сортування.
   - [ ] **Кнопка «Групування»:** Дозволяє згрупувати записи за вибраним полем SELECT, STATUS або DATE.
   - [ ] **Кнопка «Колонки»:** Чекліст відображення/приховування властивостей.
   - [ ] **Тогл «Перенос тексту»:** Вмикає/вимикає автоперенос слів у клітинках.
4. **Таблиця записів (Table View):**
   - [ ] **Стовпчик 1:** «Назва» (Name) — завжди закріплений ліворуч, не може бути прихований. Клік на назву запису відкриває його сторінку.
   - [ ] **Наступні стовпчики:** Властивості бази. Заголовки містять іконку типу поля та назву властивості. Ширина колонок регулюється перетягуванням правої межі з автоматичним збереженням на сервері.
   - [ ] **Чекбокси вибору:** З'являються при наведенні курсору на ліву крайню частину рядка для виділення записів.
   - [ ] **Рядок швидкого додавання:** Останній рядок таблиці містить кнопку `+ Новий запис`. Клік по ній створює пустий запис з автозбереженням та фокусує поле Назва.
5. **Рядок підсумків (Totals Row):**
   - [ ] Знаходиться безпосередньо під тілом таблиці.
   - [ ] Для кожного стовпчика користувач може обрати тип підсумку: `Кількість` (для всіх), `Сума`, `Середнє`, `Мінімум`, `Максимум` (для чисел та тривалостей), `Відсоток заповнення` тощо.
6. **Пагінація:**
   - [ ] Дропдаун вибору розміру сторінки: `10`, `25`, `50`, `75`, `100` записів.
   - [ ] Кнопки `‹ Назад` та `› Вперед`. Індикатор `Записи 1–25 із 120`.

### Інтерфейсні стани (UI States)

- [ ] **Loading:** Анімована смуга завантаження (Progress bar) зверху таблиці, напівпрозорий оверлей поверх клітинок.
- [ ] **Empty State (Порожня база):**
  - [ ] Замість таблиці показується ілюстрація порожньої папки.
  - [ ] Текст: `Ця база даних не містить записів`.
  - [ ] Кнопка: `Створити перший запис` або `Імпортувати з CSV`.
- [ ] **Filtered Empty State (Нічого не знайдено):**
  - [ ] Показується заголовок таблиці, але в тілі напис: `Не знайдено записів, які відповідають встановленим фільтрам`.
  - [ ] Кнопка: `Скинути всі фільтри`.
- [ ] **Bulk Actions State (Виділення кількох записів):**
  - [ ] Внизу екрану плавно виїжджає горизонтальна темна панель (Floating Action Bar): `Вибрано записів: N`.
  - [ ] Доступні кнопки: `Редагувати властивості` (відкриває поп-овер для вибору властивості та значення, яке запишеться усім), `Дублювати` та `Видалити до кошика` (з вікном підтвердження).
  - [ ] Примітки: Властивості типу FORMULA, BUTTON та RELATION заблоковані для масового редагування.

## 3.2 Редагування бази даних (Схема & Налаштування)

**Маршрут:** `/database/[id]/edit`  
**Доступ:** Захищена  
**Призначення:** Налаштування схеми властивостей, керування шаблонами та загальними параметрами бази.

### Елементи інтерфейсу

1. **Секція «Загальні налаштування»:**
   - [ ] Поле введення назви БД та вибір іконки.
   - [ ] Перемикач «Заблокувати структуру» (запобігає випадковому додаванню чи видаленню властивостей іншими користувачами/сесіями).
   - [ ] Поле «Ліміт записів» (максимум, який може містити БД. Якщо порожнє — безлімітно).
   - [ ] Дропдаун «Шаблон за замовчуванням» (який шаблон автоматично застосовувати при створенні нового запису).
2. **Секція «Властивості (Схема полів)»:**
   - [ ] Список поточних властивостей. Кожен рядок містить іконку типу, назву властивості, тип та кнопку `Видалити`.
   - [ ] Рядки можна перетягувати (Drag-and-Drop) для зміни порядку відображення колонок за замовчуванням.
   - [ ] Кнопка `+ Додати властивість`: відкриває модальне вікно вибору типу властивості (12 типів) та введення назви.
3. **Секція «Шаблони бази»:**
   - [ ] Список створених шаблонів записів.
   - [ ] Кожен шаблон показує статус: `Системний` (позначка lock, не можна видалити, але можна перейменувати або скинути до початкового) або `Власний` (можна видалити).
   - [ ] Кнопка `+ Створити новий шаблон`: перенаправляє в Редактор шаблонів.

- [ ] **Кнопки збереження:** «Зберегти зміни» (submit) та «Скасувати» (повертає на `/database/[id]`).

### Інтерфейсні стани (UI States)

- [ ] **Locked Structure (Структура заблокована):**
  - [ ] Кнопка `+ Додати властивість` заблокована (disabled) або прихована.
  - [ ] Кнопки `Видалити` біля властивостей приховані.
  - [ ] Перетягування полів вимкнене.
- [ ] **Validation (Помилки при додаванні поля):**
  - [ ] Назва поля має бути унікальною всередині бази. При спробі ввести дублікат: червона помилка `Властивість з такою назвою вже існує`.
- [ ] **Loading:** Кнопка «Зберегти зміни» показує Spinner, поля недоступні для редагування.

## 3.3 Редактор шаблону запису

**Маршрут:** `/database/[id]/template/[id]`  
**Доступ:** Захищена  
**Призначення:** Конструювання шаблону: задання статичних початкових значень властивостей та побудова структури контентної області.

### Особливості інтерфейсу

- [ ] Збігається візуально із Сторінкою запису (4.1), але має великий банер зверху: `Редагування шаблону: [Назва шаблону]`.
- [ ] **Токени у назві запису:** У полі назви шаблону можна використовувати спеціальні токени-підстановки:
  - [ ] `{{today}}` (буде замінено на поточну дату наприклад `04.06.2026`).
  - [ ] `{{month}}` / `{{year}}`.
  - [ ] `{{count}}` (порядковий номер запису в базі).
- [ ] **Поведінка значень властивостей:** Задані значення у лівій панелі будуть автоматично підставлятися у будь-який створений за цим шаблоном запис.
- [ ] **Контентна область шаблону:** Будь-які блоки, вставлені сюди (текст, чеклісти, таблиці), будуть скопійовані у новий запис як початкова розмітка.
- [ ] **Збереження:** Кнопка `Зберегти шаблон` вгорі сторінки. При успіху — перенаправлення до налаштувань бази.
- [ ] **Стан Loading:** Spinner на кнопці збереження.
- [ ] **Стан Error:** Повідомлення про помилку збереження контенту (наприклад, перевищення ліміту розміру).
