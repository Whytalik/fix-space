# UI Specification — Глобальні компоненти та інтерфейси

Документ описує інтерфейси та сценарії глобальних компонентів і спливаючих вікон системи.

---

## 8.1 Глобальний пошук (Ctrl+K / Cmd+K)

**Доступ:** Захищений простір  
**Призначення:** Пошук записів по всьому робочому простору без прив'язки до конкретної бази даних.

### Макет та елементи

- [ ] **Оверлей (Backdrop):** Затемнення фону із ефектом розмиття (backdrop blur). Центроване модальне вікно.
- [ ] **Рядок пошуку:** Велике поле вводу з іконкою лупи, кнопкою очищення (Х) та плейсхолдером `Пошук по базах, властивостях та контенту...`.
- [ ] **Блок нещодавніх пошуків:** Показується, якщо поле вводу порожнє. Список з 5 останніх запитів користувача та кнопка `Очистити історію`.
- [ ] **Список результатів:** Групується за назвою Секції та Бази даних (наприклад, `Routine / Daily Routine`).
  - [ ] Кожен результат показує назву запису, іконку та фрагмент тексту з підсвічуванням знайденого збігу (highlighting).
  - [ ] Відображається не більше 5 результатів для кожної бази. Якщо результатів більше, внизу групи з'являється лінк `Показати всі результати в {Назва бази}`.
- [ ] **Підказки гарячих клавіш (Footer):** Текст внизу вікна: `↑↓ для навігації · Enter для вибору · ESC для закриття`.

### Інтерфейсні стани (UI States)

- [ ] **Debounce State (Очікування):** Пошук ініціюється після введення ≥2 символів з затримкою 150мс. У цей момент зліва в полі пошуку крутиться маленький індикатор завантаження.
- [ ] **No Results (Нічого не знайдено):** Текст `Результатів для запиту "{query}" не знайдено. Перевірте написання`.

## 8.2 Панель сповіщень (Notifications Flyout)

**Доступ:** Захищений простір  
**Призначення:** Швидкий перегляд сповіщень про події (збої імпорту, автоматизацій, інтеграцій).

### Макет та елементи

- [ ] **Дзвіночок у хедері:** Має червоний кружечок-badge з числом непрочитаних сповіщень. Якщо прочитано все — badge зникає.
- [ ] **Панель (Flyout Panel):** Висувається праворуч при кліці на дзвіночок.
- [ ] **Заголовок:** Назва «Сповіщення» та кнопка `Позначити все як прочитане`.
- [ ] **Список сповіщень (до 50 штук):** Кожне сповіщення містить:
  - [ ] Іконку типу події: червоний знак оклику (Помилка), зелена блискавка (Автоматизація), синє коло "i" (Інформаційне).
  - [ ] Текст повідомлення із активними лінками.
  - [ ] Відносний час створення (наприклад, `10 хв тому`, `вчора`).
  - [ ] Маркер непрочитаності: блакитна крапка збоку та світліший колір фону.

### Інтерфейсні стани (UI States)

- [ ] **Empty State:** Зображення спокійного дзвіночка, текст `Нових сповіщень немає`.
- [ ] **Read State:** Клік по сповіщенню або відкриття панелі знімає виділення непрочитаності. Клік по сповіщенню виконує редірект на джерело події (наприклад, у налаштування інтеграцій).

## 8.3 Модальне вікно імпорту CSV

**Доступ:** Захищений простір, викликається з панелі бази даних  
**Призначення:** Імпорт записів із файлів CSV з маппінгом полів та валідацією.

### Кроки майстра імпорту (Wizard Steps)

1. **Крок 1: Завантаження файлу:**
   - [ ] Зона drag-and-drop для перетягування файлу `.csv` (обмеження 25 МБ) або кнопка вибору.
2. **Крок 2: Налаштування маппінгу (Mapping fields):**
   - [ ] Показується перші 5 рядків із завантаженого файлу для прев'ю.
   - [ ] Для кожної колонки з CSV відображається дропдаун із переліком сумісних властивостей поточної бази даних для зіставлення. Користувач може обрати `Ігнорувати стовпець`. RELATION-властивості виключені з вибору (не підтримують CSV-імпорт).
   - [ ] Чекбокс `Перший рядок містить заголовки`.
3. **Крок 3: Валідація та підсумок:**
   - [ ] Звіт перед імпортом: `N записів буде створено, M рядків містять помилки валідації (будуть пропущені)`.
   - [ ] Список помилок: наприклад, `Рядок 12: в полі Risk očekується число`.
   - [ ] **Попередження про ліміт:** якщо кількість рядків у файлі перевищує доступний ліміт записів бази, показується червона плашка: `База даних має ліміт Х записів. Буде імпортовано лише перші Y записів. [Імпортувати перші Y] [Скасувати]`.
4. **Крок 4: Прогрес імпорту:**
   - [ ] Анімований progress bar з відсотками виконання. Кнопка скасування процесу.
5. **Крок 5: Звіт про завершення:**
   - [ ] Текст `Імпорт завершено. Успішно додано N записів, пропущено M рядків`. Кнопка `Закрити`.

## 8.4 Конструктор автоматизацій

**Доступ:** Налаштування бази даних (`/database/[id]/edit`)  
**Призначення:** Налаштування правил без коду (no-code rules) виду «КОЛИ→ТО».

### Елементи інтерфейсу

- [ ] **Список правил:** Правила автоматизації (до 10 на базу). Кожне має назву, тогл активації/деактивації та кнопку `Логи запуску`.
- [ ] **Галерея пресетів:** Картки швидкого старту (наприклад, `Автологування помилки`, `Тижневий Performance Review`) або кнопка `Почати з нуля`.
- [ ] **Візуальний конструктор:**
  - [ ] **Блок «КОЛИ» (Trigger):** Дропдаун вибору: `Поле змінюється` (вибір поля та умови), `Запис створено` або `За розкладом` (вибір періоду та часу).
  - [ ] **Блок «ТО» (Actions):** Ланцюжок дій (до 5 штук). Доступні дії: `Встановити значення поля` (задати нове значення або скопіювати з іншого), `Створити запис в іншій базі` (з маппінгом полів), `Зв'язати записи` (пошук за фільтром та лінкування).
- [ ] **Панель тестування (Dry Run):** Кнопка `Перевірити автоматизацію`. Дозволяє обрати один тестовий запис і симулювати виконання правила. Результат показується у вигляді тексту: `Властивість "Status" змінено на "Reviewed"`. База даних при цьому не оновлюється.

### Інтерфейсні стани (UI States)

- [ ] **Executing State (Журнал запусків):**
  - [ ] Таблиця останніх 50 запусків правила. Кожен запис має статус: `Успішно` (зелений), `Пропущено` (жовтий, якщо не виконались умови фільтру), `Помилка` (червоний, детальний опис помилки).
- [ ] **Recursion protection:** Якщо дія правила намагається змінити тригерне поле цього ж правила, система підсвічує це червоним та блокує збереження: `Виявлено нескінченний цикл. Дія не може змінювати поле тригера`.

## 8.5 Конструктор формул

**Доступ:** Модальне вікно при налаштуванні властивості типу `FORMULA`.  
**Призначення:** Створення обчислюваних полів без написання коду.

### Елементи інтерфейсу

- [ ] **Галерея пресетів:** Готові формули для трейдерів: `R-Multiple`, `Запланований RR`, `Різниця між датами`, `Ризик % від балансу` тощо.
- [ ] **Майстер заповнення (Fill-in-the-blanks):** Після вибору пресету відкривається форма з випадаючими списками, де користувач обирає сумісні поля. Наприклад, для «R-Multiple» відображаються два дропдауни: `Вкажіть поле P&L` та `Вкажіть поле Risk`. Доступні лише сумісні по типу полей властивості (NUMBER).
- [ ] **Попередній перегляд (Live Preview):** Візуальне відображення результату формули на основі першого запису бази (або демо-даних) у реальному часі.

### Інтерфейсні стани (UI States)

- [ ] **Cyclic Reference Detection (Циклічне посилання):**
  - [ ] Система автоматично блокує вибір інших формульних полів у дропдаунах, які посилаються на поточну формулу, для запобігання помилкам розрахунку.

## 8.6 Налаштування BUTTON-властивості

**Доступ:** Sidebar налаштування властивості/компонента типу `BUTTON`.  
**Призначення:** Створення кнопок швидких дій у таблиці або в контентній області.

### Елементи налаштування

- [ ] Поле введення напису на кнопці (наприклад, `Закрити позицію`).
- [ ] Палітра кольорів (5 бренд-кольорів).
- [ ] Конструктор списку дій (до 3 дій): `Встановити значення поля`, `Створити запис`, `Вставити блок у контент`.
- [ ] Тогл `Запитувати підтвердження` (з полем для тексту попередження).

### Стани кнопки в таблиці/контенті (UI States)

- [ ] **Idle (Активна):** Звичайна кнопка вибраного кольору.
- [ ] **Executing (Виконується):** Кнопка блокується (disabled), текст змінюється на `Виконується...`, показується Spinner.
- [ ] **Success (Успіх):** Кнопка на 2 секунди підсвічується зеленим з іконкою галочки та текстом `Готово`, поруч з'являється дрібна дата останнього натискання.
- [ ] **Error (Помилка):** Кнопка підсвічується червоним, з'являється toast-Alert з текстом помилки виконання дій на сервері.

## 8.7 Елементи онбордингу

### 1. Демо-простір (Demo Space Dashboard)

- [ ] **Сайдбар:** База даних демо-простору маркується спеціальною іконкою `Demo`.
- [ ] **Банер демо-простору:** Плаваючий банер вгорі дашборду: `Ви перебуваєте в демо-просторі. Ви можете редагувати та створювати будь-які дані для тестування. [Скинути демо-дані до початкових]`.
- [ ] **Клік на скидання:** Викликає модальне вікно підтвердження. При згоді видаляє всі зміни та заново завантажує 36 стартових записів.

### 2. Покроковий тур (Interactive Tour)

- [ ] **Елементи:** Візуальне виділення (highlighting overlay) елемента інтерфейсу та спливаюча картка-підказка поруч.
- [ ] **Картка:** Крок (наприклад, `3 з 10`), опис функції, кнопка `Далі`, `Назад`, та лінк `Пропустити тур`.
- [ ] **Збереження стану:** Стан завершення туру записується в профілі користувача.

### 3. Чекліст активації (Activation Checklist)

- [ ] **Макет:** Віджет у правому нижньому кутку дашборду. Може згортатися в іконку.
- [ ] **Елементи:** Список з 5 стартових завдань (наприклад, `Створити базу даних`, `Зареєструвати першу угоду`). Показує прогрес-бар (наприклад, `40% виконано`). Клік на завдання автоматично відкриває потрібну сторінку.

### 4. Контекстні підказки (Feature Tooltips)

- [ ] **Макет:** Одноразові спливаючі підказки (popovers) біля складних елементів (наприклад, панель фільтрів).
- [ ] **Поведінка:** З'являється автоматично при першому фокусі елемента. Клік на кнопку `Зрозуміло` ховає підказку назавжди.
