# UI Specification — Dashboard

Документ описує інтерфейси та сценарії головної сторінки робочого простору (Dashboard).

---

## 2.1 Загальний Layout (Захищений простір) <!-- US-010, US-011, US-012, US-013, US-016, US-017, US-018, US-085 -->

Всі сторінки після авторизації використовують спільну структуру:

- [ ] **Sidebar (Лівий сайдбар):** Фіксований ліворуч, може колапсуватися у вузьку смугу іконок.
- [ ] **Header (Шапка сторінки):** Містить навігаційні хлібні крихти, іконку пошуку, кнопку сповіщень (дзвіночок з badge) та аватар швидких налаштувань.
- [ ] **Main Area (Основний контент):** Прокручувана область праворуч від сайдбару.

### Елементи сайдбару

1. **Space Switcher (Селектор просторів):** Дропдаун у верхній частині сайдбару. Показує поточний вибраний Space. При кліці відкриває список усіх Spaces користувача (макс. 10) та кнопку «+ Створити простір».
2. **Global Search Button:** Кнопка з текстом «Пошук...» та ярликом `Ctrl+K`. Клік відкриває модальне вікно пошуку.
3. **Sections & Databases List:**
   - [ ] Список секцій (Routine, Insight, Settings, тощо). Кожна секція — акордеон (колапсується із збереженням стану).
   - [ ] Всередині секцій — бази даних (БД). Кожна БД має іконку та назву.
   - [ ] Підтримка перетягування (Drag-and-Drop) для зміни порядку БД всередині секції або перенесення між секціями.
   - [ ] Біля назви пресетних БД відображається ледь помітна позначка «системна» (lock/preset indicator), при цьому в контекстному меню цієї БД заблоковано опцію «Видалити».
4. **Trash Link (Кошик):** Розташований внизу. Відображається лише якщо в кошику є хоча б один видалений запис. Показує кількість елементів поруч (наприклад, `Кошик (5)`).
5. **User Profile Card:** Аватар та нікнейм користувача у самому низу сайдбару. Клік відкриває меню: `Профіль та налаштування`, `Вихід`.

## 2.2 Dashboard — Головна сторінка <!-- US-017 -->

**Маршрут:** `/` (для авторизованого користувача)  
**Доступ:** Захищена (JWT)  
**Призначення:** Швидкий огляд торгового дня, виконання щоденної рутини та ключова аналітика.

### Блок 1: Індикатор ринкових сесій (Market Sessions)

- [ ] **Елементи:** Поточний час користувача (з секундами) та 4 плашки торгових сесій: Токіо (Tokyo), Франкфурт (Frankfurt), Лондон (London), Нью-Йорк (New York).
- [ ] **Стани (UI States):**
  - [ ] **Active (Активна сесія):** Зелений індикатор, плашка підсвічена зеленим градієнтом, відображається час до закриття (наприклад, `Нью-Йорк (Активна) · Закриється через 3г 15хв`).
  - [ ] **Inactive (Неактивна сесія):** Сірий індикатор, тьмяна плашка, відображається час до відкриття (наприклад, `Лондон (Закрита) · Відкриється через 5г 20хв`).
  - [ ] **Session Transition (Перехідний стан):** Коли одна сесія закривається, а інша відкривається, плашка анімовано змінює колір.

### Блок 2: Щоденний воркфлоу (Daily Workflow)

- [ ] **Елементи:** Горизонтальний ланцюжок з 4 кроків-завдань:
  1. **Plan Routine** (База: Daily Routine)
  2. **Execute Trade** (База: Trading Journal)
  3. **Reflect Notes** (База: Notes)
  4. **Log Mistakes** (База: Mistakes)
- [ ] **Стани (UI States):**
  - [ ] **Not Started (Не розпочато):** Сіра іконка, порожній круг-чекбокс. Клік перенаправляє до відповідної бази та одразу відкриває вікно створення нового запису.
  - [ ] **Completed (Виконано):** Яскрава іконка, зелена галочка в круг-чекбоксі. Визначається автоматично, якщо користувач сьогодні створив хоча б один запис у цій базі даних.

### Блок 3: Сьогодні (Today Overview)

- [ ] **Елементи:** 4 невеликі картки-списки для швидкого перегляду записів за сьогодні: Trading Journal, Daily Routine, Notes, Mistakes.
- [ ] **Стани (UI States):**
  - [ ] **Loading:** Пульсуючі скелетони (skeleton loaders) замість рядків.
  - [ ] **Empty (Немає записів за сьогодні):** Напис `Записів за сьогодні не знайдено` тьмяним шрифтом та кнопка `+ Додати` по центру. Клік на `+ Додати` відкриває інтерфейс швидкого додавання запису до цієї бази.
  - [ ] **Data (Є записи):** Список лінків на записи із назвою та часом створення. Внизу картки — кнопка `+ Додати новий` в кутку.

### Блок 4: Огляд аналітики (Mini Charts)

Три компактні інтерактивні графіки, побудовані на основі записів у Trading Journal:

1. **P&L Curve (Крива P&L):** Область-графік (area chart) кумулятивного P&L.
2. **Win Rate Dynamics (Динаміка Win Rate):** Анімований графік відсотку виграшів.
3. **RR Deviation (Відхилення RR):** Порівняння фактичного та запланованого Risk-to-Reward.

- [ ] **Стани (UI States):**
  - [ ] **Loading:** Анімований лоадер-заглушка на місці графіків.
  - [ ] **No Data (Немає даних для побудови):** Відображається розмита фонова сітка графіка із написом по центру: `Недостатньо даних для побудови графіків. Створіть записи в Trading Journal з вказаними значеннями P&L та RR`.
  - [ ] **Error:** Червоний текст `Помилка завантаження аналітичних даних. [Спробувати знову]`.
