# Опис проекту

**Тема:** Веб-платформа для організації діяльності CFD-трейдера

---

## Що собою являє цей проект?

Перш ніж описувати сам програмний продукт, коротко окреслю контекст діяльності, для якої він створюється. CFD-трейдинг — це форма фінансової діяльності, у межах якої учасник ринку укладає контракти на різницю цін. Фінансовий результат у таких контрактах визначається зміною вартості певного базового активу між моментом відкриття та закриття позиції, при цьому фактичне володіння активом не відбувається. У процесі торгівлі трейдер аналізує ринкову інформацію, приймає рішення щодо відкриття та супроводу позицій, управляє ризиками та фіксує результат кожної завершеної угоди. Ефективність такої діяльності значною мірою залежить від здатності системно зберігати інформацію про угоди, аналізувати статистику та накопичувати знання щодо власних ідей та помилок.

Враховуючи описане вище, створювана веб-платформа являє собою спеціалізовану інформаційну систему, призначену для організації робочого процесу CFD-трейдера. Основна ідея полягає у створенні єдиного середовища, у якому трейдер може централізовано зберігати дані про свою діяльність, структурувати їх і отримувати аналітичні узагальнення. Платформа дозволяє фіксувати торгові угоди, будувати та дотримуватися передсесійні та постсесійні рутини, систематизувати торгові стратегії, вести записи щодо помилок і спостережень, а також отримувати статистику результатів торгівлі.

При цьому система не виконує торгових операцій і не інтегрується безпосередньо з брокерськими платформами, а використовується виключно як інструмент організації інформації, аналізу даних та ведення торгового журналу.

---

## User Flow

Типовий сценарій використання платформи починається з реєстрації користувача та входу до системи. Після успішної авторизації користувач потрапляє до свого робочого простору, у межах якого організовується вся його діяльність у системі.

Під час першого входу система автоматично створює стартовий набір баз даних, які формують базову структуру роботи трейдера. Користувач може переходити між цими базами, переглядати вже наявні записи або створювати нові.

Коли користувач хоче зафіксувати певну інформацію, наприклад угоду або власне спостереження, він створює новий запис у відповідній базі даних. Після цього відкривається сторінка запису, де користувач заповнює необхідні поля та додає інформацію, пов'язану з цією подією.

Окрім заповнення властивостей, кожен запис має власну сторінку, де можна додавати додатковий контент. Тут користувач може використовувати готові блоки та компоненти системи для структурування інформації — наприклад описати контекст угоди, свої думки або зробити короткий аналіз ситуації.

Щоб не створювати однакову структуру сторінки щоразу вручну, система підтримує використання шаблонів сторінок. Шаблон містить попередньо визначений набір блоків, який може бути створений самим користувачем або наданий системою. Під час створення нового запису такий шаблон можна автоматично застосувати, після чого користувач отримує вже підготовлену структуру сторінки і може одразу переходити до заповнення її змістом.

У процесі роботи користувач може переглядати записи, редагувати їх, застосовувати фільтри та використовувати різні представлення даних для зручнішої навігації. Також, за потреби, користувач може створювати власні бази даних і налаштовувати їхню структуру відповідно до своїх задач.

## Позиціювання

Платформа не позиціонується як універсальний інструмент для організації будь-якої інформації, подібний до Notion. Платформа орієнтована на конкретну предметну область — організацію робочого процесу CFD-трейдера та систематизацію даних, пов'язаних із торгівлею.

Після створення робочого простору користувач отримує попередньо сформований набір баз даних. Такий підхід дозволяє почати роботу із системою без попереднього проєктування структури даних.

На відміну від універсальних інструментів, де користувач самостійно формує модель роботи з нуля, базові елементи вже оптимізовані під типові сценарії діяльності трейдера. Це включає попередньо визначені типи даних, спеціалізовані властивості для фіксації параметрів угод та структури, що полегшують подальший аналіз торгової статистики.

Водночас система зберігає певний рівень гнучкості. Користувач може створювати власні бази даних, додавати нові властивості та адаптувати структуру простору під власний стиль роботи, не змінюючи при цьому базової логіки організації даних.

## Функціонал платформи

### Користувач
- реєстрація користувача
- вхід до системи
- вихід із системи
- перегляд профілю
- редагування параметрів профілю
- зміна пароля

### Робочі простори
- автоматичне створення робочого простору під час реєстрації
- перегляд робочого простору
- редагування параметрів робочого простору
- створення додаткових робочих просторів
- перемикання між робочими просторами
- дублювання робочого простору
- видалення робочого простору

### Стартовий пакет системи
- автоматичне створення стартового набору баз даних
- створення бази даних Trading Journal
- створення бази даних Daily Routine
- створення бази даних Notes
- створення бази даних Mistakes
- створення бази даних Accounts
- створення бази даних Operations
- створення бази даних Trading System
- автоматичне створення базових властивостей у стартових базах даних
- автоматичне створення стандартних шаблонів сторінок

### Секції
- створення секцій для групування баз даних
- перегляд секцій
- редагування параметрів секції
- зміна порядку секцій
- переміщення баз даних між секціями
- видалення секцій

### Бази даних
- створення бази даних
- перегляд бази даних
- редагування параметрів бази даних
- дублювання бази даних
- видалення бази даних

### Властивості бази даних
- додавання нових властивостей
- редагування параметрів властивостей
- зміна порядку властивостей
- дублювання властивостей
- видалення властивостей
- визначення типу даних властивості

### Записи (records)
- створення записів
- перегляд записів
- редагування записів
- видалення записів
- дублювання записів
- заповнення значень властивостей

### Формули та обчислення
- створення формульних властивостей
- редагування формул
- використання значень інших властивостей у формулах
- автоматичне обчислення значень записів
- перегляд результатів обчислень

### Автоматизації
- створення правил автоматизації
- визначення тригерів (подій, що запускають правило)
- визначення умов виконання
- автоматичне виконання дій над записами
- редагування правил автоматизації

### Сторінка запису
- перегляд сторінки запису
- редагування вмісту сторінки
- додавання блоків контенту
- зміна порядку блоків
- видалення блоків

### Шаблони сторінок
- створення шаблонів сторінок
- редагування шаблонів
- видалення шаблонів
- застосування шаблону до запису
- автоматичне застосування шаблону під час створення запису

### Блоки та компоненти
- додавання готових блоків
- редагування блоків
- зміна порядку блоків
- видалення блоків

### Пошук, фільтрація та сортування даних
- пошук записів у межах бази даних
- глобальний пошук у межах робочого простору
- фільтрація записів за значеннями властивостей
- комбінування кількох фільтрів
- сортування записів за вибраними властивостями
- зміна порядку відображення записів

### Імпорт та експорт даних
- імпорт даних у базу даних
- експорт даних з бази даних
- мапінг полів під час імпорту
- перевірка відповідності структури даних
- попередній перегляд імпортованих даних
