# Дизайн-система FIX Space

## Естетичний напрям: Void Terminal (Термінал Порожнечі)

FIX Space — це **професійне робоче середовище трейдера**, точний інструмент, а не споживчий застосунок. Обрана естетика — **Void Terminal**: глибокі темні тони, хірургічна типографіка та єдиний вирішальний акцент. Уявіть інтерфейс Bloomberg-терміналу, переосмислений у стилі Linear. Кожна деталь має своє призначення. Жодного декорування заради декору.

**Головне відчуття користувача:** глибина без темряви, чіткість без надмірної різкості — інтерфейс, створений для тих, хто ставиться до трейдингу серйозно.

## Колірна система

### Модель рівнів висоти (Elevation Model)

Інтерфейс використовує трирівневу систему висоти елементів. Ніколи не пропускайте рівні та не вигадуйте нові.

```
Canvas   #0f0f11  — фон сторінки, порожнеча (the void)
Surface  #18181d  — картки, сайдбари, панелі
Elevated #222228  — випадаючі списки, модальні вікна, підказки, стани наведення
```

Використовуйте `bg-canvas` для зовнішньої оболонки. Використовуйте `bg-surface` для будь-якої вкладеної області (картки, рядки сайдбару, заголовки таблиць). Використовуйте `bg-elevated` для всього, що з'являється (плаває) поверх основного інтерфейсу.

`bg-hover` (`#1a1a21`) використовується лише для інтерактивних елементів — цей колір знаходиться між canvas і surface, створюючи ледь помітне відчуття підняття елемента при наведенні курсору.

### Акцент

```
Accent       #2563eb  — основні дії, рамки фокусу, активні стани
Accent Hover #1d4ed8  — стан наведення/натискання акцентного елемента
Accent Muted rgba(37,99,235,0.15)  — приглушений фон для елементів з акцентом
```

**Один акцентний колір, який використовується дуже помірно.** Акцент має з'являтися на:

- Головних кнопках заклику до дії (Primary CTA)
- Активних елементах навігації
- Рамках фокусу (focus outlines) полів введення
- Ключових індикаторах стану (активна точка, активний бейдж)

Ніколи не використовуйте акцентний колір для основного тексту, підписів чи декоративних меж. Коли він з'являється, він одразу привертає увагу, тому використовуйте його лише там, де відбувається дія.

### Чорнило (Кольори тексту / Ink)

```
Ink           #e8e8f0  — основний вміст: заголовки, значення, важливі підписи
Ink Secondary #9494a8  — допоміжний текст: назви властивостей, метадані, плейсхолдери
Ink Muted     #808098  — декоративні іконки, підписи роздільників, третинні підказки
```

**Ніколи не використовуйте чистий білий колір.** Колір `#e8e8f0` сприймається як білий у контексті, але допомагає уникнути надмірної контрастності та втоми очей на темному фоні.

Ієрархія тексту забезпечується **кольором, а не лише розміром**. Назва властивості та її значення можуть мати однаковий розмір — розділяйте їх за допомогою класів `text-ink-secondary` та `text-ink`.

**Правило контрастності для `text-ink-muted`:**

- На canvas (5.0:1) та surface (4.6:1): підходить для звичайного тексту ✓
- На elevated (4.1:1): підходить для елементів інтерфейсу, але на межі для звичайного тексту — віддавайте перевагу `text-ink-secondary` для текстових блоків у модальних вікнах та випадаючих списках.

### Штрих (Межі / Stroke)

```
Stroke        #2a2a35  — стандартні роздільники, краї карток, рамки полів введення
Stroke Subtle #1e1e28  — ледь помітні роздільники рядків всередині елементів surface
```

Межі (strokes) мають бути майже невидимими. Якщо ви чітко бачите рамку, ймовірно, вона занадто яскрава. Використовуйте `border-stroke` для меж елементів, а `divide-stroke-subtle` — для внутрішніх роздільників рядків.

### Семантичні кольори

Семантичні кольори відрізняються для темної та світлої тем — кожен режим має свій набір кольорів, налаштований для оптимального контрасту.

**Темна тема** (за замовчуванням):

```
Success  #57f287  / bg rgba(87,242,135,0.12)  — прибуткові угоди, позитивні стани
Warning  #fee75c  / bg rgba(254,231,92,0.12)   — попередження про просадку, застереження
Error    #ed4245  / bg rgba(237,66,69,0.12)    — збитки, помилки валідації, небезпечні дії
```

**Світла тема** (налаштована для контрасту 4.5:1 — використовуються зовсім інші відтінки):

```
Success  #15803d  / bg rgba(21,128,61,0.12)   — темно-зелений (яскраво-зелений не проходить контраст на білому)
Warning  #92400e  / bg rgba(146,64,14,0.10)   — темний бурштин (жовтий не проходить на білому)
Error    #b91c1c  / bg rgba(185,28,28,0.12)   — темно-червоний
```

Завжди поєднуйте колір тексту з його фоновим варіантом для бейджів та блоків сповіщень — ніколи не робіть заливку суцільним кольором. Фон із прозорістю `0.12` вибрано навмисно: він достатній для маркування стану, але не перевантажує палітру.

**WCAG 1.4.1 — колір не може бути єдиним засобом передачі інформації.** Кожен стан помилки/успіху/попередження має додатково містити принаймні один із наступних елементів: іконку, текстовий підпис, зміну межі або підкреслення.

## Типографіка

### Шрифти

**Екранний текст та інтерфейс (Display & UI):** `Geist Sans` — чистий, технічний, злегка звужений шрифт. Ідеально підходить для професійного інструменту. Використовується для всього тексту інтерфейсу, підписів, заголовків та навігації.

**Моноширинний шрифт (Monospace):** `Geist Mono` — для чисел, ідентифікаторів, коду та часових міток. Будь-які значення, які виграють від вирівнювання фіксованої ширини символів.

Обидва шрифти вже підключені в `RootLayout` через пакет `geist`. Посилайтеся на них за допомогою класів `font-sans` (Geist Sans) та `font-mono` (Geist Mono).

### Типографічна сітка (Type Scale)

Використовуйте семантичні назви класів — уникайте використання випадкових утиліт розміру Tailwind для текстових ролей.

```css
.type-page-title    text-2xl font-bold text-ink          /* Заголовки сторінок, назви записів */
.type-panel-title   text-lg font-bold text-ink           /* Заголовки розділів, шапки модальних вікон */
.type-modal-title   text-base font-bold text-ink         /* Заголовки всередині модальних вікон */
.type-field-label   text-xs font-semibold uppercase tracking-wide text-ink-secondary  /* Підписи властивостей */
.type-form-label    text-sm font-semibold text-ink-secondary  /* Підписи полів форми */
.type-hint          text-xs text-ink-muted               /* Допоміжний текст, підказки */
.type-nav-label     text-[11px] font-semibold uppercase tracking-wider text-ink-muted /* Заголовки розділів навігації */
```

**Заголовки записів** (головний заголовок на сторінці детального перегляду запису) використовують клас `text-4xl font-bold text-ink` — це значення навмисно більше за `.type-page-title`. Резервуйте цей розмір лише для основного об'єкта сторінки.

**Основний текст** за замовчуванням має стиль `text-sm text-ink` всередині компонентів. Клас `text-base` використовується дуже рідко — лише для звичайного зв'язаного тексту, ніколи для підписів чи метаданих.

**Числа та фінансові суми** завжди мають використовувати `font-mono` з `tabular-nums`. Відображення показників P&L пропорційним шрифтом вважається помилкою розробки.

### Правила типографіки

- Товщина шрифту для інтерактивних елементів має бути не менше ніж `font-medium`.
- Підписи (labels) малого розміру мають стиль `uppercase tracking-wide` для компенсації зменшення читабельності.
- Довгий зв'язаний текст: `leading-relaxed` (1.625). Короткий текст інтерфейсу: leading за замовчуванням.
- Ніколи не робіть підписи інтерфейсу курсивом — курсив зарезервовано лише для логічного виділення всередині контенту.

### Числові та фінансові дані

Фінансові показники трейдера мають бути максимально зручними для сканування очима — ніколи не виводьте їх пропорційним шрифтом.

- Усі показники P&L, ціни, відсотки, обсяги: `font-mono tabular-nums`
- Позитивні значення: `text-success` з явним префіксом `+` — `+2 340.00`
- Негативні значення: `text-error` з префіксом `−` — `−1 120.50`
- Нульові / нейтральні значення: `text-ink-secondary`
- Роздільник тисяч: пробіл (` `) — більш читабельний, ніж кома в щільних таблицях
- Ніколи не додавайте анімацію для зміни чисел — дозвольте значенню просто оновитися

## Відступи та проміжки (Spacing)

FIX Space використовує стандартну шкалу відступів Tailwind. Основні правила:

| Контекст                        | Відступ                                                                      |
| ------------------------------- | ---------------------------------------------------------------------------- |
| Горизонтальний відступ сторінки | `px-6` (компактний) / `px-8` (комфортний) / `px-12` (сторінки на всю ширину) |
| Верхній відступ сторінки        | від `pt-8` до `pt-12`                                                        |
| Внутрішній відступ картки       | від `p-4` до `p-6`                                                           |
| Відступи модальних вікон        | `px-6 py-4` (шапка/підвал) · `px-6 py-5` (тіло)                              |
| Проміжки між полями форми       | `gap-4` між полями                                                           |
| Проміжки між іконкою та текстом | від `gap-1.5` до `gap-2`                                                     |
| Висота рядка властивості        | `py-3` (відступи рядка)                                                      |

**Принцип щільності:** FIX Space — це інформаційно насичений інтерфейс. Віддавайте перевагу меншим відступам (`py-2`, `gap-3`) всередині компонентів із великою кількістю даних (таблиці, списки властивостей). Більші відступи залишайте для стартових зон: заголовків сторінок, пустих станів (empty states), сторінок авторизації.

## Шаблони компонентів

### Кнопки (Buttons)

Доступні чотири варіанти кнопок — використовуйте кожен варіант лише за його прямим призначенням.

| Варіант     | Сфера застосування             | Стиль                                                |
| ----------- | ------------------------------ | ---------------------------------------------------- |
| `primary`   | Основна дія форми, створення   | `bg-accent text-white`                               |
| `secondary` | Скасування дії, другорядна дія | `bg-surface border border-stroke text-ink-secondary` |
| `ghost`     | Іконки на панелях, inline-дії  | Прозорий фон, `hover:bg-surface`                     |
| `danger`    | Руйнівні дії (видалення)       | `bg-error text-white`                                |

Два розміри кнопок:

- `md` — стандартний розмір для більшості кнопок
- `sm` — компактний розмір для таблиць, панелей інструментів, контекстних дій
- `icon` — квадратна кнопка лише для іконки (`p-1.5`)

**Ніколи не створюйте власні стилі кнопок** шляхом перевизначення класів. Якщо жоден із чотирьох варіантів не підходить, перегляньте дизайн взаємодії.

### Поля введення (Inputs)

Усі текстові поля введення використовують клас `.field-input`:

```css
w-full rounded-lg border border-stroke bg-canvas px-3 py-2 text-sm text-ink
outline-none focus:border-accent transition-colors duration-150
placeholder:text-ink-muted
```

Стан фокусу: виключно `border-accent` — без тіней (box-shadow) та світіння навколо елемента. Чисто та професійно.

**Елементи вибору (Select)** також використовують клас `.field-input` разом із `cursor-pointer`.

### Картки (Cards)

Клас `bg-surface border border-stroke rounded-2xl` є стандартом оболонки для карток. Для елементів, що плавають поверх інтерфейсу (модальні вікна, випадаючі списки), використовуйте `bg-elevated`.

Внутрішні роздільники секцій реалізуйте через `border-t border-stroke` — уникайте порожніх блоків-роздільників.

### Бейджі та теги (Badges / Tags)

Використовуйте компонент `<Badge>` із відповідними семантичними кольорами:

- Значення статусів: кольоровий бейдж із параметром `{ color }` (береться з конфігурації STATUS-властивості)
- Нейтральні теги: `variant="neutral"` → `bg-elevated text-ink-secondary`
- Акцентне виділення: `variant="accent"` → `bg-accent/10 text-accent`

Текст бейджа завжди має стиль `text-xs font-medium`. Внутрішні відступи: `px-2 py-0.5`.

### Модальні вікна (Modals)

Структура модального вікна:

```
┌─ fixed inset-0 backdrop-blur-[3px] bg-canvas/50 z-50 ──────────────┐
│  ┌─ w-120 max-h-[80vh] rounded-2xl border-stroke bg-elevated ─────┐ │
│  │  Шапка (Header): px-6 py-4 border-b                            │ │
│  │  Тіло (Body):    px-6 py-5 overflow-y-auto scrollbar           │ │
│  │  Підвал (Footer): px-6 py-4 border-t                           │ │
│  └─────────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

Ширина модального вікна: `w-120` (480px) для стандартних форм. Широкі вікна (налаштування, складні конфігуратори): `max-w-2xl`.

Завжди рендерить модальне вікно через `createPortal(document.body)` та забезпечуйте закриття по клавіші Escape за допомогою хука `useEscape()`.

### Таблиці (Tables)

Оболонка таблиці: `rounded-lg border border-stroke overflow-hidden`.

Рядок заголовків: `bg-surface border-b border-stroke`.

Рядки з даними: `border-b border-stroke last:border-b-0`. Для парних/непарних рядків використовуйте чергування кольорів `bg-surface/30` на кожному непарному рядку — це полегшує читання без створення ефекту надмірної строкатості.

Внутрішні відступи комірок: `px-3 py-2.5`.

Заголовки стовпців: мають стиль `.type-field-label` (верхній регістр, другорядний колір).

## Іконографіка

Використовуйте виключно іконки з пакету **Lucide React**. Стандарти розмірів:

| Контекст                             | Розмір  |
| ------------------------------------ | ------- |
| Іконки типів властивостей у таблицях | `13`    |
| Навігація / іконки сайдбару          | `16`    |
| Іконки у кнопках (разом із текстом)  | `14–16` |
| Кнопки лише з іконкою                | `16–18` |
| Іконки великих порожніх станів       | `32–48` |

Колір іконки має відповідати кольору тексту: `text-ink-muted` для декоративних іконок, `text-ink-secondary` для структурних елементів та `text-ink` лише тоді, коли іконка є головним акцентом.

**Ніколи не використовуйте кольорові іконки** — це високоточний інструмент, а не інфографіка. Кольори зарезервовані лише для семантичних станів (успіх/попередження/помилка) та головного акценту.

## Рух та анімація (Motion & Animation)

Елементи в FIX Space рухаються плавно та осмислено, а не заради прикраси.

### Дозволені анімації

**`animate-fade-up`** — анімація загального призначення для появи елементів.

```css
opacity: 0 → 1  +  translateY(10px → 0)  |  duration: 0.45s  |  ease: ease
```

Застосовується для: контентних блоків сторінки, вмісту модальних вікон, нових елементів у списках.

**`animate-dropdown`** — анімація появи випадних списків (dropdowns), поповерів (popovers) та контекстних меню.

```css
opacity: 0 → 1  +  translateY(-8px → 0)  +  scale(0.98 → 1)  |  duration: 0.2s  |  ease: cubic-bezier(0.16, 1, 0.3, 1)
```

Застосовується для: випадних меню, тултипів, списків вибору (select options).

**`animate-spin`** — виключно для індикаторів завантаження.
Приклад: `w-5 h-5 rounded-full border-2 border-stroke border-t-accent animate-spin`

**`transition-colors duration-150`** — для всіх змін інтерактивних станів елементів (наведення, фокус).

**`transition-all duration-150`** — виключно для зміни станів кнопок.

### Правила анімації

- **Заборонено використовувати трансформації (transform)** для життєво важливих елементів інтерфейсу (сайдбари, шапки сторінок).
- **Жодних пружних (bounce, spring) або еластичних ефектів** — це торгова платформа, а не іграшка.
- Завантаження сторінки: застосовуйте один ефект `animate-fade-up` для всього основного блоку контенту, а не для кожного елемента окремо.
- Почергова поява елементів: використовуйте `animation-delay` з кроком від `50ms` до `75ms` для елементів списків.
- Максимальна тривалість: `300ms` для звичайних переходів, `500ms` для анімацій повної появи елементів.
- Ніколи не анімуйте зміну кольору фінансових даних — цифри мають оновлюватися миттєво.

## Нашарування інтерфейсу (Elevation & Layering)

```
Ієрархія z-index:
  z-[100]  — діалоги підтвердження дій, критичні системні повідомлення
  z-50     — модальні вікна, оверлеї сторінок
  z-40     — випадаючі списки, підказки (tooltips)
  z-30     — закріплені заголовки таблиць
  z-10     — закріплений сайдбар сторінки
  z-0      — базовий рівень контенту
```

Шапка сторінки (Header) використовує класи `z-50 sticky top-0` разом із ефектом розмиття `backdrop-blur-md bg-canvas/85` — вона знаходиться поверх контенту сторінки, але сприймається як частина інтерфейсу, а не окремий плаваючий блок.

## Принципи макетування (Layout Principles)

### Сітка та вирівнювання

FIX Space не використовує класичний лівий сайдбар як основний каркас — застосовано модель **top-nav (верхнє меню) + вміст на всю ширину**. Шапка сторінки є єдиним фіксованим елементом; все, що нижче — це безпосередньо робочий простір.

Зони відображення контенту:

- **Обмежені текстові сторінки** (авторизація, налаштування): центрування по класу `max-w-sm`.
- **Сторінки документів** (детальний перегляд запису): `px-12` на всю ширину, або `max-w-4xl` для комфортного читання.
- **Таблиці даних**: завжди розтягуються на всю ширину сторінки без обмеження максимального розміру.
- **Форми всередині карток**: мають ширину від `max-w-sm` до `max-w-md`.

### Візуальна ієрархія сторінки

Кожна сторінка повинна мати **рівно один домінуючий візуальний елемент**, на якому одразу фокусується погляд користувача при переході:

- Сторінка бази даних → безпосередньо таблиця
- Сторінка запису → назва запису
- Сторінка авторизації → картка з формою входу
- Розділ налаштувань → вміст активного розділу

Всі інші елементи мають відступати на другий план. Меню навігації, підписи, другорядні дії мають кольори `text-ink-secondary` або `text-ink-muted`.

### Порожні стани (Empty States)

Порожні стани мають виглядати естетично та спонукати до дії, а не створювати відчуття помилки. Структура порожнього стану:

```
центрований flex-col зgap-3
  іконка: 32px, колір text-ink-muted
  заголовок: text-sm font-medium, колір text-ink-secondary
  кнопка дії (опціонально): Button з варіантом variant="secondary" та розміром size="sm"
```

Ніколи не використовуйте для тексту порожнього стану основний колір `text-ink` — він має відчуватися як спокійне запрошення до дії, а не як повідомлення про помилку.

## Антипатерни (Anti-Patterns)

Ці практики є суворо забороненими в інтерфейсі FIX Space:

- **Використання `text-accent` для звичайного тексту чи підписів** — контрастність акцентного кольору становить ~3.7:1 (темна тема) / ~4.6:1 (світла тема), що підходить лише для елементів інтерфейсу (поріг 3:1). Ніколи не виводьте цим кольором звичайний текст або підказки — використовуйте його лише для посилань, активних пунктів меню та індикаторів фокусу.
- **Градієнтні фони** — заборонено використовувати класи `bg-gradient-to-*` для будь-яких структурних елементів інтерфейсу.
- **Кольорова заливка іконок** — іконки мають бути суто монохромними.
- **Використання кількох акцентних кольорів** — колір `#2563eb` є єдиним брендовим кольором; не додавайте інші.
- **Тіні (box shadows) на темних поверхнях** — тіні працюють лише на світлому фоні; в темній темі використовуйте рамки та рівні висоти (elevation).
- **Вбудовані стилі `style=` для відступів** — використовуйте виключно класи Tailwind; атрибут `style=` дозволений тільки для виведення кольорів, обраних користувачем.
- **Використання класу `text-white`** — використовуйте класи `text-ink` або `text-white` лише на кольоровому фоні (акцентний колір або стан помилки).
- **Довільні радіуси заокруглення** — використовуйте `rounded-lg` (8px) для полів введення та рядків, `rounded-2xl` (16px) для карток та модальних вікон, а `rounded-full` лише для аватарів та круглих чіпсів.
- **Стани наведення (hover) без плавного переходу** — всі класи `hover:` мають супроводжуватися класом `transition-colors duration-150`.
- **Великі текстові блоки у верхньому регістрі** — верхній регістр (uppercase) дозволений лише для типів класів `.type-field-label` та `.type-nav-label` (малі розміри тексту з додатковим інтервалом літер).

## Доступність (Accessibility / a11y)

Мінімальний набір вимог для забезпечення доступності інтерфейсу в FIX Space.

### Видимість фокусу

Усі інтерактивні елементи повинні чітко показувати рамку фокусу при навігації з клавіатури. Використовуйте класи:

```
focus-visible:ring-2 focus-visible:ring-accent focus-visible:ring-offset-1 focus-visible:ring-offset-canvas
```

Для полів введення `.field-input` ця вимога реалізується через клас `focus:border-accent`. Для кнопок та елементів лише з іконками обов'язково додавайте рамку фокусу через клас `focus-visible:`.

Ніколи не відключайте рамку фокусу (`outline: none`) на глобальному рівні без надання альтернативного стилю фокусування.

### Зменшення руху (Reduced motion)

Анімації появи елементів мають бути адаптовані під системні налаштування користувача. Додайте у файл `utilities.css`:

```css
@media (prefers-reduced-motion: reduce) {
  .animate-fade-up,
  .animate-dropdown {
    animation: none;
  }
}
```

Короткі переходи (`duration-150`) не перевантажують сприйняття та не вимагають спеціального відключення.

### Контрастність тексту

| Токен тексту         | Колір фону  | Контраст | Рівень WCAG                       |
| -------------------- | ----------- | -------- | --------------------------------- |
| `text-ink`           | `bg-canvas` | ~14:1    | AAA ✓                             |
| `text-ink-secondary` | `bg-canvas` | ~4.6:1   | AA ✓                              |
| `text-ink-muted`     | `bg-canvas` | ~2.8:1   | Тільки для декоративних елементів |

Клас `text-ink-muted` не проходить стандарт контрастності AA для тексту, тому використовуйте його виключно для іконок, невеликих підказок та неактивних станів елементів.

## Чек-лист для нових компонентів

Перед додаванням будь-якого нового компонента перевірте наступні пункти:

- [ ] Компонент використовує лише заздалегідь визначені кольорові токени — жодних жорстко прописаних кодів кольорів (HEX).
- [ ] Текст використовує семантичні класи або затверджені розміри сітки шрифтів.
- [ ] Оброблено всі інтерактивні стани елементів: наведення (`hover:`), отримання фокусу (`focus:`), неактивний стан (`disabled:`).
- [ ] Стан завантаження відображає спеціальний спінер, а не порожній простір чи заблоковану кнопку.
- [ ] Стан помилки використовує колір `text-error` та показує зрозуміле повідомлення замість тихого збою в роботі.
- [ ] Пусті стани компонентів мають чітке повідомлення (та бажано кнопку дії).
- [ ] Усі переходи (transitions) тривають рівно `150ms` (не 200, не 300 для звичайного наведення курсору).
- [ ] Числа відображаються моноширинним шрифтом `font-mono tabular-nums`, якщо вони позначають суми чи кількісні показники.
- [ ] Компонент коректно адаптується та працює на вузьких екранах (від 320px).
