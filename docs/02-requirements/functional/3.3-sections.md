## 3.3 Секції

- 📝 Додано до Issue (#99) **Секція** — це іменована група баз даних усередині робочого простору. Вона не зберігає записи і не має власних налаштувань даних — її єдина функція полягає в організації навігаційної структури сайдбару.
- 📝 Додано до Issue (#135) Секції залишаються необхідним рівнем організації незалежно від кількості баз даних: зі зростанням їх числа плоский список стає незручним для навігації, тоді як секції дозволяють зберігати чітку структуру. Можливість згортати секції в сайдбарі зменшує когнітивне навантаження під час активної торгової сесії — видно лише те, що потрібно зараз.

**Функціональні вимоги:**

- 📝 Додано до Issue (#59) Автоматичне створення стартових секцій при ініціалізації робочого простору
- 📝 Додано до Issue (#99) Створення нової секції
- 📝 Додано до Issue (#100) Редагування назви, іконки та кольору секції
- 📝 Додано до Issue (#102) Зміна порядку секцій у сайдбарі
- 📝 Додано до Issue (#136) Переміщення бази даних між секціями
- 📝 Додано до Issue (#102) Зміна порядку баз даних усередині секції
- 📝 Додано до Issue (#101) Видалення секції (бази даних із видаленої секції залишаються в просторі без прив'язки до секції та відображаються в сайдбарі нижче всіх секцій)
- 📝 Додано до Issue (#135) Згортання та розгортання секції в сайдбарі
- 📝 Додано до Issue (#135) Збереження стану згортання кожної секції між сесіями
