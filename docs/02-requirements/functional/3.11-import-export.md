## 3.11 Імпорт та Експорт (Фактична реалізація)

Імпорт та експорт даних дозволяють переносити накопичені торгові дані та вивантажувати записи для аналізу у форматі CSV (Comma-Separated Values).

### 1. Імпорт CSV

Імпорт дозволяє завантажити дані з CSV-файлів до бази даних із налаштуванням зіставлення колонок та валідацією типів.

**Фактичні функціональні вимоги та логіка роботи:**

- [x] **Обмеження файлу:** Завантаження файлу CSV кодування UTF-8 розміром до 25 МБ. Перевіряються MIME-типи (`text/csv`, `application/csv`, `text/plain`) та розширення файлу `.csv`.
- [x] **Визначення заголовків (Header Detection):** Алгоритм сканує перші 50 рядків CSV-файлу і обирає як рядок заголовків той, що містить найбільшу кількість заповнених колонок. Це дозволяє імпортувати файли з метаданими від брокерів у верхній частині.
- [x] **Попередній перегляд (Preview):** Зчитування файлу та повернення списку знайдених колонок, перших 5 рядків даних як прев'ю та загальної кількості рядків у файлі.
- [x] **Зіставлення колонок (Mapping):** Можливість налаштування мапінгу колонок CSV на властивості бази даних. Колонка може бути зіставлена з `__name__` для призначення назви запису.
- [x] **Підтримка типів:** Підтримується імпорт для властивостей типу `TEXT`, `NUMBER`, `DATE`, `CHECKBOX`, `SELECT` та `STATUS`.
- [x] **Виключені типи:** Властивості типу `RELATION` та `FORMULA` повністю виключені з імпорту (`EXCLUDED_TYPES`), оскільки RELATION потребує UUID-зв'язків, а FORMULA розраховується автоматично.
- [x] **Рядок за рядком валідація:** Значення кожного рядка конвертуються відповідно до типу властивості та валідуються правилами обробника поля. Рядки з помилками валідації пропускаються, формуючи детальний звіт про помилки.
- [x] **Обмеження за лімітом бази даних (Limit Warning):** Якщо в базі встановлено ліміт на кількість записів, імпорт перевіряє доступне місце. У разі перевищення ліміту повертається попередження із зазначенням ліміту, поточного рахунку та кількості записів, які можливо імпортувати.
- [x] **Покроковий bulk-імпорт:** Імпорт коректних записів виконується в межах однієї транзакції частинами (chunks) по 100 записів для оптимізації роботи з базою даних.
- [x] **Параметр maxRows:** Підтримується примусове обмеження кількості імпортованих рядків через опціональний параметр `maxRows`.
- [x] **Звіт про виконання:** Повертається кількість імпортованих записів, кількість пропущених та масив помилок із зазначенням номера рядка (`rowIndex`) та причини помилки.
- [x] **Системне сповіщення:** Після успішного завершення імпорту користувач отримує in-app сповіщення типу `INFO` із підсумком («Імпорт завершено: N записів додано, M пропущено»).

### 2. Експорт CSV

Експорт дозволяє вивантажити записи бази даних у CSV-файл із застосуванням фільтрів та вибором полів.

**Фактичні функціональні вимоги та логіка роботи:**

- [x] **Префікс BOM:** Файл експортується в UTF-8 із додаванням префіксу BOM (Byte Order Mark), що забезпечує коректне відображення кирилиці та спецсимволів при відкритті у Microsoft Excel.
- [x] **Форматування значень:** Числові, логічні, текстові поля та дати форматуються відповідно до системних вимог (наприклад, boolean записується як `true`/`false`).
- [x] **Вибір властивостей:** Опціональний експорт лише вибраних за ID властивостей (`propertyIds`).
- [x] **Врахування мета-полів:** За замовчуванням експорт включає мета-поля `Name`, `Created At` та `Updated At`. За необхідності їх відображення можна вимкнути (`includeMetaFields = false`).
- [x] **Виключені типи:** Властивості типу `RELATION` та `FORMULA` виключені з експорту (`EXCLUDED_EXPORT_TYPES`).
- [x] **Фільтрація за активним поданням (View Filters):** Якщо передано `viewId`, експорт застосовує активні фільтри та логіку фільтрації (AND/OR) цього подання до записів перед генерацією файлу.
- [x] **Формат імені файлу:** Генерується у форматі `${safe_database_name}_export_YYYY-MM-DD.csv`, де всі неалфавітно-цифрові символи назви бази замінюються на підкреслення `_`.
