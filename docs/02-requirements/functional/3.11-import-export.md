## 3.11 Імпорт та Експорт

Імпорт та експорт даних вирішують дві окремі задачі. **Імпорт** дозволяє перенести накопичені дані до FIX Space без ручного введення кожного запису: трейдер, що до цього вів журнал у Google Sheets або Excel, може зберегти всю свою історію угод і продовжити роботу в новій системі. **Експорт** дозволяє вивантажити записи для подальшого аналізу в зовнішніх інструментах, передачі ментору або створення резервної копії.

Підтримуваний формат — CSV (Comma-Separated Values): він підтримується всіма табличними редакторами та є стандартом вивантаження даних у більшості брокерських терміналів і торгових платформ (MT4/MT5, cTrader, IBKR).

**Функціональні вимоги:**

Імпорт:

- 📝 Додано до Issue (#110) Завантаження CSV-файлу до обраної бази даних (максимальний розмір файлу — 25 МБ; кодування UTF-8)
- 📝 Додано до Issue (#110) Попередній перегляд перших рядків файлу перед підтвердженням
- 📝 Додано до Issue (#110) Відображення зіставлення полів CSV-файлу із властивостями бази даних (field mapping); властивості типу RELATION виключені з маппінгу — CSV містить текстові значення, а не UUID записів, тому RELATION-поля не можуть бути встановлені через CSV-імпорт
- 📝 Додано до Issue (#110) Можливість вручну змінити зіставлення (вибрати яку властивість відповідає якому стовпцю CSV)
- 📝 Додано до Issue (#110) Валідація значень перед імпортом (відповідність типу властивості)
- 📝 Додано до Issue (#110) Відображення підсумку перед підтвердженням (скільки записів буде створено, скільки рядків пропущено через помилки валідації)
- 📝 Додано до Issue (#110) При перевищенні ліміту записів бази даних: у підсумку відображається попередження з кількістю рядків у файлі та поточним лімітом; пропонується імпортувати лише перші N дозволених записів або скасувати; імпорт без попередження ніколи не обрізається мовчки
- 📝 Додано до Issue (#110) Підтвердження та виконання імпорту: некоректні рядки пропускаються, коректні імпортуються
- 📝 Додано до Issue (#110) Відображення фінального звіту після імпорту (кількість імпортованих записів, список пропущених рядків із причиною)
- 📝 Додано до Issue (#165) Відображення фінального звіту після імпорту (кількість імпортованих записів, список пропущених рядків із причиною)

Експорт:

- 📝 Додано до Issue (#111) Вивантаження записів бази даних у CSV-файл (кодування UTF-8)
- 📝 Додано до Issue (#111) Вибір набору властивостей для включення до файлу (або всі)
- 📝 Додано до Issue (#111) Врахування активних фільтрів подання: експорт усіх записів або лише відфільтрованих
- 📝 Додано до Issue (#111) Включення мета-полів запису (назва, дата створення, дата оновлення)
