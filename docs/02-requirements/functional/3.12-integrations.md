## 3.12 Інтеграція із зовнішніми сервісами

FIX Space підтримує автоматичну синхронізацію торгових даних із зовнішніх джерел. Це вирішує головну рутинну задачу трейдера — необхідність вручну записувати кожну проведену угоду. Інтеграція працює у фоновому режимі та автоматично переносить закриті позиції у Trading Journal. <!-- US-091, US-092, US-094, US-095 -->

Підтримуються три класи джерел даних:

- **Криптовалютні біржі** (Binance, Bybit, OKX) — підключення через API Key / Secret (read-only); покривають Spot, Margin та Futures ринки. <!-- US-091 -->
- **CFD/Forex-термінали** (MetaTrader 5, cTrader) — через вбудовані механізми read-only доступу без права торгівлі; покривають Forex, CFD на індекси, метали та енергоносії. <!-- US-094, US-095 -->

Синхронізовані записи потрапляють до бази Trading Journal як повноцінні записи: їх можна редагувати, доповнювати нотатками та контентом, видаляти. Мітка джерела (наприклад, «Binance», «MT5») зберігається як атрибут запису. <!-- US-092 -->

### Криптовалютні біржі (Binance, Bybit, OKX)

**Підтримувані ринки:**

| Біржа       | Ринки                                                 |
| ----------- | ----------------------------------------------------- |
| **Binance** | USDT-M Perpetual Futures, COIN-M Futures, Spot        |
| **Bybit**   | USDT Perpetual, Inverse Perpetual, USDC Futures, Spot |
| **OKX**     | Perpetual Swaps, Delivery Futures, Spot               |

**Функціональні вимоги:**

- [ ] Підключення біржі через API Key + Secret (read-only) <!-- US-091 -->
- [ ] Перевірка прав ключа при підключенні: якщо ключ має права на торгівлю або виведення коштів — підключення відхиляється з відповідним повідомленням <!-- US-091 -->
- [ ] Вибір типу ринку для синхронізації (Futures / Spot або обидва) <!-- US-091 -->
- [ ] Автоматична синхронізація нових закритих угод з інтервалом 5 хвилин <!-- US-092 -->
- [ ] Ручний запуск синхронізації на вимогу <!-- US-093 -->
- [ ] Завантаження торгової історії за вибраний діапазон дат при першому підключенні (глибина залежить від біржі: до 2 років для Bybit V5; для Binance — до 30 днів через API, решта — через CSV-імпорт) <!-- US-091 -->
- [ ] Зіставлення полів біржі з властивостями бази Trading Journal: символ, напрямок (Long / Short), обсяг, ціна входу/виходу, P&L, комісія, funding rate (для perpetual), час відкриття та закриття, тип ринку, леверидж <!-- US-091 -->
- [ ] Ігнорування дублікатів при повторній синхронізації (ідентифікація за унікальним position ID біржі) <!-- US-092 -->
- [ ] Автоматичне заповнення поля `Account` у синхронізованому записі Trading Journal — підбирається запис Accounts, де поле `Account ID` відповідає ідентифікатору підключеного акаунту інтеграції; якщо збіг не знайдений — поле залишається порожнім <!-- US-091 -->
- [ ] Відображення статусу підключення: активне / помилка автентифікації / відключене <!-- US-097 -->
- [ ] Відключення інтеграції (збережені записи залишаються в журналі) <!-- US-097 -->
- [ ] Ліміт: до 3 підключень на кожну біржу на акаунт; кожне підключення отримує довільну мітку-назву (наприклад, «Binance Main», «Binance Prop») <!-- US-091 -->

### MetaTrader 5

MetaTrader 5 є основною платформою CFD та Forex-брокерів. Підключення здійснюється через Investor Password — спеціальний режим read-only доступу, вбудований у MT5, без права торгівлі та без необхідності встановлення додаткового ПЗ на стороні FIX Space.

**Підтримувані ринки:**

- [ ] Forex (EUR/USD, GBP/USD тощо) <!-- US-094 -->
- [ ] CFD на індекси (S&P 500, DAX, FTSE) <!-- US-094 -->
- [ ] CFD на метали (Gold, Silver) <!-- US-094 -->
- [ ] CFD на енергоносії (Oil, Gas) <!-- US-094 -->
- [ ] Криптовалюта (якщо пропонується брокером) <!-- US-094 -->

**Функціональні вимоги:**

- [ ] Підключення MT5-акаунту через: login, investor password, broker server name <!-- US-094 -->
- [ ] Перевірка з'єднання при збереженні налаштувань із відображенням результату <!-- US-094 -->
- [ ] Вибір діапазону дат для початкової синхронізації (завантаження накопиченої торгової історії) <!-- US-094 -->
- [ ] Автоматична синхронізація нових закритих угод <!-- US-092 -->
- [ ] Ручний запуск синхронізації на вимогу <!-- US-093 -->
- [ ] Зіставлення полів MT5 з властивостями бази Trading Journal: символ, тип (Buy / Sell), обсяг у лотах, ціна відкриття/закриття, час відкриття/закриття, прибуток, своп, комісія, коментар брокера <!-- US-094 -->
- [ ] Ігнорування дублікатів (ідентифікація за position ID MT5) <!-- US-092 -->
- [ ] Автоматичне заповнення поля `Account` у синхронізованому записі Trading Journal — підбирається запис Accounts, де поле `Account ID` відповідає login підключеного MT5-акаунту; якщо збіг не знайдений — поле залишається порожнім <!-- US-094 -->
- [ ] Відображення статусу підключення: активне / помилка автентифікації / відключене <!-- US-097 -->
- [ ] Відключення інтеграції (збережені записи залишаються в журналі) <!-- US-097 -->
- [ ] Ліміт: до 5 MT5-підключень на акаунт (різні брокери або різні акаунти одного брокера); кожне підключення отримує довільну мітку-назву <!-- US-094 -->

### cTrader

cTrader — платформа CFD та Forex-брокерів (IC Markets, Pepperstone, FxPro та 50+ інших). Підключення здійснюється через cTrader Open API (Spotware) за протоколом OAuth 2.0: користувач авторизує FIX Space у своєму cTrader-акаунті без передачі пароля — лише видаючи обмежені права на читання торгової історії.

**Підтримувані ринки:**

- [ ] Forex (EUR/USD, GBP/USD тощо) <!-- US-095 -->
- [ ] CFD на індекси (S&P 500, DAX, FTSE) <!-- US-095 -->
- [ ] CFD на метали (Gold, Silver) <!-- US-095 -->
- [ ] CFD на енергоносії (Oil, Gas) <!-- US-095 -->
- [ ] Криптовалюта (якщо пропонується брокером) <!-- US-095 -->

**Функціональні вимоги:**

- [ ] Підключення cTrader-акаунту через OAuth 2.0 — без введення пароля: користувач перенаправляється на сторінку авторизації cTrader і підтверджує доступ <!-- US-095 -->
- [ ] Перегляд списку торгових акаунтів, доступних через авторизований cTrader ID, та вибір одного або кількох для синхронізації; кожен вибраний акаунт стає окремим підключенням із власною міткою-назвою (за замовчуванням — назва брокерського акаунту з cTrader) <!-- US-095 -->
- [ ] Вибір діапазону дат для початкової синхронізації (завантаження накопиченої торгової історії через ProtoOADealListReq) <!-- US-095 -->
- [ ] Автоматична синхронізація нових закритих угод <!-- US-092 -->
- [ ] Ручний запуск синхронізації на вимогу <!-- US-093 -->
- [ ] Зіставлення полів cTrader з властивостями бази Trading Journal: символ, напрямок (Buy / Sell), обсяг, ціна відкриття/закриття, час відкриття/закриття, прибуток, своп, комісія <!-- US-095 -->
- [ ] Ігнорування дублікатів (ідентифікація за position ID cTrader) <!-- US-092 -->
- [ ] Відображення статусу підключення: активне / токен прострочено / відключене <!-- US-097 -->
- [ ] При простроченні OAuth-токена — повідомлення у застосунку з пропозицією повторно авторизуватись <!-- US-097 -->
- [ ] Відключення інтеграції з відкликанням OAuth-токена; збережені записи залишаються в журналі <!-- US-097 -->
- [ ] Ліміт: необмежена кількість cTrader-акаунтів на акаунт FIX Space (кожен вибраний акаунт після OAuth = окреме підключення) <!-- US-095 -->

### Загальна логіка синхронізації (Background Sync)

- [ ] **Одиниця синхронізації — позиція:** система синхронізує закриті позиції, а не окремі часткові виконання (fills). Одна закрита позиція (від відкриття до повного закриття) = один запис у базі Trading Journal. Ціна входу та виходу розраховуються як середньозважені по всіх fills позиції. Для Binance та Bybit використовується Position History API, який надає агреговані дані; для MT5 — групування по position ID. <!-- US-092 -->
- [ ] **Конфлікт синхронізації та редагування:** синхронізація лише **створює нові записи** — вона ніколи не оновлює та не перезаписує записи, які вже існують у базі даних. Якщо запис з відповідним position ID вже є в журналі (незалежно від того, чи редагував його користувач), повторна синхронізація його ігнорує. Це гарантує збереження всіх нотаток, тегів, значень властивостей і контентної області, доданих вручну. <!-- US-092 -->
- [ ] **Маппінг полів:** маппінг між полями брокера і властивостями бази Trading Journal є фіксованим і прив'язаний до внутрішніх ідентифікаторів системних властивостей (не до їхніх назв). Перейменування системної властивості користувачем не порушує синхронізацію. Поля брокера, що не мають відповідної системної властивості, не імпортуються. <!-- US-091, US-094, US-095 -->
- [ ] **Funding rate (для perpetual futures):** funding rate за весь час утримання позиції вже включений у значення Realized P&L, яке повертає Position History API Binance та Bybit. Окремого запису для funding rate не створюється — він є частиною підсумкового P&L угоди. <!-- US-091, US-092 -->
- [ ] **Валюта:** числові значення (P&L, комісія, обсяг) зберігаються у валюті брокерського акаунту як є (USDT для Binance/Bybit USDT-M, USD або EUR для MT5 залежно від акаунту). Конвертація валют не виконується. Валюта акаунту відображається поруч із числовими значеннями в картці запису. <!-- US-091, US-094, US-095 -->
- [ ] **Початкова синхронізація:** при першому підключенні користувач вибирає діапазон дат для завантаження торгової історії. Значення за замовчуванням — останні 90 днів. Максимальна глибина залежатиме від обмежень API: Bybit — до 2 років, Binance — до 30 днів (решту завантажити через CSV-імпорт), MT5 та cTrader — необмежено (залежить від брокера). <!-- US-091, US-094, US-095 -->
- [ ] **Помилки синхронізації:** після 3 послідовних невдалих спроб автосинку система відображає in-app повідомлення (дзвіночок) з описом причини (помилка API, прострочений токен, мережева помилка) і посиланням на розділ Налаштування → Інтеграції. Автоматичні спроби продовжуються з інтервалом 30 хвилин до усунення проблеми. <!-- US-096 -->
- [ ] **Мітки джерела:** синхронізовані записи позначаються міткою підключення (задається користувачем при налаштуванні, наприклад «Binance Main», «IC Markets cTrader», «MT5 Prop»); мітка відображається в картці запису та доступна для фільтрації. <!-- US-091, US-094, US-095 -->
- [ ] **Безпека:** Credentials (API-ключі, паролі, OAuth-токени) зберігаються у зашифрованому вигляді; після збереження відображаються лише у замаскованому вигляді. <!-- US-091, US-094, US-095 -->

### Управління інтеграціями (Налаштування)

- [ ] Всі синхронізовані записи є повноцінними записами бази даних — їх можна редагувати, доповнювати нотатками та контентною областю, видаляти <!-- US-092 -->
- [ ] Видалення запису у FIX Space не впливає на дані на біржі або в терміналі <!-- US-092 -->
- [ ] Усі налаштування інтеграцій зосереджені в розділі Налаштування → Інтеграції <!-- US-097 -->
- [ ] Перепідключення (оновлення credentials): кожне підключення має кнопку «Перепідключити» — для OAuth (cTrader) запускається повторна авторизація, для API-ключів (Binance, Bybit, OKX) та паролів (MT5) відкривається форма з новими даними; після успіху старі credentials замінюються, синхронізація продовжується <!-- US-097 -->
- [ ] Відкликання (відключення): кнопка «Відключити» зупиняє синхронізацію та видаляє збережені credentials; для OAuth-підключень виконується відкликання токена; збережені записи залишаються в журналі без змін; перед відключенням — діалог підтвердження <!-- US-097 -->
- [ ] Після 3 послідовних невдалих спроб автосинку система відображає in-app повідомлення (дзвіночок) з описом причини <!-- US-096 -->
