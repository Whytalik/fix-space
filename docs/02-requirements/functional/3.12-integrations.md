## 3.12 Інтеграція із зовнішніми сервісами

**Мета інтеграції** — автоматичне отримання закритих угод із брокерських платформ і криптобірж без ручного введення. Це доповнення до CSV-імпорту (розділ 3.11): якщо CSV є універсальним fallback для будь-якого брокера, то API-підключення забезпечує регулярну автоматичну синхронізацію без участі трейдера.

Система підтримує два класи зовнішніх підключень:

- ❌ **Криптовалютні біржі** (Binance, Bybit, OKX) — через публічний REST API з read-only ключами; покривають крипто-ф'ючерси (USDT-M, COIN-M) та спотовий ринок.
- ❌ **CFD/Forex-термінали** (MetaTrader 5, cTrader) — через вбудовані механізми read-only доступу без права торгівлі; покривають Forex, CFD на індекси, метали та енергоносії.

Синхронізовані записи потрапляють до бази Trading Journal як повноцінні записи: їх можна редагувати, доповнювати нотатками та контентом, видаляти. Мітка джерела (наприклад, «Binance», «MT5») зберігається як атрибут запису.

### Криптовалютні біржі (Binance, Bybit, OKX)

**Підтримувані ринки:**

| Біржа       | Ринки                                                    |
| ----------- | -------------------------------------------------------- |
| **Binance** | ❌ USDT-M Perpetual Futures, COIN-M Futures, Spot        |
| **Bybit**   | ❌ USDT Perpetual, Inverse Perpetual, USDC Futures, Spot |
| **OKX**     | ❌ Perpetual Swaps, Delivery Futures, Spot               |

**Функціональні вимоги:**

- Підключення біржі через API Key + Secret (read-only)
- Перевірка прав ключа при підключенні: якщо ключ має права на торгівлю або виведення коштів — підключення відхиляється з відповідним повідомленням
- Вибір типу ринку для синхронізації (Futures / Spot або обидва)
- Автоматична синхронізація нових закритих угод з інтервалом 5 хвилин
- Ручний запуск синхронізації на вимогу
- Завантаження торгової історії за вибраний діапазон дат при першому підключенні (глибина залежить від біржі: до 2 років для Bybit V5; для Binance — до 30 днів через API, решта — через CSV-імпорт)
- Зіставлення полів біржі з властивостями бази Trading Journal: символ, напрямок (Long / Short), обсяг, ціна входу/виходу, P&L, комісія, funding rate (для perpetual), час відкриття та закриття, тип ринку, леверидж
- Ігнорування дублікатів при повторній синхронізації (ідентифікація за унікальним position ID біржі)
- Автоматичне заповнення поля `Account` у синхронізованому записі Trading Journal — підбирається запис Accounts, де поле `Account ID` відповідає ідентифікатору підключеного акаунту інтеграції; якщо збіг не знайдений — поле залишається порожнім
- Відображення статусу підключення: активне / помилка автентифікації / відключене
- Відключення інтеграції (збережені записи залишаються в журналі)
- Ліміт: до 3 підключень на кожну біржу на акаунт; кожне підключення отримує довільну мітку-назву (наприклад, «Binance Main», «Binance Prop»)

### MetaTrader 5

MetaTrader 5 є основною платформою CFD та Forex-брокерів. Підключення здійснюється через Investor Password — спеціальний режим read-only доступу, вбудований у MT5, без права торгівлі та без необхідності встановлення додаткового ПЗ на стороні FIX Space.

**Підтримувані ринки:**

- Forex (EUR/USD, GBP/USD тощо)
- CFD на індекси (S&P 500, DAX, FTSE)
- CFD на метали (Gold, Silver)
- CFD на енергоносії (Oil, Gas)
- Криптовалюта (якщо пропонується брокером)

**Функціональні вимоги:**

- Підключення MT5-акаунту через: login, investor password, broker server name
- Перевірка з'єднання при збереженні налаштувань із відображенням результату
- Вибір діапазону дат для початкової синхронізації (завантаження накопиченої торгової історії)
- Автоматична синхронізація нових закритих угод
- Ручний запуск синхронізації на вимогу
- Зіставлення полів MT5 з властивостями бази Trading Journal: символ, тип (Buy / Sell), обсяг у лотах, ціна відкриття/закриття, час відкриття/закриття, прибуток, своп, комісія, коментар брокера
- Ігнорування дублікатів (ідентифікація за position ID MT5)
- Автоматичне заповнення поля `Account` у синхронізованому записі Trading Journal — підбирається запис Accounts, де поле `Account ID` відповідає login підключеного MT5-акаунту; якщо збіг не знайдений — поле залишається порожнім
- Відображення статусу підключення: активне / помилка автентифікації / відключене
- Відключення інтеграції (збережені записи залишаються в журналі)
- Ліміт: до 5 MT5-підключень на акаунт (різні брокери або різні акаунти одного брокера); кожне підключення отримує довільну мітку-назву

### cTrader

cTrader — платформа CFD та Forex-брокерів (IC Markets, Pepperstone, FxPro та 50+ інших). Підключення здійснюється через cTrader Open API (Spotware) за протоколом OAuth 2.0: користувач авторизує FIX Space у своєму cTrader-акаунті без передачі пароля — лише видаючи обмежені права на читання торгової історії.

**Підтримувані ринки:**

- Forex (EUR/USD, GBP/USD тощо)
- CFD на індекси (S&P 500, DAX, FTSE)
- CFD на метали (Gold, Silver)
- CFD на енергоносії (Oil, Gas)
- Криптовалюта (якщо пропонується брокером)

**Функціональні вимоги:**

- Підключення cTrader-акаунту через OAuth 2.0 — без введення пароля: користувач перенаправляється на сторінку авторизації cTrader і підтверджує доступ
- Перегляд списку торгових акаунтів, доступних через авторизований cTrader ID, та вибір одного або кількох для синхронізації; кожен вибраний акаунт стає окремим підключенням із власною міткою-назвою (за замовчуванням — назва брокерського акаунту з cTrader)
- Вибір діапазону дат для початкової синхронізації (завантаження накопиченої торгової історії через ProtoOADealListReq)
- Автоматична синхронізація нових закритих угод
- Ручний запуск синхронізації на вимогу
- Зіставлення полів cTrader з властивостями бази Trading Journal: символ, напрямок (Buy / Sell), обсяг, ціна відкриття/закриття, час відкриття/закриття, прибуток, своп, комісія
- Ігнорування дублікатів (ідентифікація за position ID cTrader)
- Відображення статусу підключення: активне / токен прострочено / відключене
- При простроченні OAuth-токена — повідомлення у застосунку з пропозицією повторно авторизуватись
- Відключення інтеграції з відкликанням OAuth-токена; збережені записи залишаються в журналі
- Ліміт: необмежена кількість cTrader-акаунтів на акаунт FIX Space (кожен вибраний акаунт після OAuth = окреме підключення)

### Загальні вимоги до всіх інтеграцій

- **Одиниця синхронізації — позиція:** система синхронізує закриті позиції, а не окремі часткові виконання (fills). Одна закрита позиція (від відкриття до повного закриття) = один запис у базі Trading Journal. Ціна входу та виходу розраховуються як середньозважені по всіх fills позиції. Для Binance та Bybit використовується Position History API, який надає агреговані дані; для MT5 — групування по position ID.
- **Конфлікт синхронізації та редагування:** синхронізація лише **створює нові записи** — вона ніколи не оновлює та не перезаписує записи, які вже існують у базі даних. Якщо запис з відповідним position ID вже є в журналі (незалежно від того, чи редагував його користувач), повторна синхронізація його ігнорує. Це гарантує збереження всіх нотаток, тегів, значень властивостей і контентної області, доданих вручну.
- **Маппінг полів:** маппінг між полями брокера і властивостями бази Trading Journal є фіксованим і прив'язаний до внутрішніх ідентифікаторів системних властивостей (не до їхніх назв). Перейменування системної властивості користувачем не порушує синхронізацію. Поля брокера, що не мають відповідної системної властивості, не імпортуються.
- **Funding rate (для perpetual futures):** funding rate за весь час утримання позиції вже включений у значення Realized P&L, яке повертає Position History API Binance та Bybit. Окремого запису для funding rate не створюється — він є частиною підсумкового P&L угоди.
- **Валюта:** числові значення (P&L, комісія, обсяг) зберігаються у валюті брокерського акаунту як є (USDT для Binance/Bybit USDT-M, USD або EUR для MT5 залежно від акаунту). Конвертація валют не виконується. Валюта акаунту відображається поруч із числовими значеннями в картці запису.
- **Початкова синхронізація:** при першому підключенні користувач вибирає діапазон дат для завантаження торгової історії. Значення за замовчуванням — останні 90 днів. Максимальна глибина залежатиме від обмежень API: Bybit — до 2 років, Binance — до 30 днів (решту завантажити через CSV-імпорт), MT5 та cTrader — необмежено (залежить від брокера).
- **Помилки синхронізації:** після 3 послідовних невдалих спроб автосинку система відображає in-app повідомлення (дзвіночок) з описом причини (помилка API, прострочений токен, мережева помилка) і посиланням на розділ Налаштування → Інтеграції. Автоматичні спроби продовжуються з інтервалом 30 хвилин до усунення проблеми.
- **Мітки джерела:** синхронізовані записи позначаються міткою підключення (задається користувачем при налаштуванні, наприклад «Binance Main», «IC Markets cTrader», «MT5 Prop»); мітка відображається в картці запису та доступна для фільтрації.
- **Безпека:** Credentials (API-ключі, паролі, OAuth-токени) зберігаються у зашифрованому вигляді; після збереження відображаються лише у замаскованому вигляді.

**Загальне:**

- Всі синхронізовані записи є повноцінними записами бази даних — їх можна редагувати, доповнювати нотатками та контентною областю, видаляти
- Видалення запису у FIX Space не впливає на дані на біржі або в терміналі
- Усі налаштування інтеграцій зосереджені в розділі Налаштування → Інтеграції
- Перепідключення (оновлення credentials): кожне підключення має кнопку «Перепідключити» — для OAuth (cTrader) запускається повторна авторизація, для API-ключів (Binance, Bybit, OKX) та паролів (MT5) відкривається форма з новими даними; після успіху старі credentials замінюються, синхронізація продовжується
- Відкликання (відключення): кнопка «Відключити» зупиняє синхронізацію та видаляє збережені credentials; для OAuth-підключень виконується відкликання токена; збережені записи залишаються в журналі без змін; перед відключенням — діалог підтвердження
- Після 3 послідовних невдалих спроб автосинку система відображає in-app повідомлення (дзвіночок) з описом причини

### Перспективні інтеграції

Interactive Brokers (IBKR) через Flex Query API охоплює акції, опціони, ф'ючерси та Forex. Орієнтований переважно на institutional трейдерів і виходить за межі основної цільової аудиторії платформи, тому не входить до поточного scope, але передбачений архітектурою як можливий наступний крок.
