## 3.12 Інтеграція із зовнішніми сервісами

**Мета інтеграції** — автоматичне отримання закритих угод із брокерських платформ і криптобірж без ручного введення. Це доповнення до CSV-імпорту (розділ 3.11): якщо CSV є універсальним fallback для будь-якого брокера, то API-підключення забезпечує регулярну автоматичну синхронізацію без участі трейдера.

Система підтримує два класи зовнішніх підключень:

- ❌ **Криптовалютні біржі** (Binance, Bybit, OKX) — через публічний REST API з read-only ключами; покривають крипто-ф'ючерси (USDT-M, COIN-M) та спотовий ринок.
- ❌ **CFD/Forex-термінали** (MetaTrader 5, cTrader) — через вбудовані механізми read-only доступу без права торгівлі; покривають Forex, CFD на індекси, метали та енергоносії.

Синхронізовані записи потрапляють до бази Trading Journal як повноцінні записи: їх можна редагувати, доповнювати нотатками та контентом, видаляти. Мітка джерела (наприклад, «Binance», «MT5») зберігається як атрибут запису.

- 📝 Додано до Issue (#166) Вибір типу ринку для синхронізації (Futures / Spot або обидва)

- 📝 Додано до Issue (#166) Відключення інтеграції (збережені записи залишаються в журналі)

### MetaTrader 5

MetaTrader 5 є основною платформою CFD та Forex-брокерів. Підключення здійснюється через Investor Password — спеціальний режим read-only доступу, вбудований у MT5, без права торгівлі та без необхідності встановлення додаткового ПЗ на стороні FIX Space.

**Підтримувані ринки:**

- 📝 Додано до Issue (#167) Forex (EUR/USD, GBP/USD тощо)
- 📝 Додано до Issue (#167) CFD на індекси (S&P 500, DAX, FTSE)
- 📝 Додано до Issue (#167) CFD на метали (Gold, Silver)
- 📝 Додано до Issue (#167) CFD на енергоносії (Oil, Gas)
- 📝 Додано до Issue (#167) Криптовалюта (якщо пропонується брокером)

**Функціональні вимоги:**

- 📝 Додано до Issue (#167) Підключення MT5-акаунту через: login, investor password, broker server name
- 📝 Додано до Issue (#167) Перевірка з'єднання при збереженні налаштувань із відображенням результату
- 📝 Додано до Issue (#167) Вибір діапазону дат для початкової синхронізації (завантаження накопиченої торгової історії)
- 📝 Додано до Issue (#167) Автоматична синхронізація нових закритих угод
- 📝 Додано до Issue (#167) Ручний запуск синхронізації на вимогу
- 📝 Додано до Issue (#167) Зіставлення полів MT5 з властивостями бази Trading Journal: символ, тип (Buy / Sell), обсяг у лотах, ціна відкриття/закриття, час відкриття/закриття, прибуток, своп, комісія, коментар брокера
- 📝 Додано до Issue (#167) Ігнорування дублікатів (ідентифікація за position ID MT5)
- 📝 Додано до Issue (#167) Автоматичне заповнення поля `Account` у синхронізованому записі Trading Journal — підбирається запис Accounts, де поле `Account ID` відповідає login підключеного MT5-акаунту; якщо збіг не знайдений — поле залишається порожнім
- 📝 Додано до Issue (#167) Відображення статусу підключення: активне / помилка автентифікації / відключене
- 📝 Додано до Issue (#167) Відключення інтеграції (збережені записи залишаються в журналі)
- 📝 Додано до Issue (#167) Ліміт: до 5 MT5-підключень на акаунт (різні брокери або різні акаунти одного брокера); кожне підключення отримує довільну мітку-назву

### cTrader

cTrader — платформа CFD та Forex-брокерів (IC Markets, Pepperstone, FxPro та 50+ інших). Підключення здійснюється через cTrader Open API (Spotware) за протоколом OAuth 2.0: користувач авторизує FIX Space у своєму cTrader-акаунті без передачі пароля — лише видаючи обмежені права на читання торгової історії.

**Підтримувані ринки:**

- 📝 Додано до Issue (#168) Forex (EUR/USD, GBP/USD тощо)
- 📝 Додано до Issue (#168) CFD на індекси (S&P 500, DAX, FTSE)
- 📝 Додано до Issue (#168) CFD на метали (Gold, Silver)
- 📝 Додано до Issue (#168) CFD на енергоносії (Oil, Gas)
- 📝 Додано до Issue (#168) Криптовалюта (якщо пропонується брокером)

**Функціональні вимоги:**

- 📝 Додано до Issue (#168) Підключення cTrader-акаунту через OAuth 2.0 — без введення пароля: користувач перенаправляється на сторінку авторизації cTrader і підтверджує доступ
- 📝 Додано до Issue (#168) Перегляд списку торгових акаунтів, доступних через авторизований cTrader ID, та вибір одного або кількох для синхронізації; кожен вибраний акаунт стає окремим підключенням із власною міткою-назвою (за замовчуванням — назва брокерського акаунту з cTrader)
- 📝 Додано до Issue (#168) Вибір діапазону дат для початкової синхронізації (завантаження накопиченої торгової історії через ProtoOADealListReq)
- 📝 Додано до Issue (#168) Автоматична синхронізація нових закритих угод
- 📝 Додано до Issue (#168) Ручний запуск синхронізації на вимогу
- 📝 Додано до Issue (#168) Зіставлення полів cTrader з властивостями бази Trading Journal: символ, напрямок (Buy / Sell), обсяг, ціна відкриття/закриття, час відкриття/закриття, прибуток, своп, комісія
- 📝 Додано до Issue (#168) Ігнорування дублікатів (ідентифікація за position ID cTrader)
- 📝 Додано до Issue (#168) Відображення статусу підключення: активне / токен прострочено / відключене
- 📝 Додано до Issue (#168) При простроченні OAuth-токена — повідомлення у застосунку з пропозицією повторно авторизуватись
- 📝 Додано до Issue (#168) Відключення інтеграції з відкликанням OAuth-токена; збережені записи залишаються в журналі
- 📝 Додано до Issue (#168) Ліміт: необмежена кількість cTrader-акаунтів на акаунт FIX Space (кожен вибраний акаунт після OAuth = окреме підключення)

### Загальні вимоги до всіх інтеграцій

- 📝 Додано до Issue (#166) **Одиниця синхронізації — позиція:** система синхронізує закриті позиції, а не окремі часткові виконання (fills). Одна закрита позиція (від відкриття до повного закриття) = один запис у базі Trading Journal. Ціна входу та виходу розраховуються як середньозважені по всіх fills позиції. Для Binance та Bybit використовується Position History API, який надає агреговані дані; для MT5 — групування по position ID.
- 📝 Додано до Issue (#166) **Конфлікт синхронізації та редагування:** синхронізація лише **створює нові записи** — вона ніколи не оновлює та не перезаписує записи, які вже існують у базі даних. Якщо запис з відповідним position ID вже є в журналі (незалежно від того, чи редагував його користувач), повторна синхронізація його ігнорує. Це гарантує збереження всіх нотаток, тегів, значень властивостей і контентної області, доданих вручну.
- 📝 Додано до Issue (#166) **Маппінг полів:** маппінг між полями брокера і властивостями бази Trading Journal є фіксованим і прив'язаний до внутрішніх ідентифікаторів системних властивостей (не до їхніх назв). Перейменування системної властивості користувачем не порушує синхронізацію. Поля брокера, що не мають відповідної системної властивості, не імпортуються.
- 📝 Додано до Issue (#166) **Funding rate (для perpetual futures):** funding rate за весь час утримання позиції вже включений у значення Realized P&L, яке повертає Position History API Binance та Bybit. Окремого запису для funding rate не створюється — він є частиною підсумкового P&L угоди.
- 📝 Додано до Issue (#166) **Валюта:** числові значення (P&L, комісія, обсяг) зберігаються у валюті брокерського акаунту як є (USDT для Binance/Bybit USDT-M, USD або EUR для MT5 залежно від акаунту). Конвертація валют не виконується. Валюта акаунту відображається поруч із числовими значеннями в картці запису.
- 📝 Додано до Issue (#166) **Початкова синхронізація:** при першому підключенні користувач вибирає діапазон дат для завантаження торгової історії. Значення за замовчуванням — останні 90 днів. Максимальна глибина залежатиме від обмежень API: Bybit — до 2 років, Binance — до 30 днів (решту завантажити через CSV-імпорт), MT5 та cTrader — необмежено (залежить від брокера).
- 📝 Додано до Issue (#169) **Помилки синхронізації:** після 3 послідовних невдалих спроб автосинку система відображає in-app повідомлення (дзвіночок) з описом причини (помилка API, прострочений токен, мережева помилка) і посиланням на розділ Налаштування → Інтеграції. Автоматичні спроби продовжуються з інтервалом 30 хвилин до усунення проблеми.
- 📝 Додано до Issue (#166) **Мітки джерела:** синхронізовані записи позначаються міткою підключення (задається користувачем при налаштуванні, наприклад «Binance Main», «IC Markets cTrader», «MT5 Prop»); мітка відображається в картці запису та доступна для фільтрації.
- 📝 Додано до Issue (#166) **Безпека:** Credentials (API-ключі, паролі, OAuth-токени) зберігаються у зашифрованому вигляді; після збереження відображаються лише у замаскованому вигляді.

**Загальне:**

- 📝 Додано до Issue (#166) Всі синхронізовані записи є повноцінними записами бази даних — їх можна редагувати, доповнювати нотатками та контентною областю, видаляти
- 📝 Додано до Issue (#166) Видалення запису у FIX Space не впливає на дані на біржі або в терміналі
- 📝 Додано до Issue (#170) Усі налаштування інтеграцій зосереджені в розділі Налаштування → Інтеграції
- 📝 Додано до Issue (#170) Перепідключення (оновлення credentials): кожне підключення має кнопку «Перепідключити» — для OAuth (cTrader) запускається повторна авторизація, для API-ключів (Binance, Bybit, OKX) та паролів (MT5) відкривається форма з новими даними; після успіху старі credentials замінюються, синхронізація продовжується
- 📝 Додано до Issue (#170) Відкликання (відключення): кнопка «Відключити» зупиняє синхронізацію та видаляє збережені credentials; для OAuth-підключень виконується відкликання токена; збережені записи залишаються в журналі без змін; перед відключенням — діалог підтвердження
- 📝 Додано до Issue (#169) Після 3 послідовних невдалих спроб автосинку система відображає in-app повідомлення (дзвіночок) з описом причини

### Перспективні інтеграції

Interactive Brokers (IBKR) через Flex Query API охоплює акції, опціони, ф'ючерси та Forex. Орієнтований переважно на institutional трейдерів і виходить за межі основної цільової аудиторії платформи, тому не входить до поточного scope, але передбачений архітектурою як можливий наступний крок.
