## 3.19 Сповіщення

**Сповіщення** — це центр in-app повідомлень про системні події, що потребують уваги користувача. На відміну від email-сповіщень (автентифікація, безпека), in-app сповіщення охоплюють операційні події: помилки синхронізації, збої автоматизацій, завершення фонових операцій. Сповіщення доступні через іконку дзвіночка у шапці застосунку.

Система не генерує сповіщень у відповідь на кожну успішну операцію — лише на події, що потребують реакції або інформування. Успішна синхронізація не породжує сповіщення; її збій — породжує.

**Типи сповіщень:**

| Тип           | Приклад тексту                                                                  |
| ------------- | ------------------------------------------------------------------------------- |
| Помилка       | «Помилка синхронізації з Binance: API-ключ недійсний. Перейдіть до налаштувань» |
| Автоматизація | «Автоматизація "Weekly Review Creation" виконана успішно»                       |
| Інфо          | «Імпорт завершено: 154 записи додано, 12 пропущено»                             |

**Функціональні вимоги:**

- 📝 Додано до Issue (#164) Іконка дзвіночка у шапці відображає badge-лічильник непрочитаних сповіщень; зникає після того, як усі сповіщення прочитані
- 📝 Додано до Issue (#186) Клік на дзвіночок відкриває flyout-панель із переліком сповіщень, відсортованих за датою (нові — зверху)
- 📝 Додано до Issue (#163) Кожне сповіщення містить: тип (іконка), текст події, відносний час («5 хв тому», «вчора»), стан (непрочитане / прочитане)
- 📝 Додано до Issue (#164) Непрочитані сповіщення візуально виділені; після відкриття flyout-панелі всі сповіщення позначаються як прочитані автоматично
- 📝 Додано до Issue (#164) Клік на сповіщення переходить до джерела події (відповідне підключення в Налаштуваннях, конкретний запис, деталі імпорту)
- 📝 Додано до Issue (#186) Кнопка «Позначити всі прочитаними» у заголовку панелі
- 📝 Додано до Issue (#163) Зберігається не більше 50 сповіщень; при перевищенні — найстаріші видаляються автоматично
- 📝 Додано до Issue (#163) Сповіщення зберігаються між сесіями та пристроями
