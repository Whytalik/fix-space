## 3.19 Сповіщення (Фактична реалізація)

Сповіщення — це центр in-app повідомлень про системні події, що потребують уваги користувача (інфо-повідомлення, попередження або помилки).

**Фактичні функціональні вимоги та логіка роботи:**

- [x] **Типи сповіщень:** Підтримуються типи сповіщень `INFO`, `WARNING` та `ERROR`.
- [x] **Структура сповіщення:** Кожне сповіщення містить: унікальний `id`, текст повідомлення (`text`), опціональне посилання на джерело події (`link`), статус прочитання (`read`) та дату створення (`createdAt`).
- [x] **Отримання сповіщень:** Ендпоінт `GET /notifications` повертає список усіх сповіщень користувача, відсортованих за часом створення (нові зверху).
- [x] **Badge непрочитаних:** Ендпоінт `GET /notifications/unread-count` повертає поточну кількість непрочитаних сповіщень для відображення лічильника у шапці інтерфейсу.
- [x] **Прочитання одного сповіщення:** Ендпоінт `PATCH /notifications/:id/read` маркує конкретне сповіщення як прочитане. Якщо сповіщення не належить користувачу або не існує, повертається `404 Not Found`.
- [x] **Прочитання всіх сповіщень:** Ендпоінт `PATCH /notifications/mark-all-as-read` маркує всі сповіщення користувача як прочитані.
- [x] **Видалення сповіщень:** Ендпоінт `DELETE /notifications` видаляє абсолютно всі сповіщення поточного користувача.
- [x] **Ліміт накопичення (Auto-cleanup):** Користувачу зберігається максимум **50 сповіщень**. При створенні нового сповіщення, якщо ліміт досягнуто, система автоматично видаляє найстаріші сповіщення, залишаючи рівно 50 останніх записів.
- [x] **Збереження стану:** Стан сповіщень повністю персистується у базі даних, зберігаючись між пристроями та сесіями користувача.
