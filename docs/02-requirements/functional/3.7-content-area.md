## 3.7 Вміст запису (Контентна область)

Для трейдера контентна область є місцем, де фіксується якісний контекст: хід думок під час угоди, скріншоти точки входу та виходу, чеклісти передсесійної підготовки, ключові уроки. Ці дані не мають жорсткої структури, але є критично важливими для ретроспективного аналізу та відстеження прогресу.

Вміст організований за трирівневою ієрархією:

- [x] **Рядок** — горизонтальний контейнер, що ділиться на колонки. Рядок може бути вкладений у колонку іншого рядка; максимальна глибина вкладення — 2 рівні.
- [x] **Колонка** — вертикальний розділ усередині рядка (від 1 до 5 колонок); ширина кожної колонки регулюється перетягуванням меж.
- [x] **Компонент** — атомарна одиниця контенту певного типу. Компоненти розміщуються безпосередньо в колонках.

Редагування здійснюється в режимі inline безпосередньо на сторінці запису. Зміни зберігаються автоматично. Для виправлення помилок підтримуються дії скасування та повторення (undo / redo) на основі знімків історії.

**Функціональні вимоги:**

Рядки та колонки:

- [x] Додавання рядка до контентної області
- [x] Вибір кількості колонок у рядку (1–5)
- [x] Зміна ширини колонок шляхом перетягуванням меж (режим inline-resize)
- [x] Вкладення рядка в колонку іншого рядка (максимум 2 рівні вкладення)
- [x] Зміна порядку рядків перетягуванням
- [x] Видалення рядка

Компоненти:

- [x] Додавання компонента безпосередньо до колонки через панель «Бібліотека»
- [x] Редагування вмісту компонента у режимі inline
- [x] Зміна порядку компонентів перетягуванням (всередині колонки та між колонками/рядками)
- [x] Вирівнювання тексту та заголовків: вліво, по центру, вправо, по ширині
- [x] Підтримка рівнів заголовків (H1–H6) в межах одного компонента
- [x] Видалення компонента
- [x] Підтримка зображень: формати PNG, JPG/JPEG, WebP; максимальний розмір файлу — 5 МБ

Редагування та збереження:

- [x] Автоматичне збереження змін контентної області (debounce 700мс)
- [x] Автоматичне збереження знімків (snapshots) контентної області: кожні 5 хвилин або при примусовому збереженні
- [x] Перегляд списку збережених знімків із мітками часу
- [x] Візуальне порівняння поточної версії із будь-яким знімком перед відновленням
- [x] Відновлення контентної області із обраного знімку
