## 3.7 Контентна область запису

**Контентна область** — це вільна зона кожного запису, розташована нижче властивостей. На відміну від властивостей, що зберігають структуровані дані у визначених полях, контентна область призначена для довільного вмісту: розгорнутих нотаток, скріншотів графіків, чеклістів та іншого матеріалу, який не вкладається у формат колонок.

Для трейдера контентна область є місцем, де фіксується якісний контекст: хід думок під час угоди, скріншоти точки входу та виходу, чеклісти передсесійної підготовки, ключові уроки. Ці дані не мають жорсткої структури, але є критично важливими для ретроспективного аналізу та відстеження прогресу.

Вміст організований за чотирирівневою ієрархією:

- [ ] ❌ **Рядок** — горизонтальний контейнер, що ділиться на колонки. Рядок може бути вкладений у колонку іншого рядка; максимальна глибина вкладення — 2 рівні.
- [ ] ❌ **Колонка** — вертикальний розділ усередині рядка (від 1 до 5 колонок); ширина кожної колонки регулюється перетягуванням меж.
- [ ] ❌ **Блок** — іменований контейнер, що групує компоненти в одну логічну одиницю (наприклад, «Аналіз входу» або «Передсесійна рутина»). Назва блоку є обов'язковою. Блок має власне візуальне оформлення: фон, рамку та відступи. Використання блоку є необов'язковим — компонент може розміщуватися безпосередньо в колонці.
- [ ] ❌ **Компонент** — атомарна одиниця контенту певного типу. Може знаходитися як усередині блоку, так і безпосередньо в колонці.

Редагування здійснюється в режимі inline безпосередньо на сторінці запису. Зміни зберігаються автоматично. Для виправлення помилок підтримуються дії скасування та повторення (undo / redo). Система зберігає історію змін контентної області запису.

Бібліотека готових блоків є спільною для всіх просторів користувача. Вона містить системні пресети та власні збережені блоки. Користувач може налаштовувати видимість блоків у бібліотеці.

**Функціональні вимоги:**

Рядки та колонки:

- [ ] Додавання рядка до контентної області
- [ ] Вибір кількості колонок у рядку (1–5)
- [ ] Зміна ширини колонок шляхом перетягуванням меж
- [ ] Вкладення рядка в колонку іншого рядка (максимум 2 рівні вкладення)
- [ ] Зміна порядку рядків перетягуванням
- [ ] Видалення рядка

Блоки:

- [ ] Додавання блоку до колонки через візуальний конструктор
- [ ] Обов'язкове введення назви блоку при його створенні
- [ ] Редагування назви блоку
- [ ] Налаштування візуального оформлення блоку:
- [ ] **Фон**: один із 9 пресетних кольорів (8 акцентних відтінків + «без фону»)
- [ ] **Рамка**: стиль (без рамки / суцільна / штрихова / пунктирна), колір із палітри, товщина (1 px / 2 px / 3 px)
- [ ] **Заокруглення кутів**: none / small (4 px) / medium (8 px) / large (16 px)
- [ ] **Внутрішні відступи**: none / small / medium / large (4 пресети, однакові з усіх сторін)
- [ ] **Колір назви блоку**: успадковує акцентний колір фону або задається окремо з палітри
- [ ] Зміна порядку блоків перетягуванням (у межах колонки та між колонками)
- [ ] Видалення блоку
- [ ] Збереження блоку як власного готового блоку до бібліотеки
- [ ] Використання готового блоку з бібліотеки (системні пресети та власні збережені)
- [ ] Бібліотечні блоки у записі є лише для читання — редагувати їх вміст безпосередньо в записі неможливо
- [ ] Перехід до редагування бібліотечного блоку здійснюється через кнопку «Редагувати блок», яка відкриває загальний редактор бібліотеки
- [ ] При збереженні змін у редакторі бібліотеки пропонуються три варіанти:
- [ ] **Оновити всюди** — оновлення версії блоку в бібліотеці з поширенням змін на всі записи та шаблони, де він використовується
- [ ] **Оновити шаблон** — оновлення версії блоку в бібліотеці без поширення на вже існуючі записи (зміни застосовуються лише при наступних вставках)
- [ ] **Зберегти локально** — вміст поточного екземпляра у запису зберігається як локальна копія; блок у бібліотеці залишається без змін; зв'язок між цим записом і бібліотечним блоком розривається
- [ ] Налаштування видимості блоків у бібліотеці

Компоненти:

- [ ] Додавання компонента безпосередньо до колонки або всередину блоку
- [ ] Редагування вмісту компонента у режимі inline (#85)
- [ ] Зміна порядку компонентів перетягуванням (у межах блоку та між блоками й колонками)
- [ ] Видалення компонента
- [ ] Підтримка типів компонентів визначається у розділі 2.4; для зображень: формати PNG, JPG/JPEG, WebP; максимальний розмір файлу — 5 МБ

Редагування та збереження:

- [ ] Автоматичне збереження змін контентної області
- [ ] Скасування (undo) та повторення (redo) дій; стан зберігається на сервері та доступний після перезавантаження сторінки
- [ ] Автоматичне збереження знімків (snapshots) контентної області: кожні 5 хвилин та при значних змінах — додавання або видалення компонента, або зміна тексту більш ніж на 200 символів за раз (максимум 50 знімків на запис; при досягненні ліміту найстаріший знімок видаляється)
- [ ] Перегляд списку збережених знімків із мітками часу
- [ ] Відновлення контентної області із будь-якого збереженого знімку
