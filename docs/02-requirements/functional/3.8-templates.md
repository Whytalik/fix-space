## 3.8 Шаблони

**Шаблон** — це заготовка запису, що містить попередньо визначений стан: назву, іконку, значення властивостей та структуру контентної області. При створенні запису з шаблону всі ці параметри автоматично переносяться в новий запис — без необхідності заповнювати поля вручну.

Для трейдера шаблони вирішують задачу повторюваних сценаріїв: щоденний звіт за однаковою структурою, фіксація угоди з переднаповненими полями (Session, Strategy, Risk %), передсесійна перевірка з готовим чеклістом. Один раз налаштований шаблон усуває монотонне введення однакових даних щодня.

Кожна база даних має щонайменше один системний шаблон, створений автоматично при ініціалізації простору. Пресетні шаблони відображають типові сценарії використання бази (наприклад, шаблон щоденного звіту в Daily Routine з готовою структурою фаз). Користувач може створювати власні шаблони під специфічні потреби. Контентна область шаблону будується за тими ж правилами, що і контентна область запису: рядки, колонки, блоки та компоненти; блоки з бібліотеки доступні при побудові шаблону.

**Функціональні вимоги:**

- Автоматичне створення системних шаблонів при ініціалізації бази даних
- Перегляд списку шаблонів бази даних
- Створення власного шаблону для бази даних
- Редагування шаблону: назва, іконку, опис, значення властивостей, структура контентної області
- Редагування системних шаблонів (системний шаблон залишається редагованим)
- Скидання системного шаблону до початкового стану («Скинути до стандартного») — відновлює оригінальні значення властивостей і структуру контентної області, встановлені при ініціалізації простору
- Використання блоків із бібліотеки при побудові контентної області шаблону
- Дублювання шаблону
- Видалення шаблону
- Встановлення шаблону за замовчуванням для бази даних
- Вибір шаблону при створенні запису (відображається назва, іконка та опис) або створення запису без шаблону
- Налаштування **Name Pattern** шаблону — рядок із токенами, що обчислюється синхронно в момент створення запису і підставляється у поле `Name`; підтримувані токени:
- `{{today}}` — поточна дата у форматі `DD.MM.YYYY`
- `{{month}}` — назва поточного місяця (January…December)
- `{{month_num}}` — номер місяця (01…12)
- `{{year}}` — поточний рік (2026)
- `{{quarter}}` — номер поточного кварталу (1…4)
- `{{count}}` — кількість активних записів у базі + 1 (наступний порядковий номер; записи у Кошику не враховуються)
- `{{count:PropertyName=value}}` — те саме, але лише серед записів де `PropertyName = value` (наприклад, `{{count:Period Type=Weekly}}` для нумерації тільки weekly-оглядів)
- Якщо Name Pattern не задано — поле `Name` залишається порожнім (поведінка за замовчуванням)
