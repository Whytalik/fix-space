## 3.8 Шаблони

**Шаблон** — це заготовка запису, що містить попередньо визначений стан: назву, іконку, значення властивостей та структуру контентної області. При створенні запису з шаблону всі ці параметри автоматично переносяться в новий запис — без необхідності заповнювати поля вручну.

Для трейдера шаблони вирішують задачу повторюваних сценаріїв: щоденний звіт за однаковою структурою, фіксація угоди з переднаповненими полями (Session, Strategy, Risk %), передсесійна перевірка з готовим чеклістом. Один раз налаштований шаблон усуває монотонне введення однакових даних щодня.

Кожна база даних має щонайменше один системний шаблон, створений автоматично при ініціалізації простору. Пресетні шаблони відображають типові сценарії використання бази (наприклад, шаблон щоденного звіту в Daily Routine з готовою структурою фаз). Користувач може створювати власні шаблони під специфічні потреби. Контентна область шаблону будується за тими ж правилами, що і контентна область запису: рядки, колонки, блоки та компоненти; блоки з бібліотеки доступні при побудові шаблону.

**Функціональні вимоги:**

- 📝 Додано до Issue (#81) Автоматичне створення системних шаблонів при ініціалізації бази даних
- 📝 Додано до Issue (#81) Перегляд списку шаблонів бази даних
- 📝 Додано до Issue (#148) Створення власного шаблону для бази даних
- 📝 Додано до Issue (#148) Редагування шаблону: назва, іконку, опис, значення властивостей, структура контентної області
- 📝 Додано до Issue (#81) Редагування системних шаблонів (системний шаблон залишається редагованим)
- 📝 Додано до Issue (#150) Скидання системного шаблону до початкового стану («Скинути до стандартного») — відновлює оригінальні значення властивостей і структуру контентної області, встановлені при ініціалізації простору
- 📝 Додано до Issue (#144) Використання блоків із бібліотеки при побудові контентної області шаблону
- 📝 Додано до Issue (#148) Дублювання шаблону
- 📝 Додано до Issue (#182) Видалення шаблону
- 📝 Додано до Issue (#82) Встановлення шаблону за замовчуванням для бази даних
- 📝 Додано до Issue (#83) Вибір шаблону при створенні запису (відображається назва, іконка та опис) або створення запису без шаблону
- 📝 Додано до Issue (#149) Налаштування **Name Pattern** шаблону — рядок із токенами, що обчислюється синхронно в момент створення запису і підставляється у поле `Name`; підтримувані токени:
  - 📝 Додано до Issue (#149) `{{today}}` — поточна дата у форматі `DD.MM.YYYY`
  - 📝 Додано до Issue (#149) `{{month}}` — назва поточного місяця (January…December)
  - 📝 Додано до Issue (#149) `{{month_num}}` — номер місяця (01…12)
  - 📝 Додано до Issue (#149) `{{year}}` — поточний рік (2026)
  - 📝 Додано до Issue (#149) `{{quarter}}` — номер поточного кварталу (1…4)
  - 📝 Додано до Issue (#149) `{{count}}` — кількість активних записів у базі + 1 (наступний порядковий номер; записи у Кошику не враховуються)
  - 📝 Додано до Issue (#149) `{{count:PropertyName=value}}` — те саме, але лише серед записів де `PropertyName = value` (наприклад, `{{count:Period Type=Weekly}}` для нумерації тільки weekly-оглядів)
- 📝 Додано до Issue (#149) Якщо Name Pattern не задано — поле `Name` залишається порожнім (поведінка за замовчуванням)
