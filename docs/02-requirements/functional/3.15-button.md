## 3.15 Кнопка <!-- US-108, US-109, US-110, US-111 -->

**Кнопка** — це тип дії з двома формами розміщення. Як **BUTTON-властивість** (колонка бази даних) — кожен запис отримує кнопку прямо в рядку таблиці, доступну без відкриття картки. Як **компонент контентної області** — кнопка розміщується вільно серед блоків тексту, зображень та інших елементів, коли дія логічно прив'язана до конкретного місця в документі, а не до рядка таблиці. В обох випадках кнопка виконує той самий набір налаштованих дій над поточним записом. На відміну від формул (що обчислюють автоматично) та автоматизацій (що спрацьовують при подіях), кнопка запускається виключно вручну. <!-- US-108, US-110 -->

Типовий сценарій: трейдер завершив угоду і натискає «Закрити угоду» — система автоматично заповнює поле «Дата виходу» поточною датою, встановлює статус «Closed» і створює запис у Performance Review. Три дії — один клік, без ручного переходу між базами. <!-- US-108, US-109 -->

**Налаштування кнопки:** <!-- US-108, US-109, US-110, US-111 -->

При додаванні BUTTON-властивості відкривається конфігураційна панель із трьох блоків:

1. **Мітка** — текст, що відображається на кнопці (наприклад, «Закрити угоду», «Логувати помилку»)
2. **Колір** — нейтральний / синій / зелений / червоний / акцентний
3. **Дії** — послідовний список до 3 дій, що виконуються при натисканні

**Доступні дії (no-code, вибір із дропдауну):** <!-- US-109, US-110 -->

| Дія                      | Що робить                                                                                                                                  |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------ |
| Встановити значення поля | Записує в одне обране поле поточного запису: статичне значення, «Поточна дата», «Поточний час» або значення іншого поля цього ж запису     |
| Встановити кілька полів  | Те саме, але одночасно для 2–5 полів поточного запису                                                                                      |
| Створити запис в базі    | Створює новий запис в обраній базі з передачею значень поточного запису у вказані поля нового запису                                       |
| Вставити блок у контент  | Вставляє заздалезідь визначений блок у контентну область поточного запису; доступна лише для BUTTON-компонента (не для BUTTON-властивості) |

Дії виконуються у визначеному порядку. Якщо одна з дій не вдалася — наступні не виконуються, відображається повідомлення про помилку.

**Загальні шаблони кнопок:** <!-- US-109 -->

| Шаблон                 | Дія                                                       |
| ---------------------- | --------------------------------------------------------- |
| Позначити як виконане  | Встановлює CHECKBOX-поле «Виконано» = відмічено           |
| Зафіксувати дату       | Встановлює DATE-поле = поточна дата та час                |
| Скопіювати в іншу базу | Створює запис в обраній базі з перенесенням обраних полів |

**Шаблони для трейдерів:** <!-- US-109 -->

| Шаблон                          | Дія                                                                                       |
| ------------------------------- | ----------------------------------------------------------------------------------------- |
| Закрити угоду                   | Встановлює DATE-поле «Дата виходу» = сьогодні + STATUS-поле «Статус» = «Closed» одночасно |
| Логувати помилку                | Створює запис у базі Mistakes із назвою поточної угоди та поточною датою                  |
| Відправити на ревʼю             | Встановлює CHECKBOX «Для огляду» = відмічено                                              |
| Скопіювати в Performance Review | Створює запис у Performance Review з полями Symbol, Date, P&L та Notes                    |

**Функціональні вимоги:**

Налаштування:

- [ ] При додаванні BUTTON-властивості відкривається конфігураційна панель: мітка та мінімум одна дія є обов'язковими <!-- US-108 -->
- [ ] Для дій «Встановити значення поля» / «Встановити кілька полів» доступні всі властивості поточної бази, крім інших BUTTON- та FORMULA-властивостей <!-- US-109 -->
- [ ] Для дії «Створити запис в базі» доступні всі бази поточного простору; після вибору бази з'являється маппінг полів: яке поле поточного запису потрапляє у яке поле нового <!-- US-109 -->
- [ ] Порядок дій змінюється перетягуванням <!-- US-109 -->
- [ ] Ліміт: до 3 дій на одну кнопку <!-- US-108 -->
- [ ] Опційний діалог підтвердження перед виконанням — налаштовується для кожної кнопки окремо <!-- US-111 -->

Виконання:

- [ ] Кнопка відображається в кожному рядку таблиці та у картці запису <!-- US-108 -->
- [ ] Під час виконання — стан «Виконується…» з індикатором <!-- US-108 -->
- [ ] После успішного завершення — тимчасовий стан «Готово» протягом 2 с; під кнопкою відображається час останнього успішного виконання; цей час зберігається як метадані `ButtonExecution { recordId, propertyId, executedAt }` — не є значенням властивості та не бере участі у фільтрації, сортуванні або формулах <!-- US-109 -->
- [ ] При помилці — повідомлення з коротким описом причини <!-- US-108 -->

Обмеження:

- [ ] BUTTON-властивість не підтримує сортування, фільтрацію або групування — вона є дією, а не значенням <!-- US-108 -->
- [ ] BUTTON-властивість не доступна як вхід у формулах та автоматизаціях <!-- US-108 -->
- [ ] BUTTON-властивості виключені з масового редагування <!-- US-108 -->
- [ ] У шаблонах записів конфігурація BUTTON-властивості не зберігається — кнопки налаштовуються виключно на рівні бази даних <!-- US-108 -->

BUTTON як компонент контентної області:

- [ ] Додається до контентної області запису або шаблону як окремий компонент (поруч із текстом, зображеннями тощо) <!-- US-110 -->
- [ ] Конфігурація (label, color, actions) задається окремо від BUTTON-властивості бази — один і той самий запис може мати і колонку-кнопку, і кнопку в контенті з різними діями <!-- US-110 -->
- [ ] Доступний набір дій — той самий, що й для BUTTON-властивості (встановити значення поля, встановити кілька полів, створити запис в базі), плюс ексклюзивна дія **«Вставити блок у контент»**: вставляє заздалегідь визначений блок у контентну область поточного запису із заданою позицією (`after_last` · `before_first` · `after_section`) та опційним підстановленням мітки часу у назву блоку <!-- US-110 -->
- [ ] У шаблонах конфігурація BUTTON-компонента зберігається разом із контентною областю та переноситься в новий запис при створенні з шаблону <!-- US-110 --> поточної бази, крім інших BUTTON- та FORMULA-властивостей
- [ ] Для дії «Створити запис в базі» доступні всі бази поточного простору; після вибору бази з'являється маппінг полів: яке поле поточного запису потрапляє у яке поле нового
- [ ] Порядок дій змінюється перетягуванням
- [ ] Ліміт: до 3 дій на одну кнопку
- [ ] Опційний діалог підтвердження перед виконанням — налаштовується для кожної кнопки окремо

Виконання:

- [ ] Кнопка відображається в кожному рядку таблиці та у картці запису
- [ ] Під час виконання — стан «Виконується…» з індикатором
- [ ] После успішного завершення — тимчасовий стан «Готово» протягом 2 с; під кнопкою відображається час останнього успішного виконання; цей час зберігається як метадані `ButtonExecution { recordId, propertyId, executedAt }` — не є значенням властивості та не бере участі у фільтрації, сортуванні або формулах
- [ ] При помилці — повідомлення з коротким описом причини

Обмеження:

- [ ] BUTTON-властивість не підтримує сортування, фільтрацію або групування — вона є дією, а не значенням
- [ ] BUTTON-властивість не доступна як вхід у формулах та автоматизаціях
