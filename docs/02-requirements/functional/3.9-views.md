## 3.9 Подання (Views)

**Подання** — це серверно збережена конфігурація бази даних, що об'єднує всі параметри відображення та фільтрації: активні фільтри, сортування, групування, пошуковий запит, видимість і ширину стовпців, розмір сторінки. Оскільки всі налаштування зберігаються на сервері, вони залишаються незмінними між сесіями та пристроями. Зміна будь-якого параметра не впливає на самі дані.

Табличне подання є єдиним типом відображення в системі. Подання може бути заблоковане, щоб захистити поточну конфігурацію від випадкових змін.

**Функціональні вимоги:**

Відображення:

- 📝 Додано до Issue (#75) Відображення записів у вигляді таблиці зі стовпцями-властивостями
- 📝 Додано до Issue (#78) Показ та приховування властивостей (визначає які стовпці відображаються)
- 📝 Додано до Issue (#90) Регулювання ширини стовпців перетягуванням меж
- 📝 Додано до Issue (#75) Перемикання режиму перенесення тексту в комірках (wrap / no wrap)
- 📝 Додано до Issue (#92) Закріплення першого стовпця (Name) при горизонтальному прокручуванні
- 📝 Додано до Issue (#75) Серверне збереження всіх налаштувань відображення (доступні з будь-якого пристрою)

Пошук:

- 📝 Додано до Issue (#79) Пошук записів у межах бази даних за назвою та значеннями властивостей
- 📝 Додано до Issue (#79) Пошук починається з 2 введених символів

Фільтрація:

- 📝 Додано до Issue (#76) Фільтрація записів за значеннями властивостей з підтримкою логічних операторів AND / OR між умовами (#130)
- 📝 Додано до Issue (#76) Набір операторів порівняння залежить від типу властивості (специфікація — у розділі 2.2)
- 📝 Додано до Issue (#127) Створення та збереження іменованих фільтр-пресетів (наприклад, «Збиткові угоди», «London session»)
- 📝 Додано до Issue (#127) Застосування та видалення іменованих фільтр-пресетів

Сортування:

- 📝 Додано до Issue (#125) Сортування записів за кількома властивостями або мета-полями одночасно (дата створення, дата оновлення)
- 📝 Додано до Issue (#125) Визначення пріоритету між критеріями сортування
- 📝 Додано до Issue (#125) Вибір напрямку (за зростанням / за спаданням) для кожного критерію

Групування:

- 📝 Додано до Issue (#126) Групування записів за одним критерієм: за мета-полем (дата створення) або за значенням властивості
- 📝 Додано до Issue (#126) При групуванні за властивістю типу DATE — вибір рівня деталізації: День / Тиждень / Місяць / Квартал / Рік; за замовчуванням — Місяць; записи без дати виносяться в окрему групу «Без дати»
- 📝 Додано до Issue (#126) При активному групуванні пагінація вимикається — відображаються всі записи
- 📝 Додано до Issue (#126) Показ та приховування окремих груп
- 📝 Додано до Issue (#126) Зміна кольору фону заголовка групи

Пагінація:

- 📝 Додано до Issue (#124) Вибір розміру сторінки: 10, 25, 50, 75 або 100 записів
- 📝 Додано до Issue (#124) Пагінація вимикається при активному групуванні

Управління поданнями:

- 📝 Додано до Issue (#128) Кожна база даних підтримує до 10 іменованих подань
- 📝 Додано до Issue (#128) Перемикання між поданнями через вкладки у заголовку бази даних
- 📝 Додано до Issue (#128) Створення нового подання
- 📝 Додано до Issue (#128) Перейменування подання
- 📝 Додано до Issue (#128) Дублювання подання (зі збереженням усіх параметрів конфігурації)
- 📝 Додано до Issue (#128) Видалення подання (не дозволяється видалити єдине подання бази даних)
- 📝 Додано до Issue (#128) Встановлення подання за замовчуванням для бази даних (відкривається при кожному вході до бази; при першому відкритті — перше подання у списку)
- 📝 Додано до Issue (#129) Блокування та розблокування налаштувань подання (запобігає випадковій зміні фільтрів, сортування та групування)
