## 3.9 Подання (Views)

**Подання** — це серверно збережена конфігурація бази даних, що об'єднує всі параметри відображення та фільтрації: активні фільтри, сортування, групування, пошуковий запит, видимість і ширину стовпців, розмір сторінки. Оскільки всі налаштування зберігаються на сервері, вони залишаються незмінними між сесіями та пристроями. Зміна будь-якого параметра не впливає на самі дані.

Табличне подання є єдиним типом відображення в системі. Подання може бути заблоковане, щоб захистити поточну конфігурацію від випадкових змін.

**Функціональні вимоги:**

Відображення:

- [x] Відображення записів у вигляді таблиці зі стовпцями-властивостями
- [x] Показ та приховування властивостей (визначає які стовпці відображаються)
- [x] Регулювання ширини стовпців перетягуванням меж
- [x] Перемикання режиму перенесення тексту в комірках (wrap / no wrap)
- [x] Закріплення першого стовпця (Name) при горизонтальному прокручуванні
- [x] Серверне збереження всіх налаштувань відображення (доступні з будь-кого пристрою)

Пошук:

- [x] Пошук записів у межах бази даних за назвою та значеннями властивостей
- [x] Пошук починається з 2 введених символів

Фільтрація:

- [x] Фільтрація записів за значеннями властивостей з підтримкою логічних операторів AND / OR між умовами (#130)
- [x] Набір операторів порівняння залежить від типу властивості (специфікація — у розділі 2.2)
- [x] Створення та збереження іменованих фільтр-пресетів (реалізовано через Подання)
- [x] Застосування та видалення іменованих фільтр-пресетів

Сортування:

- [x] Сортування записів за кількома властивостями або мета-полями одночасно (дата створення, дата оновлення)
- [x] Визначення пріоритету між критеріями сортування
- [x] Вибір напрямку (за зростанням / за спаданням) для кожного критерію

Групування:

- [x] Групування записів за одним критерієм: за мета-полем (дата створення) або за значенням властивості
- [x] При групуванні за властивістю типу DATE — вибір рівня деталізації: День / Тиждень / Місяць / Квартал / Рік; за замовчуванням — Місяць; записи без дати виносяться в окрему групу «Без дати»
- [x] При активному групуванні пагінація вимикається — відображаються всі записи
- [x] Показ та приховування окремих груп
- [x] Зміна кольору фону заголовка групи

Пагінація:

- [x] Вибір розміру сторінки: 10, 25, 50, 75 або 100 записів
- [x] Пагінація вимикається при активному групуванні

Управління поданнями:

- [x] Кожна база даних підтримує до 5 іменованих подань
- [x] Перемикання між поданнями через вкладки у заголовку бази даних
- [x] Створення нового подання
- [x] Перейменування подання
- [x] Дублювання подання (зі збереженням усіх параметрів конфігурації)
- [x] Видалення подання (не дозволяється видалити єдине подання бази даних)
- [x] Встановлення подання за замовчуванням для бази даних (відкриється при кожному вході до бази; при першому відкритті — перше подання у списку)
- [x] Блокування та розблокування налаштувань подання (запобігає випадковій зміні фільтрів, сортування та групування)
