## 3.9 Подання (Views)

**Подання** — це серверно збережена конфігурація бази даних, що об'єднує всі параметри відображення та фільтрації: активні фільтри, сортування, групування, пошуковий запит, видимість і ширину стовпців, розмір сторінки. Оскільки всі налаштування зберігаються на сервері, вони залишаються незмінними між сесіями та пристроями. Зміна будь-якого параметра не впливає на самі дані.

Табличне подання є єдиним типом відображення в системі. Подання може бути заблоковане, щоб захистити поточну конфігурацію від випадкових змін.

**Функціональні вимоги:**

Відображення:

- [ ] Відображення записів у вигляді таблиці зі стовпцями-властивостями
- [ ] Показ та приховування властивостей (визначає які стовпці відображаються)
- [ ] Регулювання ширини стовпців перетягуванням меж
- [ ] Перемикання режиму перенесення тексту в комірках (wrap / no wrap)
- [ ] Закріплення першого стовпця (Name) при горизонтальному прокручуванні
- [ ] Серверне збереження всіх налаштувань відображення (доступні з будь-якого пристрою)

Пошук:

- [ ] Пошук записів у межах бази даних за назвою та значеннями властивостей
- [ ] Пошук починається з 2 введених символів

Фільтрація:

- [ ] Фільтрація записів за значеннями властивостей з підтримкою логічних операторів AND / OR між умовами (#130)
- [ ] Набір операторів порівняння залежить від типу властивості (специфікація — у розділі 2.2)
- [ ] Створення та збереження іменованих фільтр-пресетів (наприклад, «Збиткові угоди», «London session»)
- [ ] Застосування та видалення іменованих фільтр-пресетів

Сортування:

- [ ] Сортування записів за кількома властивостями або мета-полями одночасно (дата створення, дата оновлення)
- [ ] Визначення пріоритету між критеріями сортування
- [ ] Вибір напрямку (за зростанням / за спаданням) для кожного критерію

Групування:

- [ ] Групування записів за одним критерієм: за мета-полем (дата створення) або за значенням властивості
- [ ] При групуванні за властивістю типу DATE — вибір рівня деталізації: День / Тиждень / Місяць / Квартал / Рік; за замовчуванням — Місяць; записи без дати виносяться в окрему групу «Без дати»
- [ ] При активному групуванні пагінація вимикається — відображаються всі записи
- [ ] Показ та приховування окремих груп
- [ ] Зміна кольору фону заголовка групи

Пагінація:

- [ ] Вибір розміру сторінки: 10, 25, 50, 75 або 100 записів
- [ ] Пагінація вимикається при активному групуванні

Управління поданнями:

- [ ] Кожна база даних підтримує до 10 іменованих подань
- [ ] Перемикання між поданнями через вкладки у заголовку бази даних
- [ ] Створення нового подання
- [ ] Перейменування подання
- [ ] Дублювання подання (зі збереженням усіх параметрів конфігурації)
- [ ] Видалення подання (не дозволяється видалити єдине подання бази даних)
- [ ] Встановлення подання за замовчуванням для бази даних (відкривається при кожному вході до бази; при першому відкритті — перше подання у списку)
- [ ] Блокування та розблокування налаштувань подання (запобігає випадковій зміні фільтрів, сортування та групування)
