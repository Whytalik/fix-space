## 3.4 Бази даних

- [ ] **База даних** — це основний контейнер структурованих даних усередині робочого простору. Вона складається з властивостей — визначених полів із фіксованим типом даних — та записів, кожен з яких заповнює ці поля конкретними значеннями. Саме на рівні бази даних трейдер взаємодіє з основним масивом інформації: переглядає, фільтрує, сортує та аналізує записи. <!-- US-024 -->
- [ ] При ініціалізації простору система автоматично створює набір пресет-баз даних, оптимізованих під типові задачі трейдера. Кожна пресет-база має попередньо визначену структуру властивостей і не потребує початкового налаштування — користувач одразу може починати вводити дані. Детальна специфікація властивостей для кожної пресетної бази наведена у [Предметній моделі пресетних даних](../../03-architecture/preset-model.md). Водночас система залишається гнучкою: користувач може створювати власні бази під специфічні потреби, які виходять за межі стандартного набору. <!-- US-024 -->
- [ ] Дублювання бази дозволяє безпечно експериментувати зі структурою — спочатку змінюється копія, і лише після підтвердження — оригінал. <!-- US-029 -->

**Функціональні вимоги:**

- [ ] Автоматичне створення стартового набору баз даних при ініціалізації робочого простору <!-- US-024 -->
- [ ] Перегляд бази даних зі списком записів та значеннями властивостей <!-- US-072 -->
- [ ] Створення нової бази даних із довільною структурою <!-- US-025 -->
- [ ] Редагування назви та іконки бази даних <!-- US-026 -->
- [ ] Переміщення бази даних між секціями <!-- US-023 -->
- [ ] Зміна порядку баз даних усередині секції <!-- US-023 -->
- [ ] Дублювання бази даних (з усіма властивостями та шаблонами; без записів) <!-- US-029 -->
- [ ] Блокування та розблокування структури бази даних (унеможливлює додавання, редагування та видалення властивостей; введення даних у записи залишається доступним) <!-- US-028 -->
- [ ] Видалення бази даних (пресетні бази даних видалити не можна) <!-- US-027 -->
- [ ] Відображення індикатора пресетної (системної) бази даних у сайдбарі та заголовку <!-- US-024, US-027 -->
- [ ] Відображення індикатора заблокованої структури у заголовку бази даних <!-- US-028 -->
- [ ] Встановлення ліміту кількості записів у базі даних (після досягнення ліміту створення нових записів блокується) <!-- US-031 -->
- [ ] Налаштування використання шаблону за замовчуванням при створенні запису <!-- US-067 -->
- [ ] Відображення підсумкових показників у нижній частині кожного стовпця бази даних: <!-- US-030 -->
- [ ] для числових властивостей: кількість, сума, середнє, медіана, мінімум, максимум, діапазон <!-- US-030 -->
- [ ] для властивостей-прапорців: кількість відмічених, кількість невідмічених, відсоток відмічених <!-- US-030 -->
- [ ] для властивостей з вибором (SELECT, STATUS): кількість унікальних значень <!-- US-030 -->
- [ ] для властивостей-дат: найраніша, найпізніша, діапазон <!-- US-030 -->
- [ ] для всіх типів: загальна кількість записів, кількість заповнених, кількість порожніх <!-- US-030 -->
- [ ] Підсумкові показники враховують лише записи, видимі в поточному поданні (з урахуванням активних фільтрів і пошукового запиту) <!-- US-030 -->
