## 3.4 Бази даних (Databases)

**Бази даних** — основний елемент організації даних у системі. Вони дозволяють зберігати записи з різними типами властивостей.

**Функціональні вимоги:**

Відображення та структура:

- [x] Відображення списку баз даних у бічній панелі
- [x] Кожна база даних має назву, іконку та колір (опціонально)
- [x] Підтримка різних типів властивостей (текст, число, дата, вибір, прапорець, формула тощо)
- [x] Можливість додавання, видалення та зміни порядку властивостей

Підсумкові показники:

- [x] Відображення підсумкових показників у нижній частині кожного стовпця бази даних:
- [x] для числових властивостей: кількість, сума, середнє, медіана, мінімум, максимум, діапазон
- [x] для властивостей-прапорців: кількість відмічених, кількість невідмічених, відситок відмічених
- [x] для властивостей з вибором (SELECT, STATUS): кількість унікальних значень
- [x] для властивостей-дат: найраніша, найпізніша, діапазон
- [x] для всіх типів: загальна кількість записів, кількість заповнених, кількість порожніх
- [x] Підсумкові показники враховують лише записи, видимі в поточному поданні (з урахуванням активних фільтрів і пошукового запиту)

Управління базами:

- [x] Створення нової бази даних
- [x] Редагування налаштувань бази даних (назва, іконка, секція)
- [x] Дублювання бази даних (разом із структурою, властивостями, шаблонами та автоматизаціями)
- [x] Видалення бази даних (перенесення до кошика)
- [x] Блокування бази даних для запобігання випадковим змінам структури
