## 3.4 Бази даних

- 📝 Додано до Issue (#75) **База даних** — це основний контейнер структурованих даних усередині робочого простору. Вона складається з властивостей — визначених полів із фіксованим типом даних — та записів, кожен з яких заповнює ці поля конкретними значеннями. Саме на рівні бази даних трейдер взаємодіє з основним масивом інформації: переглядає, фільтрує, сортує та аналізує записи.
- 📝 Додано до Issue (#59) При ініціалізації простору система автоматично створює набір пресет-баз даних, оптимізованих під типові задачі трейдера. Кожна пресет-база має попередньо визначену структуру властивостей і не потребує початкового налаштування — користувач одразу може починати вводити дані. Водночас система зберігає гнучкість: користувач може створювати власні бази під специфічні потреби, які виходять за межі стандартного набору.
- 📝 Додано до Issue (#137) Дублювання бази дозволяє безпечно експериментувати зі структурою — спочатку змінюється копія, і лише після підтвердження — оригінал.

**Функціональні вимоги:**

- 📝 Додано до Issue (#59) Автоматичне створення стартового набору баз даних при ініціалізації робочого простору
- 📝 Додано до Issue (#75) Перегляд бази даних зі списком записів та значеннями властивостей
- 📝 Додано до Issue (#93) Створення нової бази даних із довільною структурою
- 📝 Додано до Issue (#94) Редагування назви та іконки бази даних
- 📝 Додано до Issue (#136) Переміщення бази даних між секціями
- 📝 Додано до Issue (#102) Зміна порядку баз даних усередині секції
- 📝 Додано до Issue (#137) Дублювання бази даних (з усіма властивостями та шаблонами; без записів)
- 📝 Додано до Issue (#117) Блокування та розблокування структури бази даних (унеможливлює додавання, редагування та видалення властивостей; введення даних у записи залишається доступним)
- 📝 Додано до Issue (#95) Видалення бази даних (пресетні бази даних видалити не можна)
- 📝 Додано до Issue (#95) Відображення індикатора пресетної (системної) бази даних у сайдбарі та заголовку
- 📝 Додано до Issue (#117) Відображення індикатора заблокованої структури у заголовку бази даних
- 📝 Додано до Issue (#181) Встановлення ліміту кількості записів у базі даних (після досягнення ліміту створення нових записів блокується)
- 📝 Додано до Issue (#82) Налаштування використання шаблону за замовчуванням при створенні запису
- 📝 Додано до Issue (#118) Відображення підсумкових показників у нижній частині кожного стовпця бази даних:
  - 📝 Додано до Issue (#118) для числових властивостей: кількість, сума, середнє, медіана, мінімум, максимум, діапазон
  - 📝 Додано до Issue (#118) для властивостей-прапорців: кількість відмічених, кількість невідмічених, відсоток відмічених
  - 📝 Додано до Issue (#118) для властивостей з вибором (SELECT, STATUS): кількість унікальних значень
  - 📝 Додано до Issue (#118) для властивостей-дат: найраніша, найпізніша, діапазон
  - 📝 Додано до Issue (#118) для всіх типів: загальна кількість записів, кількість заповнених, кількість порожніх
- 📝 Додано до Issue (#118) Підсумкові показники враховують лише записи, видимі в поточному поданні (з урахуванням активних фільтрів і пошукового запиту)
