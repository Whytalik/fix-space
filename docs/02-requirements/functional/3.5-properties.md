## 3.5 Властивості

- 📝 Додано до Issue (#70) **Властивість** — це іменований стовпець бази даних із фіксованим типом даних. Сукупність властивостей визначає структуру бази: які поля існують, яке значення вони приймають та в якому порядку відображаються. Кожен запис у базі заповнює ці поля конкретними значеннями.
- 📝 Додано до Issue (#71) Кожна властивість має тип, що визначає формат значення та доступні параметри конфігурації. Деякі типи потребують додаткового налаштування при створенні (наприклад, вибір з переліку варіантів або зазначення цільової бази для зв'язку). Повний перелік типів властивостей та їх специфікація наведені в розділі 2.2.
- 📝 Додано до Issue (#71) У пресетних базах даних розрізняються два класи властивостей. **Системні** властивості створені при ініціалізації простору: їх не можна видалити та не можна змінити тип, але можна перейменувати та перемістити — це зберігає цілісність формул і зв'язків. **Користувацькі** властивості додає вручну сам користувач і має над ними повний контроль. У кастомних базах даних усі властивості є користувацькими.
- 📝 Додано до Issue (#71) Властивість **Name** є первинною для всіх баз даних — вона визначає заголовок запису, що відображається в таблиці, пошуку та переходах. Name не може бути видалена або змінена в іншу позицію.

**Функціональні вимоги:**

- 📝 Додано до Issue (#70) Перегляд переліку властивостей бази даних
- 📝 Додано до Issue (#180) Створення нової властивості: вибір типу та введення назви
- 📝 Додано до Issue (#71) Редагування назви властивості; **увага:** перейменування системної властивості, яка використовується у шаблонах через Record Context (`thisRecord.<PropertyName>`), FORMULA-виразах або автоматизаціях, може порушити ці посилання — система не оновлює їх автоматично при перейменуванні
- 📝 Додано до Issue (#138) Зміна типу вже створеної користувацької властивості (значення конвертуються там, де можливо; де конвертація неможлива — замінюються значенням за замовчуванням)
- 📝 Додано до Issue (#71) Налаштування параметрів властивості залежно від типу (специфікація в розділі 2.2)
- 📝 Додано до Issue (#139) Позначення властивості як обов'язкової для заповнення (при збереженні запису без значення обов'язкової властивості відображається попередження)
- 📝 Додано до Issue (#92) Зміна порядку властивостей (визначає порядок стовпців у базі та полів у картці запису)
- 📝 Додано до Issue (#140) Дублювання властивості (створює копію з тим самим типом і налаштуваннями)
- 📝 Додано до Issue (#141) Призначення властивості до іменованої групи (групи відображаються в картці запису як окремі секції)
- 📝 Додано до Issue (#141) Створення, перейменування та видалення груп властивостей
- 📝 Додано до Issue (#141) Зміна порядку груп властивостей
- 📝 Додано до Issue (#141) Налаштування умови видимості групи властивостей: група показується лише коли обрана властивість запису відповідає заданому значенню або набору значень; якщо умова не виконується — вся група прихована в картці запису та у формі редагування (значення наявних полів зберігаються, але не відображаються)
- 📝 Додано до Issue (#91) Видалення користувацької властивості разом з усіма її значеннями в записах
- 📝 Додано до Issue (#91) При видаленні властивості — автоматичне видалення всіх фільтрів, сортувань і групувань у поданнях, що на неї посилаються; перед видаленням відображається попередження з переліком вразливих подань
- 📝 Додано до Issue (#67) Якщо властивість типу RELATION посилається на видалену базу даних — вона переходить у стан «broken»: нові значення не можна додати, наявні відображаються як простий текст із назвою запису без посилання; tooltip повідомляє причину та пропонує видалити або перепризначити властивість
- 📝 Додано до Issue (#71) Заборона видалення та зміни типу системних властивостей пресетних баз даних
- 📝 Додано до Issue (#71) Заборона видалення та зміни властивості Name
