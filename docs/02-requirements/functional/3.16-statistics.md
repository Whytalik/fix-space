## 3.16 Статистика

**Статистика** — це аналітична сторінка, що надає агреговану інформацію поверх даних пресетних баз. На відміну від баз даних, сторінка Статистики не зберігає і не редагує записи — вона лише обчислює та відображає метрики.

Ключова властивість системи — **схема-орієнтована аналітика**: сторінка автоматично адаптується до структури пресетних баз даних. Якщо користувач додав нову властивість із вибором значень — вона без жодного додаткового налаштування з'являється як новий вимір аналізу. Таким чином, розширення доступної статистики відбувається через збагачення схеми бази даних, а не через конфігурацію самої сторінки Статистики.

Сторінка Статистики має два незалежні шари: **Торгова статистика** та **Власні звіти**. Такий розподіл вирішує ключову проблему: кастомна база даних (наприклад, «Weekly Watchlist») не має заздалегідь відомої семантики — система не знає, що означає «перемога» або «результат» у довільній базі. Тому торгові метрики виведені в окремий фіксований шар, а для кастомних баз — надається стандартний шаблон аналітики, що адаптується за назвами полів.

**Шар 1 — Торгова статистика** _(фіксований, завжди присутній)_

Ядро аналітики на основі пресетних баз даних. Структура цього шару не залежить від наявності кастомних баз. Адаптивність тут — лише всередині пресетних баз: нова SELECT або STATUS властивість у Trading Journal автоматично з'являється як новий вимір аналізу, нова кастомна база даних на цей шар не впливає.

**Шар 2 — Власні звіти** _(opt-in, будь-яка база)_

Будь-яку базу даних (пресетну або кастомну) можна підключити до сторінки Статистики через налаштування бази. Після підключення система автоматично генерує стандартний набір візуалізацій: розподіл по SELECT та STATUS властивостях, динаміка NUMBER-полів у часі, кількість записів за обраний період. Назви метрик відповідають назвам властивостей.

**Функціональні вимоги:**

Торгова статистика:

- [ ] Відображення ключових торгових метрик на основі пресетних баз даних
- [ ] Фільтрація всіх метрик за часовим діапазоном
- [ ] Режим порівняння двох часових діапазонів: метрики обох діапазонів відображаються поруч із відносним відхиленням
- [ ] Автоматична генерація розбивок (breakdown) за кожною SELECT та STATUS властивістю пресетних баз — нова властивість з'являється як новий вимір без будь-якого налаштування
- [ ] Відображення поведінкових метрик: динаміка оцінок у часі та зв'язок дотримання правил із результатом
- [ ] Відображення облікових показників по акаунтах: динаміка балансу, drawdown-крива, використання денних лімітів

Власні звіти:

- [ ] Підключення бази даних до Власних звітів через перемикач у налаштуваннях бази
- [ ] Після підключення база з'являється у окремій вкладці Власних звітів на сторінці Статистики
- [ ] Автоматична генерація для кожної підключеної бази: розподіл по SELECT та STATUS властивостях у вигляді діаграм, динаміка NUMBER-полів у часі, кількість записів за обраний діапазон дат
- [ ] Назви метрик і осей відповідають назвам властивостей бази даних
- [ ] Фільтрація за часовим діапазоном застосовується до всіх графіків підключеної бази одночасно
- [ ] Відключення бази від Власних звітів через ті ж налаштування (дані бази не видаляються)
