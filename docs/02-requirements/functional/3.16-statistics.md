## 3.16 Статистика (Фактична реалізація)

Статистика — це аналітичний простір, який динамічно розраховує та агрегує метрики поверх баз даних користувача за вибраний часовий діапазон. Сторінка складається з двох шарів: Торгової статистики (для пресету Trading Journal) та Власних звітів (для будь-якої бази).

---

### 1. Торгова статистика (Trading Journal)

Аналізує результати на основі закритих угод у пресетній базі даних `trading-journal`. Угоди фільтруються за часом закриття (`exitDate`) за параметрами `from` та `to` в UTC.

**Фактичні функціональні вимоги та розраховані метрики:**

- [x] **Загальна кількість угод (Total Trades):** Рахує загальну кількість угод за період.
- [x] **Відсоток перемог (Win Rate):** Частка угод із чистим прибутком (Net P&L > 0) від загальної кількості.
- [x] **Профіт-фактор (Profit Factor):** Відношення суми прибуткових угод (Gross Profit) до абсолютної суми збиткових угод (Gross Loss). Якщо збитків немає, профіт-фактор дорівнює 0.
- [x] **Середній прибуток / збиток (Average Win / Average Loss):** Середнє значення Net P&L окремо для прибуткових (Net P&L > 0) та збиткових (Net P&L < 0) угод.
- [x] **Чистий прибуток (Net P&L):** Сумарний Net P&L за всіма угодами у часовому діапазоні.
- [x] **Крива прибутковості (Equity Curve):** Динаміка балансу у часі у вигляді точок графіка `ChartPointDto` (дата виходу, кумулятивний баланс).
- [x] **Максимальна просадка (Max Drawdown):** Максимальне відсоткове зниження балансу від його локального піку (equity peak) за період.
- [x] **Середній час утримання (Average Holding Time):** Середня різниця в годинах між датою входу та датою виходу угод.
- [x] **Автоматичні розбивки (Breakdowns):** Динамічна генерація аналітики за кожною властивістю типу `SELECT` та `STATUS` у Trading Journal. Для кожного значення розраховується: кількість угод, сумарний Net P&L та Win Rate.

---

### 2. Власні звіти (Custom Reports)

Дозволяє генерувати аналітичні звіти для будь-якої бази даних користувача на основі наявних властивостей.

**Фактичні функціональні вимоги та логіка роботи:**

- [x] **Вкладки баз даних:** Усі бази даних робочого простору автоматично виводяться у вигляді окремих вкладок на сторінці статистики.
- [x] **Аналітика SELECT / STATUS:** Автоматично генерує розподіли значень у вигляді діаграм: підраховує кількість записів, сумарний P&L та Win Rate для кожного окремого тегу/статусу.
- [x] **Аналітика NUMBER-полів:** Розраховує узагальнені числові показники для всіх NUMBER властивостей бази даних (середнє значення, мінімальне, максимальне та сума).
- [x] **Аналітика DATE-полів (Activity Curve):** Відображає графік активності користувача — кількість створених записів у розрізі дат за вибраний діапазон.
- [x] **Глобальна часова фільтрація:** Усі графіки та числові показники в межах обраної вкладки бази фільтруються за датою створення записів відповідно до параметрів `from` та `to` в UTC.
