## 3.10 Пошук

**Глобальний пошук** охоплює всі бази даних поточного робочого простору одночасно. На відміну від пошуку всередині бази (розділ 2.4), глобальний пошук дозволяє знайти запис без необхідності знати, в якій базі він зберігається, — достатньо пам'ятати фрагмент назви, значення властивості або уривок тексту з контентної області.

Для трейдера це вирішує типову задачу: «я пам'ятаю, що десь записував про помилку з FOMO, але не пам'ятаю — у Mistakes чи у Daily Routine». Пошук повертає всі збіги з усіх баз і дозволяє перейти до потрібного запису в один клік.

**Функціональні вимоги:**

- 📝 Додано до Issue (#108) Відкриття пошуку з будь-якої сторінки платформи (клавіатурний скорочення Ctrl+K / Cmd+K)
- 📝 Додано до Issue (#108) Пошук починається з 2 введених символів; результати оновлюються в реальному часі
- 📝 Додано до Issue (#108) Область пошуку: назва запису, значення властивостей, текст контентної області
- 📝 Додано до Issue (#108) Результати згруповані за базами даних з назвою секції та бази
- 📝 Додано до Issue (#108) Відображається до 5 результатів від кожної бази; посилання «показати всі результати в [назва бази]»
- 📝 Додано до Issue (#108) Виділення знайденого фрагменту в рядку результату
- 📝 Додано до Issue (#108) Перехід до запису при кліку на результат
- 📝 Додано до Issue (#108) Відображення нещодавніх пошукових запитів при відкритті порожнього пошуку
