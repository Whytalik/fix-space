## 3.2 Робочий простір

- 📝 Додано до Issue (#59) **Робочий простір** — це верхній рівень організаційної ієрархії платформи. Він є ізольованим контейнером, що об'єднує всі секції, бази даних і записи конкретного користувача. Кожен простір є повністю незалежним: зміни в одному просторі не впливають на інший.
- 📝 Додано до Issue (#96) Можливість створення додаткових просторів і дублювання існуючого вирішує задачу безпечного експериментування зі структурою даних: трейдер може спробувати нову організацію баз даних або властивостей на копії простору, не ризикуючи накопиченими даними в основному. Якщо нова структура підходить — зміни вносяться в основний простір; якщо ні — копія видаляється (#153).
- 📝 Додано до Issue (#116) Кожен простір має **домашню сторінку** — окрему сторінку-дашборд, доступну через кнопку у сайдбарі. Вона є точкою входу до простору та відкривається автоматично при перемиканні між просторами.
- 📝 Додано до Issue (#116) Домашня сторінка складається з чотирьох функціональних блоків:
  - 📝 Додано до Issue (#116) 1. **Індикатор ринкових сесій** — відображає поточний час, день тижня та активні торгові сесії (Токіо, Франкфурт, Лондон, Нью-Йорк) із кольоровими індикаторами, а також час відкриття наступної сесії.
  - 📝 Додано до Issue (#116) 2. **Щоденний воркфлоу** — горизонтальна панель із 4 кроками послідовного процесу трейдера: «Plan Routine → Execute Trade → Reflect Notes → Log Mistakes». Кожен крок відображає статус виконання поточного дня (виконано / не виконано). Натискання на крок відкриває нову сторінку запису у відповідній базі даних.
  - 📝 Додано до Issue (#116) 3. **Сьогодні** — 4 картки (Trading Journal, Daily Routine, Notes, Mistakes) із переліком записів, створених за поточний день, та кнопкою швидкого додавання.
  - 📝 Додано до Issue (#116) 4. **Огляд** — 3 міні-графіки: P&L Curve (динаміка прибутку), Win Rate Dynamics (динаміка відсотку прибуткових угод), RR Deviation (відхилення фактичного RR від запланованого).

**Функціональні вимоги:**

- 📝 Додано до Issue (#59) Автоматичне створення робочого простору при реєстрації
- 📝 Додано до Issue (#59) Автоматичне наповнення простору стартовим пакетом секцій і баз даних
- 📝 Додано до Issue (#116) Перегляд домашньої сторінки (дашборду) поточного простору
- 📝 Додано до Issue (#116) Перехід до домашньої сторінки через кнопку у сайдбарі
- 📝 Додано до Issue (#116) Відображення поточного часу та активних торгових сесій у режимі реального часу
- 📝 Додано до Issue (#116) Відображення статусу виконання щоденного воркфлоу (чи був створений хоча б один запис сьогодні в кожній із 4 баз)
- 📝 Додано до Issue (#116) Швидке створення запису з домашньої сторінки (перехід до бази та відкриття нового запису в режимі редагування)
- 📝 Додано до Issue (#116) Відображення записів, створених за поточний день, у секції «Сьогодні»
- 📝 Додано до Issue (#116) Відображення міні-графіків поточної торгової статистики
- 📝 Додано до Issue (#97) Редагування назви та іконки робочого простору
- 📝 Додано до Issue (#96) Створення додаткового робочого простору
- 📝 Додано до Issue (#153) Дублювання існуючого робочого простору (з усією структурою: секціями, базами даних, властивостями, шаблонами, автоматизаціями; без записів); автоматизації, що посилаються на IntegrationConnection, можуть потребувати ручного налаштування після дублювання — підключення інтеграцій не копіюються
- 📝 Додано до Issue (#152) Перемикання між робочими просторами (відкривається домашня сторінка обраного простору)
- 📝 Додано до Issue (#151) Встановлення основного робочого простору
- 📝 Додано до Issue (#98) Видалення робочого простору (не дозволяється видаляти простір, позначений як основний)
- 📝 Додано до Issue (#96) Ліміт: до 10 робочих просторів на акаунт
