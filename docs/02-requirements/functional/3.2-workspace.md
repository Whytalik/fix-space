## 3.2 Робочий простір

- [ ] **Робочий простір** — це верхній рівень організаційної ієрархії платформи. Він є ізольованим контейнером, що об'єднує всі секції, бази даних і записи конкретного користувача. Кожен простір є повністю незалежним: зміни в одному просторі не впливають на інший.
- [ ] Можливість створення додаткових просторів і дублювання існуючого вирішує задачу безпечного експериментування зі структурою даних: трейдер може спробувати нову організацію баз даних або властивостей на копії простору, не ризикуючи накопиченими даними в основному. Якщо нова структура підходить — зміни вносяться в основний простір; якщо ні — копія видаляється (#153).
- [ ] Кожен простір має **домашню сторінку** — окрему сторінку-дашборд, доступну через кнопку у сайдбарі. Вона є точкою входу до простору та відкривається автоматично при перемиканні між просторами.
- [ ] Домашня сторінка складається з чотирьох функціональних блоків:
- [ ] 1. **Індикатор ринкових сесій** — відображає поточний час, день тижня та активні торгові сесії (Токіо, Франкфурт, Лондон, Нью-Йорк) із кольоровими індикаторами, а також час відкриття наступної сесії.
- [ ] 2. **Щоденний воркфлоу** — горизонтальна панель із 4 кроками послідовного процесу трейдера: «Plan Routine → Execute Trade → Reflect Notes → Log Mistakes». Кожен крок відображає статус виконання поточного дня (виконано / не виконано). Натискання на крок відкриває нову сторінку запису у відповідній базі даних.
- [ ] 3. **Сьогодні** — 4 картки (Trading Journal, Daily Routine, Notes, Mistakes) із переліком записів, створених за поточний день, та кнопкою швидкого додавання.
- [ ] 4. **Огляд** — 3 міні-графіки: P&L Curve (динаміка прибутку), Win Rate Dynamics (динаміка відсотку прибуткових угод), RR Deviation (відхилення фактичного RR від запланованого).

**Функціональні вимоги:**

- [ ] Автоматичне створення робочого простору при реєстрації
- [ ] Автоматичне наповнення простору стартовим пакетом секцій і баз даних
- [ ] Перегляд домашньої сторінки (дашборду) поточного простору
- [ ] Перехід до домашньої сторінки через кнопку у сайдбарі
- [ ] Відображення поточного часу та активних торгових сесій у режимі реального часу
- [ ] Відображення статусу виконання щоденного воркфлоу (чи був створений хоча б один запис сьогодні в кожній із 4 баз)
- [ ] Швидке створення запису з домашньої сторінки (перехід до бази та відкриття нового запису в режимі редагування)
- [ ] Відображення записів, створених за поточний день, у секції «Сьогодні»
- [ ] Відображення міні-графіків поточної торгової статистики
- [ ] Редагування назви та іконки робочого простору
- [ ] Створення додаткового робочого простору
- [ ] Дублювання існуючого робочого простору (з усією структурою: секціями, базами даних, властивостями, шаблонами, автоматизаціями; без записів); автоматизації, що посилаються на IntegrationConnection, можуть потребувати ручного налаштування після дублювання — підключення інтеграцій не копіюються
- [ ] Перемикання між робочими просторами (відкривається домашня сторінка обраного простору)
- [ ] Встановлення основного робочого простору
- [ ] Видалення робочого простору (не дозволяється видаляти простір, позначений як основний)
- [ ] Ліміт: до 5 робочих просторів на акаунт
