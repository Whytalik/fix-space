## 3.21 NUMBER

**Числове поле** — зберігає числове значення з гнучким форматуванням відображення. `prefix`, `suffix` та `currencySymbol` впливають лише на те, як число виглядає — збережене значення завжди є чистим числом.

**Функціональні вимоги:**

- 📝 Додано до Issue (#61) Редагування в режимі inline; відхиляє нечислові символи
- 📝 Додано до Issue (#61) Формат `integer`: десяткова крапка заблокована; при введенні нецілого значення округлюється автоматично
- 📝 Додано до Issue (#61) Формат `currency`: `currencySymbol` відображається перед числом (наприклад, `$150.00`)
- 📝 Додано до Issue (#61) Формат `percentage`: значення відображається з суфіксом `%`
- 📝 Додано до Issue (#61) `prefix` і `suffix` відображаються навколо числа в таблиці та картці (наприклад, `2.5 R`, `0.1 lots`, `×5`)
- 📝 Додано до Issue (#61) `decimalPlaces` визначає кількість знаків після коми при відображенні; збережене значення зберігається з повною точністю
- 📝 Додано до Issue (#61) Порожнє значення відображається як порожня комірка; null форматується як `0` при обчисленнях у формулах та підсумках
- 📝 Додано до Issue (#61) Підтримує масове редагування
