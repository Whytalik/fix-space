## 3.20 TEXT

**Текстове поле** — зберігає рядок або форматований текст. Підтримує два режими: простий однорядковий рядок та rich text із розміткою. Налаштування `urlHandling` визначає поведінку URL у відображенні.

**Функціональні вимоги:**

- [ ] Редагування в режимі inline — безпосередньо в комірці таблиці або полі картки запису
- [ ] У режимі `isRichText: false` — однорядкове текстове поле
- [ ] У режимі `isRichText: true` — відкривається rich text редактор: жирний, курсив, підкреслення, закреслення, код, посилання, виділення кольором
- [ ] `urlHandling: detect` — URL автоматично перетворюються на клікабельні посилання при відображенні
- [ ] `urlHandling: preview` — для підтримуваних URL (зображення, YouTube, інші сервіси) відображається вбудований попередній перегляд
- [ ] Вставка через Ctrl+V зберігає форматування в режимі rich text; Ctrl+Shift+V — вставка без форматування
- [ ] Порожнє значення зберігається як порожній рядок; null форматується як `""` при відображенні
- [ ] Підтримує масове редагування (перезаписати значення для всіх вибраних записів)

**Параметри конфігурації:**

| Параметр       | Тип     | Допустимі значення            | За замовчуванням |
| -------------- | ------- | ----------------------------- | ---------------- |
| `defaultValue` | string  | Будь-який рядок               | `""`             |
| `isRichText`   | boolean | true / false                  | `true`           |
| `urlHandling`  | enum    | `none` · `detect` · `preview` | `"detect"`       |

`urlHandling: none` — URL як звичайний текст; `detect` — автоматичне посилання; `preview` — вбудований попередній перегляд.

**Значення:** рядок або null (null → порожній рядок).

**Оператори фільтрації:** `equals`, `not_equals`, `contains`, `not_contains`, `starts_with`, `ends_with`, `is_empty`, `is_not_empty`
