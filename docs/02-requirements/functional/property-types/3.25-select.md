## 3.25 SELECT

**Поле вибору** — зберігає один або кілька варіантів зі згрупованого списку. Варіанти організовані у іменовані категорії, що відображаються як розділи у випадаючому меню.

**Функціональні вимоги:**

Відображення та вибір:

- 📝 Додано до Issue (#65) Обрані варіанти відображаються як кольорові chip-мітки у комірці таблиці
- 📝 Додано до Issue (#65) При `isMultiSelect: false` — один chip або порожня комірка
- 📝 Додано до Issue (#65) При `isMultiSelect: true` — кілька chips; якщо не вміщуються — обрізаються з лічильником `+N`
- 📝 Додано до Issue (#65) Клік на комірку відкриває dropdown з категоріями та варіантами
- 📝 Додано до Issue (#65) Пошук у dropdown починається з 1 символу; фільтрує по назві варіанту
- 📝 Додано до Issue (#65) Зняття вибору: повторний клік на вже обраний варіант або × на chip

Управління варіантами та категоріями (в налаштуваннях властивості):

- 📝 Додано до Issue (#65) Додавання, перейменування та видалення категорій; видалення категорії вимагає вибору долі її варіантів: перемістити до іншої категорії або видалити
- 📝 Додано до Issue (#65) Перетягування порядку категорій
- 📝 Додано до Issue (#65) Додавання нового варіанту в категорію; редагування назви варіанту
- 📝 Додано до Issue (#65) Призначення кольору варіанту (колірна палітра)
- 📝 Додано до Issue (#65) Призначення іконки варіанту (emoji або системна іконка)
- 📝 Додано до Issue (#65) Перетягування порядку варіантів всередині категорії та між категоріями
- 📝 Додано до Issue (#65) Видалення варіанту: записи з цим значенням отримують null (одиничний вибір) або видаляють це значення з масиву (множинний)
- 📝 Додано до Issue (#65) Підтримує масове редагування
