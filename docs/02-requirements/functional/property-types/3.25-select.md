## 3.25 SELECT

**Поле вибору** — зберігає один або кілька варіантів зі згрупованого списку. Варіанти організовані у іменовані категорії, що відображаються як розділи у випадаючому меню.

**Функціональні вимоги:**

Відображення та вибір:

- [ ] Обрані варіанти відображаються як кольорові chip-мітки у комірці таблиці
- [ ] При `isMultiSelect: false` — один chip або порожня комірка
- [ ] При `isMultiSelect: true` — кілька chips; якщо не вміщуються — обрізаються з лічильником `+N`
- [ ] Клік на комірку відкриває dropdown з категоріями та варіантами
- [ ] Пошук у dropdown починається з 1 символу; фільтрує по назві варіанту
- [ ] Зняття вибору: повторний клік на вже обраний варіант або × на chip

Управління варіантами та категоріями (в налаштуваннях властивості):

- [ ] Додавання, перейменування та видалення категорій; видалення категорії вимагає вибору долі її варіантів: перемістити до іншої категорії або видалити
- [ ] Перетягування порядку категорій
- [ ] Додавання нового варіанту в категорію; редагування назви варіанту
- [ ] Призначення кольору варіанту (колірна палітра)
- [ ] Призначення іконки варіанту (emoji або системна іконка)
- [ ] Перетягування порядку варіантів всередині категорії та між категоріями
- [ ] Видалення варіанту: записи з цим значенням отримують null (одиничний вибір) або видаляють це значення з масиву (множинний)

**Параметри конфігурації:**

| Параметр        | Тип     | За замовчуванням |
| --------------- | ------- | ---------------- |
| `isMultiSelect` | boolean | `false`          |
| `categories`    | array   | `[]`             |

Структура категорії: `{ label: string, options: [{ value: string, color?: string, icon?: string }] }`.

**Значення:** рядок (single) або масив рядків (multi), або null.

**Оператори фільтрації:** `equals`, `not_equals`, `contains`, `not_contains`, `is_empty`, `is_not_empty`
