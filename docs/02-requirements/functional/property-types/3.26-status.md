## 3.26 STATUS

**Поле статусу** — семантичний аналог SELECT з трьома фіксованими категоріями (`todo`, `in_progress`, `complete`). Призначений для відстеження прогресу запису; категорії мають стабільну семантику в усіх поданнях та інтеграціях.

**Функціональні вимоги:**

Відображення та вибір:

- 📝 Додано до Issue (#66) Відображається як кольоровий pill з назвою поточної опції
- 📝 Додано до Issue (#66) Клік відкриває dropdown, розбитий на три чітко відокремлені секції з заголовками категорій
- 📝 Додано до Issue (#66) Колір заголовка секції фіксований за семантикою: `todo` — сірий, `in_progress` — жовтий/помаранчевий, `complete` — зелений
- 📝 Додано до Issue (#66) Зміна статусу в один клік з таблиці; обрана опція закриває dropdown

Управління (в налаштуваннях властивості):

- 📝 Додано до Issue (#66) Відображувана назва кожної категорії (`label`) редагується: наприклад, `todo` → «Open», «Planned», «Backlog»
- 📝 Додано до Issue (#66) Внутрішній ідентифікатор категорії (`todo` / `in_progress` / `complete`) незмінний
- 📝 Додано до Issue (#66) Додавання, перейменування та видалення опцій всередині кожної категорії
- 📝 Додано до Issue (#66) Перетягування порядку опцій всередині категорії; переміщення опцій між категоріями
- 📝 Додано до Issue (#66) Видалення опції: записи з цим значенням отримують `defaultOption` тієї ж категорії
- 📝 Додано до Issue (#66) Встановлення `defaultOption` на рівні всієї властивості та на рівні кожної категорії окремо
- 📝 Додано до Issue (#66) Підтримує масове редагування
