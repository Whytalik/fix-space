## 3.26 STATUS

**Поле статусу** — семантичний аналог SELECT з трьома фіксованими категоріями (`todo`, `in_progress`, `complete`). Призначений для відстеження прогресу запису; категорії мають стабільну семантику в усіх поданнях та інтеграціях.

**Функціональні вимоги:**

Відображення та вибір:

- [x] Відображається як кольоровий pill з назвою поточної опції
- [x] Клік відкриває dropdown, розбитий на три чітко відокремлені секції з заголовками категорій
- [x] Колір заголовка секції фіксований за семантикою: `todo` — сірий, `in_progress` — жовтий/помаранчевий, `complete` — зелений
- [x] Зміна статусу в один клік з таблиці; обрана опція закриває dropdown

Управління (в налаштуваннях властивості):

- [x] Відображувана назва кожної категорії (`label`) редагується: наприклад, `todo` → «Open», «Planned», «Backlog»
- [x] Внутрішній ідентифікатор категорії (`todo` / `in_progress` / `complete`) незмінний
- [x] Додавання, перейменування та видалення опцій всередині кожної категорії
- [x] Перетягування порядку опцій всередині категорії; переміщення опцій між категоріями
- [x] Видалення опції: записи з цим значенням отримують `defaultOption` тієї ж категорії
- [x] Встановлення `defaultOption` на рівні всієї властивості та на рівні кожної категорії окремо

**Параметри конфігурації:**

| Параметр        | Тип    | За замовчуванням       |
| --------------- | ------ | ---------------------- |
| `defaultOption` | string | `"Not started"`        |
| `categories`    | array  | 3 стандартні категорії |

Структура: `{ category: "todo" | "in_progress" | "complete", label: string, defaultOption?: string, options: [{ name: string, color: string, icon?: string }] }`.

**Значення:** рядок (назва опції) або null.

**Оператори фільтрації:** `equals`, `not_equals`, `is_empty`, `is_not_empty`
