## 3.29 PROGRESS

**Поле прогресу** — числове значення у заданому діапазоні `[min, max]`. Відображається як горизонтальний прогрес-бар із опційним числовим значенням поруч.

**Функціональні вимоги:**

- [x] Відображається як горизонтальний бар заповнення; відсоток заповнення = `(value - min) / (max - min) × 100%`
- [x] Якщо `showLabel: true` — числове значення відображається поруч із баром (праворуч або під баром залежно від ширини комірки)
- [x] Редагування: клік на комірку відкриває slider та числове поле введення
- [x] Drag по slider або введення числа вручну; значення обрізається до `[min, max]`
- [x] Крок зміни slider відповідає `step`; клавіші ←/→ змінюють значення на один `step`
- [x] Колір бару визначається масивом `thresholds`: перевіряються зліва направо, застосовується перший поріг, для якого `value ≤ upTo`; якщо жоден не підходить — використовується останній поріг; при порожньому `thresholds` — акцентний колір поточної теми
- [x] null відображається як порожній бар (0% заповнення)

**Параметри конфігурації:**

| Параметр       | Тип            | За замовчуванням |
| -------------- | -------------- | ---------------- |
| `defaultValue` | number \| null | `null`           |
| `minValue`     | number         | `0`              |
| `maxValue`     | number         | `100`            |
| `step`         | number         | `1`              |
| `showLabel`    | boolean        | `true`           |
| `thresholds`   | array          | `[]`             |

Threshold: `{ upTo: number, color: string }`.

**Значення:** число в діапазоні [min, max] або null.

**Оператори фільтрації:** `equals`, `not_equals`, `greater_than`, `less_than`, `greater_than_or_equal`, `less_than_or_equal`, `is_empty`, `is_not_empty`
