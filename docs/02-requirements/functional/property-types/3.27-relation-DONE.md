## 3.27 RELATION

**Поле зв'язку** — посилається на один або кілька записів іншої бази даних у межах того самого робочого простору. При видаленні цільової бази переходить у стан «broken».

**Функціональні вимоги:**

Відображення та вибір:

- [x] Пов'язані записи відображаються як pill-chips із назвою та іконкою запису в комірці таблиці
- [x] Клік на комірку відкриває пошуковий діалог: поле введення + список записів цільової бази
- [x] Пошук за назвою запису починається з 1 символу; результати оновлюються в реальному часі
- [x] При `multiple: false` — вибір одного запису автоматично закриває діалог
- [x] При `multiple: true` — мультивибір через чекбокси; підтверджується кнопкою «Apply Changes»
- [x] Видалення окремого зв'язку: × на chip у комірці або зняття вибору в діалозі
- [x] Навігація до пов'язаного запису: клік на chip відкриває запис; Ctrl+клік — у новій вкладці
- [x] Якщо пов'язаний запис видалено — значення очищується автоматично

Стан «Broken» (цільова база видалена):

- [x] Нові значення додати неможливо; поле відображається у режимі лише читання
- [x] Наявні збережені значення відображаються як plain text (ID) із позначкою про видалення
- [x] Повідомляє причину: «Цільова база даних видалена» (через стилізацію та позначку)

**Параметри конфігурації:**

| Параметр          | Тип           | За замовчуванням |
| ----------------- | ------------- | ---------------- |
| `relatedEntityId` | string (UUID) | —                |
| `multiple`        | boolean       | `false`          |

**Значення:** UUID або масив UUID, або null.

**Оператори фільтрації:** `equals`, `not_equals`, `is_empty`, `is_not_empty`

Якщо RELATION посилається на видалену базу — переходить у стан «broken»: нові значення неможливі, наявні відображаються як plain text.
