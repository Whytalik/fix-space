## 3.27 RELATION

**Поле зв'язку** — посилається на один або кілька записів іншої бази даних у межах того самого робочого простору. При видаленні цільової бази переходить у стан «broken».

**Функціональні вимоги:**

Відображення та вибір:

- 📝 Додано до Issue (#67) Пов'язані записи відображаються як pill-chips із назвою запису в комірці таблиці
- 📝 Додано до Issue (#67) Клік на комірку відкриває пошуковий діалог: поле введення + список записів цільової бази
- 📝 Додано до Issue (#67) Пошук за назвою запису починається з 1 символу; результати оновлюються в реальному часі
- 📝 Додано до Issue (#67) При `multiple: false` — вибір одного запису автоматично закриває діалог
- 📝 Додано до Issue (#67) При `multiple: true` — мультивибір через чекбокси; підтверджується кнопкою «Застосувати»
- 📝 Додано до Issue (#67) Видалення окремого зв'язку: × на chip у комірці або зняття вибору в діалозі
- 📝 Додано до Issue (#67) Навігація до пов'язаного запису: клік на chip відкриває запис; Ctrl+клік — у новій вкладці
- 📝 Додано до Issue (#190) Якщо пов'язаний запис переміщено до Кошика — chip відображається з приглушеним виглядом (без посилання); після відновлення з Кошика — chip повертається до звичайного стану; після постійного видалення — значення очищується автоматично

Стан «Broken» (цільова база видалена):

- 📝 Додано до Issue (#67) Нові значення додати неможливо; поле відображається у режимі лише читання
- 📝 Додано до Issue (#67) Наявні збережені значення відображаються як plain text із назвою запису без посилання
- 📝 Додано до Issue (#67) Tooltip над комірками повідомляє причину: «Цільова база даних видалена»; пропонує видалити або перепризначити властивість на іншу базу

Масове редагування:

- 📝 Додано до Issue (#67) Підтримується лише очищення (встановити порожнє значення); заміна конкретних значень у масовому режимі не підтримується
