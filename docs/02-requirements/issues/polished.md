**Milestone:** Polished

- [ ] Анімація згортання/розгортання (плавний перехід)

### Technical Notes

**Web:**

- Стан зберігається у профілі користувача (Settings) або окремій таблиці метаданих.
- Рекомендується використовувати `localStorage` як первинний кеш та синхронізувати з API.
- API Endpoint: `PATCH /api/sections/:id` з полем `isCollapsed: boolean`.

### Implementation Checklist

**Backend (API):**

- [ ] Додати поле `isCollapsed` до моделі `Section` у Prisma
- [ ] Оновити `UpdateSectionDto` та сервіс

**Frontend (Web):**

- [ ] Додати стан `isCollapsed` до компонента `SidebarSection`
- [ ] Реалізувати persistence через API call при перемиканні

### Functional Requirements

- [ ] **[§3.3]** Секції залишаються необхідним рівнем організації незалежно від кількості баз даних: зі зростанням їх числа плоский список стає незручним для навігації, тоді як секції дозволяють зберігати чітку структуру. Можливість згортати секції в сайдбарі зменшує когнітивне навантаження під час активної торгової сесії — видно лише те, що потрібно зараз.
- [ ] **[§3.3]** Згортання та розгортання секції в сайдбарі

## #138 [Property] Change property type with value conversion

**State:** OPEN
**Milestone:** Polished
**Labels:** `feature`, `mod:property`, `layer:api`, `layer:web`, `priority:should`

### User Story

**US-037:** Як авторизований користувач, я хочу змінювати тип кастомної властивості.

### Description

Реалізація можливості зміни типу існуючої властивості без видалення даних, де це можливо. Система повинна намагатися конвертувати існуючі значення записів у новий формат (наприклад, NUMBER у TEXT). Якщо конвертація неможлива (наприклад, TEXT «abc» у NUMBER), значення замінюється на порожнє (null) або дефолтне.

### Acceptance Criteria

- [ ] Вибір нового типу властивості у вікні налаштувань
- [ ] Конвертація значень:
  - Any → TEXT: завжди успішно (string representation)
  - TEXT (numeric) → NUMBER: конвертація, інакше null
  - DATE → TEXT: формат ISO або локальний
  - SELECT/STATUS → TEXT: назва обраної опції
- [ ] Де конвертація неможлива — значення стає null
- [ ] Попередження про можливу втрату даних перед підтвердженням
- [ ] Системні властивості: зміна типу заблокована

### Technical Notes

**Backend:**

- Логіка конвертації на рівні `PropertyValueService`
- SQL-міграція або масове оновлення значень у `PropertyValue` таблиці
- Endpoint: `PATCH /api/properties/:id/type`
- Payload: `{ newType: PropertyType }`

### Implementation Checklist

**Backend (API):**

- [ ] Створити `PropertyTypeConverter` утиліту
- [ ] Реалізувати транзакційне оновлення всіх значень властивості
- [ ] Додати валідацію на заборону зміни типу системних властивостей

**Frontend (Web):**

- [ ] Оновити діалог налаштувань властивості
- [ ] Додати модальне вікно підтвердження з переліком наслідків

### Functional Requirements

- [ ] **[§3.5]** Зміна типу вже створеної користувацької властивості (значення конвертуються там, де можливо; де конвертація неможлива — замінюються значенням за замовчуванням)

### References

- Functional: §3.5 Властивості

## #149 [Template] Dynamic name tokens in template ({{today}}, {{count}}, etc.)

**State:** OPEN
