# @fixspace/domain

Shared DTOs, entities, enums, and property-type interfaces for the FIX Space monorepo.
Acts as the typed contract between the API and the web layers — the API validates
and produces these shapes; the web consumes them.

The package compiles to `dist/` via `tsc` and exposes two export paths:

- `@fixspace/domain` — all DTOs, entities, and types (main entry)
- `@fixspace/domain/enums` — enums and runtime constants without class-validator overhead

## Structure

```
packages/domain/src/
├── auth/                    # Login, register, verify, reset DTOs + session DTO
├── automation/              # Automation DTOs, entities, trigger/status enums
├── common/                  # Shared utility DTOs (e.g., DuplicateOptionsDto)
├── database/                # Database DTOs and entity
├── import-mapping/          # ImportMapping and ImportHistory DTOs
├── integration-connection/  # IntegrationConnection DTOs + service/status enums
├── notification/            # Notification response DTO and entity
├── property/
│   ├── dto/                 # CreatePropertyDto, UpdatePropertyDto, PropertyResponseDto
│   ├── entities/            # Property entity
│   ├── property-type.enum.ts
│   └── types/               # Per-type config interfaces + constants (one file each)
├── property-group/          # PropertyGroup DTOs and entity
├── property-value/          # PropertyValue DTOs and entity
├── record/                  # Record DTOs, filter/sort/group DTOs, mass-update, search
├── record-content/          # RecordContent DTOs, snapshot DTO, block-type enum
├── section/                 # Section DTOs and entity
├── settings/                # Settings DTOs, entity, and per-category interfaces + constants
├── space/                   # Space DTOs and entity
├── statistics/              # Custom reporting, breakdowns, and metrics DTOs
├── template/                # Template DTOs and entity
├── template-property-value/ # TemplatePropertyValue DTOs and entity
├── user/                    # User DTOs and entity
├── view/                    # View DTOs and entity
├── generated/               # i18n message keys (auto-generated by nestjs-i18n — do not edit)
├── enums.ts                 # Secondary barrel: enums + constants only
└── entry.ts                 # Primary barrel: all exports
```

## Property type configs

Each property type has a typed `config` object used in `CreatePropertyDto` and
`PropertyResponseDto`. The config shape is enforced by a discriminated union in
`UpdatePropertyDto`. Each type also exports a `DEFAULT_*_PROPERTY` constant.

| Type       | Interface          | Key config fields                                                                    |
| ---------- | ------------------ | ------------------------------------------------------------------------------------ |
| `TEXT`     | `TextProperty`     | `defaultValue`                                                                       |
| `NUMBER`   | `NumberProperty`   | `defaultValue`, `format`, `decimalPlaces`, `currencySymbol`, `prefix`, `suffix`      |
| `DATE`     | `DateProperty`     | `defaultValue`, `format`, `includeTime`, `timeFormat`                                |
| `CHECKBOX` | `CheckboxProperty` | `defaultValue`                                                                       |
| `DURATION` | `DurationProperty` | `defaultValue`, `format` (`HH:mm` · `HH:mm:ss` · `Xh Ym` · `minutes` · `seconds`)    |
| `SELECT`   | `SelectProperty`   | `isMultiSelect`, `categories: SelectCategory[]`                                      |
| `STATUS`   | `StatusProperty`   | `defaultOption`, `categories: StatusCategoryConfig[]`                                |
| `RELATION` | `RelationProperty` | `relatedEntityId`, `multiple`, `sourceDatabaseType`                                  |
| `FORMULA`  | `FormulaProperty`  | `expression`, `resultType`                                                           |
| `RATING`   | `RatingProperty`   | `defaultValue`, `maxStars`, `allowHalf`                                              |
| `PROGRESS` | `ProgressProperty` | `defaultValue`, `min`, `max`, `step`, `showLabel`, `thresholds: ProgressThreshold[]` |

## Constants exported from `enums.ts`

The secondary export path (`@fixspace/domain/enums`) exposes runtime constants
alongside the enums — useful in client-side code where importing class-validator
decorators would add unnecessary bundle weight:

```ts
import {
  PropertyType,
  FilterLogic,
  DEFAULT_TEXT_PROPERTY,
  DEFAULT_STATUS_PROPERTY,
  STATUS_CATEGORY_VALUES,
  NUMBER_FORMAT_VALUES,
  DATA_FORMATS_VALUES,
  DEFAULT_SECTION_SETTINGS,
} from "@fixspace/domain/enums";
```

## Usage

```ts
// API — runtime import (validators run at request time)
import {
  CreatePropertyDto,
  PropertyResponseDto,
  PropertyType,
} from "@fixspace/domain";

// Web — type-only import (devDependency; no runtime bundle cost)
import type { RecordResponseDto, ViewResponseDto } from "@fixspace/domain";

// Web — enum/constant import (values needed at runtime in browser)
import { FilterLogic, PropertyType } from "@fixspace/domain/enums";
```

## Validation

Validators use `class-validator` with `nestjs-i18n` message keys. This means
validation error messages are automatically translated based on the request's
`Accept-Language` header. Supported locales: `uk` (default), `en`.

Key rules:

- `@IsOptional()` must come before all other validators on optional fields
- `IntegrationConnectionResponseDto` deliberately omits `credentials` — credentials are write-only
- `generated/i18n.generated.ts` is auto-generated by `nestjs-i18n` — do not edit manually
- RELATION `sourceDatabaseType` is a symbolic reference resolved at runtime by `InitializeUserSpaceUseCase`
