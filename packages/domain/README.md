# `@nucleus/domain`

Shared DTOs, entities, and property-type interfaces for the Nucleus monorepo. Acts as the contract between the API and web layers.

## Structure

```
packages/domain/
├── src/
│   ├── auth/                    # Auth DTOs, session DTO, auth entities
│   ├── automation/              # Automation DTOs and entities
│   ├── button-execution/        # ButtonExecution DTO and entity
│   ├── content-block-library/   # ContentBlockLibrary DTOs and entity
│   ├── database/                # Database DTOs and entity
│   ├── import-mapping/          # ImportMapping and ImportHistory DTOs
│   ├── integration-connection/  # IntegrationConnection DTOs and enums
│   ├── notification/            # Notification DTO and entity
│   ├── onboarding/              # OnboardingProgress DTOs and entity
│   ├── property/
│   │   ├── dto/                 # Create/update/response DTOs
│   │   ├── entities/            # Property entity
│   │   └── types/               # Per-type config interfaces (text, number, select, …)
│   ├── property-group/          # PropertyGroup DTOs and entity
│   ├── property-value/          # PropertyValue DTOs and entity
│   ├── record/                  # Record DTOs, filters, sort, group, mass-update, search
│   ├── record-content/          # RecordContent DTOs, snapshot DTO, block-type enum
│   ├── section/                 # Section DTOs and entity
│   ├── settings/                # Settings DTOs, entity, and per-category interfaces
│   ├── space/                   # Space DTOs and entity
│   ├── template/                # Template DTOs and entity
│   ├── template-property-value/ # TemplatePropertyValue DTOs and entity
│   ├── user/                    # User DTOs and entity
│   ├── view/                    # View DTOs and entity
│   ├── generated/               # i18n message keys (auto-generated by nestjs-i18n)
│   └── entry.ts                 # Single barrel export
└── tsconfig.json
```

### Property type configs

Each property type has a typed `config` interface used in `CreatePropertyDto` / `PropertyResponseDto`. All interfaces are exported from `@nucleus/domain`.

| Type       | Interface          | Key fields                                                                           |
| ---------- | ------------------ | ------------------------------------------------------------------------------------ |
| `TEXT`     | `TextProperty`     | `defaultValue`, `isRichText`, `urlHandling`                                          |
| `NUMBER`   | `NumberProperty`   | `defaultValue`, `format`, `decimalPlaces`, `currencySymbol`, `prefix`, `suffix`      |
| `DATE`     | `DateProperty`     | `defaultValue`, `format`, `includeTime`, `timeFormat`                                |
| `CHECKBOX` | `CheckboxProperty` | `defaultValue`                                                                       |
| `DURATION` | `DurationProperty` | `defaultValue`, `format` (HH:mm · HH:mm:ss · Xh Ym · minutes · seconds)              |
| `SELECT`   | `SelectProperty`   | `isMultiSelect`, `categories: SelectCategory[]`                                      |
| `STATUS`   | `StatusProperty`   | `defaultOption`, `categories: StatusCategoryConfig[]`                                |
| `RELATION` | `RelationProperty` | `relatedEntityId`, `multiple`                                                        |
| `FORMULA`  | `FormulaProperty`  | `formula`, `output: FormulaOutput`                                                   |
| `RATING`   | `RatingProperty`   | `defaultValue`, `maxStars`, `allowHalf`                                              |
| `PROGRESS` | `ProgressProperty` | `defaultValue`, `min`, `max`, `step`, `showLabel`, `thresholds: ProgressThreshold[]` |
| `BUTTON`   | `ButtonProperty`   | `label`, `color`, `confirmDialog`, `actions: ButtonAction[]`                         |

Each type also exports a `DEFAULT_*_PROPERTY` constant with safe defaults.

## Usage

```ts
// API — runtime import
import {
  CreatePropertyDto,
  PropertyResponseDto,
  AutomationTrigger,
} from "@nucleus/domain";

// Web — type-only import (devDependency)
import type { RecordResponseDto, ViewResponseDto } from "@nucleus/domain";
```

## Commands

```bash
# Build (required before first dev session if dist/ is missing)
pnpm --filter @nucleus/domain build

# Watch-compile on save (run alongside turbo dev)
pnpm --filter @nucleus/domain dev

# Lint
pnpm --filter @nucleus/domain lint
```

## Notes

- Compiles to `dist/` via `tsc`. The `dist/` directory is git-ignored — run `build` before the first dev session.
- `@nucleus/domain` is a `devDependency` in `@nucleus/web` (types only at build time). It is a runtime dependency in `@nucleus/api`.
- Validators use `class-validator` with `nestjs-i18n` message keys — validation errors are automatically translated based on the `Accept-Language` header.
- `@IsOptional()` must come before other validators on optional fields.
- `IntegrationConnectionResponseDto` deliberately omits the `credentials` field — credentials are write-only and never returned by the API.
- `generated/i18n.generated.ts` is auto-generated by `nestjs-i18n` — do not edit manually.
